故事是这样的。

前阵子我发公众号文章，每次写完稿子，真正的折磨才刚开始。

排版要用专门的工具转一遍，图片要一张一张手动上传到微信素材库，封面要单独处理，最后还要手写一个 JSON 推到草稿箱。写稿子可能花两小时，这些体力活又要再花半小时。

图片

而且每一步都是断的，得自己搬货到下一站，像流水线上的工人，只不过这条流水线是我自己用腿跑的。

后来我发现了一个开源工具叫 md2wechat，能把 Markdown 文章转成微信公众号的排版格式，还支持各种主题。

但问题是，它只解决了「排版」这一步。

我还需要生成插图，用 ChatGPT。需要上传图片，拿到微信的 URL。需要推草稿，组装成微信要求的 JSON 格式。四件事，四个独立工具，手动串联，任何一步出错整条链路就断了。

图片

我当时就想，既然每个环节都有对应的 CLI 工具，为什么不把它们串起来？

于是我打开了 Claude Code，跟它说，我有一个 md2wechat 的排版工具，还有一个 ChatGPT 生图的工具，帮我写一个脚本，从 Markdown 到公众号草稿箱，一条命令搞定。

Claude Code 看了看我现有的工具链，开始帮我搭。

先是生图，我在文章里用一种特殊语法标记需要 AI 生成的图片，比如写 `![插图](__generate:一个程序员在使用AI编程__)`，脚本自动解析这些标记，调用 ChatGPT 生成图片，然后把标记替换成本地路径。

然后是检查，脚本会先跑一遍 inspect，确认文章标题、作者、摘要都填了，图片都在，没有遗漏。

接着是上传，把文章里的所有图片上传到微信，拿到微信的 URL，替换掉本地路径。

最后是推草稿，把排版好的 HTML、封面、标题等信息组装成微信要求的格式，推到草稿箱。

整个过程脚本自动串联，哪一步失败就停下来报错。

说到这，听着挺顺利的对吧。

不是的。

实际搭建的过程中踩了不少坑，这些坑才是这篇文章真正有价值的部分。

第一个坑，手写 JSON 嵌 HTML 必定翻车。

微信的草稿接口要求提交一个 JSON，里面包含 HTML 格式的文章内容。我一开始手写这个 JSON，结果 HTML 里的双引号和 JSON 的双引号打架，解析直接报错，`invalid character 'v' after object key:value pair`。

这种 bug 特别难定位，因为报错信息跟真正的问题完全不相关。你盯着那个 `v` 看了半天，根本想不到是因为引号冲突。

最后的解法其实很简单，用 Python 的 `json.dump()` 来生成 JSON，让它自动处理转义。从此再没出过这个问题。

第二个坑，Chrome 下载路径不固定。

ChatGPT 生成的图片会通过 Chrome 下载到本地，但 Chrome 的下载路径会变，有时候在 `~/Downloads/` 下面，有时候跑到另一个目录去了。

脚本第一次能找到图，第二次就找不到了。

我寻思了一下我没寻思明白为什么 Chrome 要这样。

后来想了个办法，在多个候选目录里搜索最新生成的文件，而不是硬编码一个路径。土办法，但是管用。

就这两个坑，花了我大半天时间。但每解决一个，脚本就稳一分。

说到最终效果，现在我的发布流程变成了这样。一条命令。

`uv run publish.py --article article.md --cover cover.png --mode api`

回车一敲，五件事全自动完成，解析文章里的 AI 生图标记调用 ChatGPT 生成图片，检查文章是否就绪，上传所有图片到微信素材库，按主题排版生成 HTML，推送到公众号草稿箱。

图片

从半小时的手动操作，变成了一分钟的一条命令。

说真的，这个过程里面有几个我觉得还挺重要的经验。

一个是先手动跑通，再自动化。我不是一开始就写脚本的，先手动把每一步都走通了，确认每个工具的输入输出，然后再让 AI 帮我串起来。如果一开始就写脚本，中间任何一步出问题都不知道是工具的 bug 还是脚本的 bug。

一个是把踩过的坑记下来。每一个坑都写进了文档里，让 AI 下次执行的时候能主动避开。我现在回头看，这些「陷阱规避」比正常的使用说明有价值得多。

再一个就是不追求完美。第一版只要能用就行，后面再慢慢优化。比如第一版还是两个脚本，后来才合并成一个。

突然想起了以前做产品的时候学的一个概念，叫 MVP，最小可行产品。先跑通一个能用的版本，别纠结细节。很多项目死掉不是因为做得不够好，是因为还没做完。

这条原则放在用 AI 造工具这件事上也完全适用。

其实这篇文章本身，就是这个工具链的一个验证。Markdown 写稿，AI 生图，自动排版，一键推送，整条链路跑通了。

如果你也在做公众号，或者任何重复性的内容发布工作，试试把流程拆解成步骤，然后让 AI 帮你自动化。不需要会写代码，只要你能把每一步说清楚，AI 就能帮你串起来。

工具不重要，思维方式才重要。
