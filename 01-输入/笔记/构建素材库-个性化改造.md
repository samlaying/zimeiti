## 第一步：确认你的定位关键词

在开始建库之前，先明确你所有素材都围绕的核心——这决定了你该读什么书、上什么课、找什么信息源。

> **你的定位**：一个0代码基础的产品人，教有想法但不会写代码的普通人，从GitHub上找到AI项目并改造为己用。

**核心关键词：**
- GitHub使用
- AI项目落地
- 零基础改造
- 非技术人员的AI实操
- 想法→原型的转化

这些关键词就是你**所有认知素材的筛选器**——跟你定位相关的才收，无关的一律不收。

### 2.2 课程

你需要的课程方向是**补充你自己的知识框架**，不是碎片技巧。

【问题2】以下方向，你目前最缺哪个？选1~2个优先：

| 方向                 | 具体内容                             | 为什么需要                  |
| ------------------ | -------------------------------- | ---------------------- |
| **Git/GitHub基础操作** | clone、fork、pull request、分支管理     | 这是你改造项目的基本工具，如果还不熟需要补  |
| **Python基础（够用就行）** | 不需要精通，但要能看懂基本语法、能改参数             | 改造AI项目你不可避免会碰到Python代码 |
| **AI/LLM应用开发入门**   | API调用、Prompt Engineering、Agent框架 | 你的核心内容领域，需要有体系化认知      |
| **产品设计/需求分析**      | 如何把模糊的想法变成清晰的需求                  | 你教别人的第一步就是"先把想法变成需求"   |

> 你不需要全都学。选你最缺的1~2个，找一门**体系化的课**去上，然后按照课程里的方法做笔记和框架拆解。

## 第三步：搭建业务素材库

### 3.1 个人故事库

你需要梳理的故事，要围绕你的定位来选。以下是最适合你的故事方向：

**【行动】现在就试着回答以下每个问题（不用写得很完整，想到什么写什么）：**

| 故事方向 | 具体问题 | 为什么跟你的定位相关 |
|---------|---------|------------------|
| **你是怎么开始碰GitHub的？** | 第一次打开GitHub是什么时候？当时什么感受？为什么想去用它？ | 这是你的"从零到有"故事，目标用户会共鸣 |
| **你第一次改造项目成功的经历** | 改了什么项目？为了实现什么想法？花了多长时间？ | 核心案例，证明你做的事是有价值的 |
| **你踩过最深的一个坑** | 什么报错让你卡了最久？最后怎么解决的？ | 目标用户马上会遇到的坑，建立"他懂我"的信任 |
| **你用AI辅助改造项目的经历** | 你是怎么让AI帮你改代码的？效果如何？ | 你的差异化方法论——不写代码但用AI改造 |
| **你身边朋友问你最多的问题** | 他们问你什么？你怎么回答的？ | 直接对应你的内容选题 |
| **你为什么0基础却敢去碰GitHub？** | 是什么驱动你的？你的心态是什么？ | 传递你的价值观，目标用户需要这份勇气 |

**写故事的框架（每个故事都用这个结构）：**

```
标题：【一句话概括这个故事】

背景：当时我在什么情况下？
任务：我想做什么/我面临什么问题？
行动：我具体做了什么？
结果：最后怎么样了？
思考：我从中学到了什么？这个道理对你的受众有什么用？
```

**【问题4】你能不能现在就试着写第一个故事？就写"你第一次打开GitHub"的经历。用上面的框架，写个大概就行，我帮你优化。**

### 3.3 案例库

【问题6】你现在有没有以下任何一个经历？

| 案例类型                     | 具体情况                      |
| ------------------------ | ------------------------- |
| 你自己改造过的一个完整项目            | 从"有想法"到"找到项目"到"改造成功"的完整过程 |
| 你帮朋友/同事用过AI工具解决了一个问题     | 哪怕很简单的也行                  |
| 你在实习中用GitHub/AI完成的一个具体任务 | 工作场景的真实案例                 |

如果有，按照这个模板记录：

```
用户/场景：谁？什么背景？
之前的问题：他/你想做什么但做不到？
我用了什么方法：找了什么项目？怎么改造的？用了什么工具？
最终结果：效果怎么样？
```
