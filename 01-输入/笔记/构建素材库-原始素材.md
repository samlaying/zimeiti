# 百万IP个人素材库 — 详细课程笔记

---

## 一、定位作业的重要性

### 1.1 定位作业是整个课程中最难的部分

- 定位作业可能是整个学期四天课程中**最难的一个作业**
- 难度来源：
  - 需要回答非常多的问题
  - 问题背后需要做**大量的市场调研**
  - 需要和**AI智能体反复碰撞**——通过AI精通的定位方法、互动点梳理、用户场景分析，才能得到比较好的答案
  - 一定需要花很长时间

### 1.2 为什么值得花这个时间

- 一个公司要做商业架构，请咨询公司来梳理，至少需要**2~3周时间**，费用**至少3万，多则上百万**
- 你做定位这件事，本质上就是在梳理你未来**年赚10万、50万、100万甚至几百万**的基础
- 如果定位上精力花得不够多、没有认真对待，后续可能会**一步步走偏**——花了很多时间努力，最后发现方向错了
- 把技能和能力放到另一个场景、面向另一群用户，可能得到更大的营收、更快地进入想要的生活

## 二、为什么大多数人无法持续输出内容

### 2.1 大多数人做自媒体的心路历程

```
开始做自媒体
  → 只想一件事：怎么出爆款、怎么涨流量、怎么学流量技巧
  → 抄爆款，学标题、学文案、学标签
  → 可能爆一两篇，也可能一篇没爆
  → 发现坚持太难了
  → 选择放弃
```

### 2.2 内容输出的层级模型

大多数人只看到最表层的东西：

| 层级 | 内容 | 说明 |
|------|------|------|
| **最表层** | 公众号文章、小红书图文、朋友圈 | 这是输出的末端，是最表面的东西 |
| **第二层** | 素材——生活中的持续思考和记录 | 来自读书、上课、用户反馈、实践纠错等 |
| **最底层** | 信息管理能力 | 决定你能不能在生活中有持续思考的能力 |

> **类比**：就像看到成功人士做了成功的事（行为），但看不到他们背后的方法、思维、认知、价值观。

**学表层的流量技巧 = 学碎片化的东西**，无法支撑你长期输出。

### 2.3 抄爆款的根本问题

- 所有人都去抄同一个爆款 → 没有差异化
- 你的核心知识体系无法体现 → 用户看不到你到底厉害在哪
- 只是在单纯搞流量，内容没有形成有机的系统
- **碎片化输出**：今天抄A博主的爆款，明天抄B的，后天抄C的——跟你的定位、你的核心价值主张完全脱节

## 三、信息管理——持续输出的底层能力

### 3.1 信息处理的四个阶段

```
接收 → 存储 → 分析 → 决策（行动）
```

| 阶段 | 说明 | 对应能力 |
|------|------|---------|
| **接收** | 眼睛看到、耳朵听到、身体感受到——一切外界信息的输入 | 注意力管理 |
| **存储** | 大脑记忆/外部工具记录 | 知识管理 |
| **分析** | 判断信息的对错、与已有经验的关系、如何运用 | 思考能力 |
| **决策** | 基于分析做出行动 | 行动力 |

> **公众号/小红书的输出发生在"分析"阶段**——你必须先对信息有分析、有思考，才能输出内容。

### 3.2 为什么你分析不了信息

**分析的产生需要两个前提：**

| 前提 | 说明 |
|------|------|
| **基础知识框架** | 你对这个领域必须有基本认知，否则面对信息完全无法分析（如小学生听商业课 = 零框架 = 什么都分析不出来） |
| **信息与框架产生关系** | 新信息要么**与你的认知冲突**（激发反驳和思考），要么**促进框架完善**（提供新维度） |

**两种有价值的信息关系：**

| 类型 | 说明 | 举例 |
|------|------|------|
| **冲突型** | 新信息与你的现有认知相矛盾 → 激发你去反驳、去深入思考 | 你一直认为标题要加数字，有人说标题要根据人群和内容目的来起 |
| **促进型** | 新信息补充了你的框架，让你从一个点看到一个面 | 你只知道起标题的几个技巧，有人给你一个完整的标题框架 |

> **类比**：信息就像人和人交谈——认知完全一样 = 跟自己对话，聊不出新东西；差距太大 = 没法沟通（如国际新闻对我们来说）。有价值的信息一定在一个"有差距但能理解"的范围内。

### 3.3 框架的建立方式

> **在同一个主题下，持续输入同一类信息，才能持续完善你对这件事的知识体系。**

- 就像上学：语文课→接受语文信息→做作业实践→反馈→建立语文知识框架
- 每个学科都是这样的循环
- 讲师做公众号：从2024年11月开始，持续做→持续思考→持续分析→半年后才有了完整的框架
- 你现在写不了定位报告 = 你对商业的框架还没到60分；到60分就能写了，好坏只是80分和60分的区别

### 3.4 为什么大多数人做不到

**核心问题：信息源混乱无序**

- 今天看情绪管理，明天看写作技巧，后天看成功学
- 完全被外界信息推着走
- 没有一个主题的持续输入

> **类比**：你在拼乐高，但把不同模型的模块放在一起，妄想拼成一个好看的模型——它们本来就不属于同一个东西。

**结果链：**

```
主题混乱 → 信息被外界推着走 → 思路混乱 → 无法构建知识框架 → 无法持续输出
```

> 你的输入是混乱的，你还妄想能够持续输出——不可能。所有的步骤都是连在一起的。

## 四、认知素材库

### 4.1 什么是认知素材库

解决的核心问题：**建立你的知识框架/知识体系**。

认知素材的三个基本来源：

| 来源 | 说明 | 要求 |
|------|------|------|
| **课程** | 付费上的课 | 必须是**体系化、系统化**的课，不是碎片化知识打包的课 |
| **书籍** | 专业的、经典的书籍 | 长期输入的重要来源 |
| **信息源** | 关注的博主、长视频、长文、播客 | 建议找**长内容**（1000字以上的公众号文章、YouTube/B站长视频、长播客），而非短视频 |

### 4.2 三个来源的优先级

```
课程 + 书籍（最重要，构建框架）> 信息源（点状补充）
```

- 真正要构建框架 = **上课 + 读书 + 写读书笔记 + 梳理框架**
- 信息源更多是在框架建立后做**点状补充**
- 短视频给不了你真正的干货——能跑起来的很短的短视频，靠的是搞笑、情感、猎奇等人性底层需求

### 4.3 输入的原则

| 原则 | 说明 |
|------|------|
| **目标导向/问题导向** | 这条信息能解决我哪个问题？能帮我实现哪个目标？ |
| **同一主题持续输入** | 不要东一榔头西一棒槌，围绕你的定位方向持续积累 |
| **质量 > 数量** | 读一万本书但都没有转化为行动，不如认真读透一本书 |

### 4.4 存储的问题——工具太分散

大多数人存储信息的方式：

| 工具 | 场景 |
|------|------|
| 飞书/石墨 | 工作相关 |
| 印象笔记 | 个人笔记 |
| 手机备忘录 | 灵感记录 |
| 实体笔记本 | 会议/讨论 |
| 各种APP | 碎片信息 |

**问题**：信息物理位置分散 → 想分析时找不到 → 找到时已经忘了为什么找 → 索性不找了 → 信息丢失 → 无法构建框架

> **类比**：你要炼金，但把原材料放在不同的地方，怎么炼？炼金厂必须在同一个地方。

**解决方案**：把所有信息集中到**一个工具**中（如Notion）。

### 4.5 分析的两个原则

| 原则 | 说明 |
|------|------|
| **问题导向/目标导向** | 每接收一条信息，问自己：它能解决我的哪个问题？能帮我实现哪个目标？ |
| **解决方案式输出** | 不仅记零碎的灵感和想法，还要对读过的书/上过的课做**系统化的结构拆解**——从头到尾梳理框架，形成完整的解决方案 |

> **处理一万条信息但都停留在接收/存储阶段，不如认真处理一条信息并把它转化为行动。**

### 4.6 决策——从行动中持续完善

```
做出决策 → 行动 → 观察结果 → 找到原因 → 记录思考灵感 → 优化 → 新的决策
```

这就是一个持续循环的过程，复盘是关键。

## 五、业务素材库

### 5.1 什么是业务素材库

在做业务过程中可以存下来的、能长期复用并持续产生价值的东西。

**四大类：**

| 类别 | 说明 |
|------|------|
| **个人故事库** | 你的人生故事，用于建立人设和信任 |
| **用户反馈库** | 用户好评截图等社会证明 |
| **案例库** | 完整的用户成功案例（问题→方法→结果） |
| **个人照片库** | 你的生活照、工作照，用于IP化 |

---

### 5.2 个人故事库

#### 5.2.1 为什么故事这么重要

- 人设不是装出来的——是通过对你自己的了解，把能展现的故事**梳理出来、放大展现**
- 人设造假 = 前面几年积累的信任全部白费
- **故事才是最深入人心的**——人天生喜欢听故事，如果故事还跟自己有关，更会产生共鸣和信任

> **类比**：你直接跟别人讲干货"定位怎么做1234"，别人不会听。但你讲故事带入，别人愿意看下去，最后从故事中听到你思考到但他没思考到的点 → 信任 + 点赞关注收藏。

**故事的作用：**
- 传递你的价值观和性格
- 拉近与用户的距离
- 可以插入到小红书、公众号、朋友圈、私聊、直播、课程等**所有场景**

#### 5.2.2 七大类个人故事

| 类别 | 说明 |
|------|------|
| **1. 家庭关系** | 与父母、孩子的关系故事 |
| **2. 情感与恋爱** | 亲密关系、如何看待情感 |
| **3. 童年故事** | 小时候的事情 |
| **4. 求学经历** | 小学、中学、大学、读研、业余学习 |
| **5. 职场与职业发展** | 工作经历、副业、创业（失败也可以讲） |
| **6. 爱好** | 运动、读书、打球、极限运动等 |
| **7. 人生意外事件** | 上法院、公司破产、受伤等 |

#### 5.2.3 故事的写法框架

每个故事都遵循：

| 要素 | 说明 |
|------|------|
| **背景** | 当时的环境是什么？ |
| **任务** | 当时需要完成什么目标？ |
| **行动** | 你具体怎么做的？ |
| **结果** | 最后拿到了什么结果？ |
| **总结/思考** | 你从这件事中看到了什么道理？（最关键） |

> **不是记流水账**，不是记日记。最后一定要给出**思考结论**，而且这个结论最好能和你的知识框架、你的定位方向契合，吸引你的目标用户。

#### 5.2.4 关于"失败"的故事

- 创业失败、做错事情——**恰恰是最真实的，最能获得共鸣**
- 大部分人去做困难的事情都是失败的，你的失败故事反而能让更多人代入
- 时代变了：过去炫富有流量，现在大家看烦了。真实的、接地气的故事更有吸引力

#### 5.2.5 梳理建议

- 每天坚持梳理2个故事，每个花15分钟
- 一个月下来 = 60个故事，对于大多数自媒体人来说已经非常厉害了
- 这是一个**长期持续做的事**，讲师自己也在持续做

### 5.3 用户反馈库 & 案例库

| 维度 | 用户反馈库 | 案例库 |
|------|----------|--------|
| **内容** | 零散的好评截图 | 完整的用户转变过程 |
| **粒度** | 简单截图+简单记录 | 完整记录：用户之前的问题→我用了什么方法→用户拿到了什么结果 |
| **作用** | 日常发朋友圈、内容中插入，营造"很多人说我好"的氛围 | 深度说服——让用户看到和自己背景相似的人获得了效果 |
| **复用** | 同一条好评隔一两个月可以再发，没人记得住 | 每个案例可以在多个场景反复使用 |

**案例库的写法框架（用户案例模板）：**

| 字段 | 说明 |
|------|------|
| 用户是谁 | 基本背景 |
| 之前的问题 | 用户原来的痛点和状态 |
| 我用了什么方法 | 你的方法论/产品/服务 |
| 用户的结果 | 转变后的状态 |

> **案例的力量**：当用户在公众号、朋友圈、小红书、私聊、直播中反复听到跟你背景相似的人都成功了，他一定会心动。

### 5.4 个人照片库

#### 5.4.1 为什么一定要露脸

- **IP化是趋势**——有真人面孔，账号才"活"了
- 机构号没有IP感，很难积累信任
- 不是只有好看的人才能露脸——**发了就能多挣钱，不发就少挣钱**
- 有争议的内容露脸 → 可能被攻击，但也有人更喜欢你、更愿意付费

#### 5.4.2 照片怎么拍

| 要求 | 说明 |
|------|------|
| **自然** | 不需要艺术照/精修，生活化的正常照片就好 |
| **丰富** | 出门拍、工作拍、吃饭拍、旅行拍——展现你是什么样的人 |
| **持续** | 持续积累，存在一个页面上方便随时取用 |

#### 5.4.3 照片的使用场景

| 场景 | 用途 |
|------|------|
| 朋友圈配图 | 有配图 vs 没配图 → 营收有明显增长 |
| 公众号封面/头图 | 增加辨识度 |
| 小红书图文 | 增加真实感 |
| 短视频素材 | 口播中穿插照片，增加视觉丰富度 |

> **用户对你是有画像构建的**——他通过你的照片知道你喝什么咖啡、在哪个城市、什么工作状态，潜意识中在评估"你值不值得信任"。

## 六、Notion素材库实操演示

### 6.1 整体结构

```
百万IP个人素材库
├── 认知素材库
│   ├── 信息源
│   ├── 课程
│   └── 书籍/阅读
└── 业务素材库
    ├── 家庭关系故事
    ├── 情感与恋爱故事
    ├── 童年故事
    ├── 求学故事
    ├── 职场与职业发展故事
    ├── 爱好故事
    ├── 人生意外故事
    ├── 个人照片
    ├── 用户案例
    └── 用户好评
```

### 6.2 信息源模块

| 字段 | 说明 |
|------|------|
| 标题 | 信息源的名称/主题 |
| 链接 | 文章/视频/博客的链接 |
| 创建时间 | 自动生成，不可更改 |
| 核心观点 | 这条信息的核心观点是什么？ |
| 行文框架 | 这篇文章/视频的结构是什么？ |
| 帮助解决哪些问题 | 能解决我的哪个问题？能帮我实现哪个目标？ |
| 归档状态 | 未归档 / 已归档 |

**归档逻辑：**

| 状态 | 含义 |
|------|------|
| **未归档** | 这条信息还在分析中，还没有完全榨取价值，后续还需要继续发表内容 |
| **已归档** | 这条信息的价值已经全部提取完毕——核心观点已梳理、已转化为行动、已沉淀到任务表中 |

**操作：**
- 点击卡片中的"归档"分类标签即可切换状态
- 误触归档 → 去归档表中找到该条目，点击标签改回未归档
- 删除 → 右键删除

### 6.3 课程模块

| 字段 | 说明 |
|------|------|
| 课程名称 | 上了什么课 |
| 价格 | 多少钱 |
| 内容主题标签 | 讲什么方向（如"内容创作""文案""商业定位"等） |
| 评价 | 上完课后的评价（上完再写） |
| 你想解决的问题 | 上这门课前你希望解决什么问题？ |
| 上课中的疑问 | 上课过程中产生的疑问 |
| 优化行动 | 基于课程内容，你接下来要做什么？ |

### 6.4 书籍/阅读模块

| 字段 | 说明 |
|------|------|
| 书名 | 读的什么书 |
| 作者 | 谁写的 |
| 内容标签 | 讲什么方向 |
| 读书笔记 | 按模板记录 |

**读书笔记模板：**

| 步骤 | 操作 |
|------|------|
| **第一遍阅读** | 边看边记：这条信息能帮我解决什么问题？我的想法是什么？ |
| **读完后** | 结合零碎笔记 + 快速重看目录 → 做一次**系统化总结**（解决方案式输出） |
| **效果** | 框架会长在脑子里，而不是碎片化的知识点 |

### 6.5 个人故事模块

每个故事类别都有统一模板：

| 字段 | 说明 |
|------|------|
| 标题 | 故事标题 |
| 总结 | 简短的故事总结（方便分类查找） |
| 背景 | 当时的环境和情况 |
| 任务 | 当时要完成什么目标？ |
| 行动 | 你具体怎么做的？ |
| 结果 | 最终拿到了什么结果？ |
| 思考/结论 | 你从中得出了什么道理？ |

### 6.6 用户案例模块

每个案例页面点击"用户案例"模板，自动生成结构：

| 字段 | 说明 |
|------|------|
| 用户是谁 | 用户的名字/背景 |
| 用户的问题 | 之前的痛点和状态 |
| 我的方法 | 我用了什么方法帮助他？ |
| 用户的结果 | 转变后的效果 |

### 6.7 用户好评模块

## 七、关于"信息源选择"的核心提醒

### 7.1 不要碎片化输入

| 碎片化输入（不推荐） | 系统化输入（推荐） |
|-------------------|------------------|
| 刷短视频 | 看长视频/YouTube |
| 看短文章 | 读1000字以上的深度文章 |
| 东看一点西看一点 | 围绕同一主题持续输入 |
| 被算法推着走 | 带着目标和问题主动选择 |

### 7.2 信息筛选的核心标准

> **这条信息到底能解决我的哪个问题？到底能帮我实现哪个目标？**

- 如果答案是"都不能" → 不值得接收和存储
- 处理一万条无用信息 < 认真处理一条有用信息并转化为行动

### 7.3 关于"认知资产"

- 账号、粉丝数据、流量——**这些都是表面的，不能跟你一辈子**
- 真正跟一辈子的 = **你脑子里面的认知和知识框架** = 你的认知资产
- 即使账号被封，只要有认知资产，随时可以重新做起来
