# 第一篇：拓展版实操路线图

## 你的定位再确认

> **一个0代码基础的产品人，怎么从GitHub上找到合适的AI项目，改造成自己想要的样子。**

**目标受众画像：**

| 人群 | 真实困境 | 他们心里的声音 |
|------|---------|---------------|
| 想转AI产品的传统PM | 简历上没有AI相关项目，面试聊不出东西 | "我知道AI是趋势，但我怎么证明我会？" |
| 产品实习生 | 公司不给你碰核心项目，自己又不知道做什么练手 | "实习3个月，简历写不出亮点" |
| 有想法的创业者/自由职业者 | 想做个AI工具试水，但招不起开发 | "我不需要融资，我只需要一个能跑的Demo" |
| 设计师/运营转产品 | 有审美有感觉，但技术是黑箱 | "我知道产品应该长什么样，但我做不出来" |

**核心差异化：** 大号技术博主不屑于教"笨方法"，纯小白博主又没实操经验——你恰好卡在中间，这是蓝海。

**为什么这个定位能持续？**
- GitHub上的AI项目只会越来越多，选型需求是刚需
- "0基础"的标签让你永远有新的受众进来（人会长大，但永远有新人）
- 你教的不是"技术"，是"翻译能力"——把技术语言翻译成产品语言，这件事AI还做不好


## 实操路线图（分4个阶段，约3个月跑通）


### 第一阶段：搭建内容地基（第1-2周）

**目标：** 让人3秒内知道你是谁、能解决什么问题。

#### 1.1 账号基础三件套

| 要素       | 具体内容                | 示例                                                | 常见错误                             |
| -------- | ------------------- | ------------------------------------------------- | -------------------------------- |
| **昵称**   | 直接体现"0基础+GitHub+改造" | "GitHub改造家XX"、"0代码AI产品人XX"                        | ❌ 起文艺名"XX的产品笔记"——没有辨识度           |
| **简介**   | 一句话说清价值             | "0代码基础，但我能把GitHub上的AI项目改成我想要的样子。教你不用写代码，也能把想法落地。" | ❌ 写一堆标签"AI/产品/创业/读书"——什么都等于什么都没有 |
| **置顶内容** | 你的"代表作"或"定位宣言"      | 一篇《我是谁、我能帮你做什么》+ 一个你已经改造成功的项目展示                   | ❌ 置顶一篇流水账日记——浪费最好的展示位            |

**置顶内容的写法建议：**

```
标题：《我不会写代码，但我用30分钟从GitHub上"借"了一个AI项目，改成了我的XX工具》

结构：
1. 我是谁？（一句话背景）
2. 我遇到了什么问题？（具体场景）
3. 我怎么在GitHub上找到的？（搜索过程截图）
4. 我怎么改的？（问AI + 改配置）
5. 最终效果（录屏/截图）
6. "如果你也想这样做，关注我，我会持续更新"
```

#### 1.2 内容风格定调

参考哈伦说的"稳定的气味"：
- **视觉统一**：封面风格一致（建议用深色背景+亮色文字，偏技术感但不冰冷）
- **语言风格**：口语化、带节奏感、不装，像朋友聊天
- **固定开场白**：比如"我又来改造GitHub项目了，今天这个..."
- **固定的视觉锚点**：比如每篇封面都有一个"改造前后"的对比图，形成辨识度

**封面设计的具体建议：**

| 元素   | 规范                             |
| ---- | ------------------------------ |
| 背景色  | 深灰/深蓝（#1a1a2e 或 #16213e），显得技术感 |
| 主文字  | 白色或亮黄（#e2b714），字号大，3行以内        |
| 辅助元素 | GitHub的Logo小图标 + 代码片段截图（做装饰用）  |
| 统一模板 | 用Canva做一个固定模板，每次只改文字           |


### 第二阶段：内容生产SOP（第3-6周）

**目标：** 建立可持续的内容流水线，每周产出2-3篇。

#### 2.1 选题库（按优先级排序）

| 选题类型 | 示例标题 | 为什么有效 | 预期数据表现 |
|---------|---------|-----------|------------|
| **痛点共鸣型** | 《想转AI产品，但简历上没项目？我用20分钟从GitHub搞了一个》 | 直击目标受众最大焦虑 | 高点击率、高转发 |
| **过程记录型** | 《0基础改造GitHub项目第X天：今天踩了5个坑》 | 真实、有代入感、天然带"追更"属性 | 中等流量、高互动 |
| **成果展示型** | 《我不会写代码，但我用AI把这个GitHub项目改成了自动周报工具》 | 结果导向，激发"我也能做到" | 高收藏、高私信 |
| **工具教程型** | 《GitHub上全是英文看不懂？我教你3步快速判断一个项目值不值得用》 | 降低门槛，建立信任 | 高收藏、长尾流量 |
| **避坑指南型** | 《改造GitHub项目最容易踩的7个坑，第3个我卡了3天》 | 实用价值高，收藏率高 | 高收藏、适合SEO |
| **对比测评型** | 《GitHub上3个AI写作项目，我全跑了一遍，这个最适合产品人》 | 帮用户做选择，省时间 | 高收藏、高信任 |
| **热点结合型** | 《XX刚发布的AI工具，GitHub上已经有人做了开源版，我来改改看》 | 蹭热点+差异化角度 | 高曝光、短期爆发 |

#### 2.2 内容结构模板（3种，直接套用）

**模板A：过程记录型（最推荐，因为你的真实过程就是最好的内容）**

```
【开头钩子】（3句话内）
"昨天有个朋友问我：'我想转AI产品，但简历上连一个AI项目都没有，怎么办？'
我说：'你去GitHub上找一个，改改就能用。'
他回：'我连README都看不懂...'
好，今天我就带你走一遍，0基础怎么搞。"

【正文】
1. 我的需求是什么？（越具体越好，比如"我想做一个能自动总结会议纪要的工具"）
2. 我在GitHub上怎么搜的？（展示搜索关键词和筛选过程，截图）
3. 怎么判断这个项目能不能用？（3个判断标准——下面会详细讲）
4. 改造过程中遇到了什么问题？（真实报错截图，越丑越好）
5. 我是怎么解决的？（用AI辅助的具体步骤，展示你和AI的对话）
6. 最终效果展示（录屏或截图，对比"改造前"和"改造后"）

【结尾】
"你看，我不需要会写代码，我只需要知道我要什么、怎么找、怎么改。
这就是AI产品人最核心的能力——不是写代码，是把技术变成产品。
关注我，下一期我改造的是XX项目。"
```

**模板B：避坑指南型（适合小红书图文）**

```
标题：改造GitHub项目最容易踩的7个坑（新手必看）

格式：
封面：大字"7个坑"+ 暗背景

每张图：
- 坑的名称（大字）
- 一句话描述（小字）
- 我的真实截图（报错截图/翻车截图）
- 怎么解决的（一句话）

最后一张：
- 总结图
- "关注我，持续分享0基础改造GitHub项目的实战经验"
```

**模板C：对比测评型（适合公众号长文）**

```
标题：GitHub上5个AI[XX]项目，我全跑了一遍，只推荐这2个

结构：
1. 我的需求是什么（统一标准）
2. 搜索过程（我怎么找到这5个的）
3. 逐个测评：
   - 项目简介（一句话）
   - 跑起来有多难？（⭐评级）
   - 改造空间有多大？（⭐评级）
   - 最终效果截图
4. 推荐结论 + 推荐理由
5. "想看我改造其中哪个？评论区告诉我"
```

#### 2.3 判断GitHub项目的"3个标准"（你的核心方法论）

这是你反复要教的东西，做成一套固定框架：

| 标准 | 怎么判断 | 具体操作 |
|------|---------|---------|
| **① README能不能看懂大意** | 不需要看懂代码，只需要看懂README的"是什么"和"能做什么" | 用浏览器翻译功能，看前3段 |
| **② 有没有Demo/截图** | 有截图/Demo的项目，说明作者自己用过，大概率能跑 | 看README里有没有GIF、截图、在线Demo链接 |
| **③ 最近更新时间** | 超过6个月没更新的项目慎选，依赖版本可能过时 | 看GitHub页面右边的"Last commit" |

**补充：怎么用AI辅助判断**
```
把README丢给AI，问这3个问题：
1. "这个项目是做什么的？用一句话说清楚"
2. "如果我不会写代码，能跑起来吗？需要什么前置条件？"
3. "我想用它做[你的需求]，可行吗？需要改什么？"
```

#### 2.4 AI协作工作流

参考哈伦的"一窗一事"原则：

| 窗口 | 工具 | 任务 | 注意事项 |
|------|------|------|---------|
| **选题窗口** | Claude/ChatGPT | 输入你的项目改造过程，让AI帮你提炼3个标题选项 | AI出100个，你只取1-3个，扔掉97% |
| **脚本窗口** | Claude | 给定受众画像+情绪目标，生成脚本框架 | 开头和结尾必须自己写，确保"人味" |
| **润色窗口** | 人工 | 调整口语感、加真实细节 | "报了8个错"比"遇到了一些问题"有力10倍 |
| **封面/配图** | Canva+截图 | 统一模板，真实操作截图为主 | 不要用AI生成的假截图，一眼假 |
| **录屏** | OBS/手机录屏 | 操作过程实时录 | 不用剪太精，真实感比精致感重要 |

**关键原则：**
- 所有"报错截图""操作录屏"必须真实，这是你的核心竞争力
- 你和AI的对话过程本身就是内容——展示你问AI什么问题、AI怎么回答的
- 每篇内容至少有1个"可带走"的东西：搜索关键词、判断清单、配置模板


### 第三阶段：平台分发与数据验证（第7-10周）

**目标：** 测试哪个平台反馈最好，集中火力。

#### 3.1 平台策略（详细版）

| 平台 | 内容形式 | 核心策略 | 发布时间建议 | 标题技巧 |
|------|---------|---------|------------|---------|
| **公众号** | 长文（1500-3000字） | 首发阵地，建立深度信任；尽早发低价付费文章测试变现 | 周二/周四晚8-10点 | 直接、不标题党、信息量大 |
| **小红书** | 图文（9图+300字） | 重"活人感"和"身份幻想"；封面要惊艳 | 晚7-9点、午12-1点 | 带"0基础""GitHub""改造"，用数字 |
| **视频号** | 短视频（1-3分钟） | 把小红书内容同步搬运；利用微信社交链 | 晚8-10点 | 前3秒必须有钩子 |
| **即刻** | 短动态（100-300字） | 产品经理圈子活跃，适合建人脉 | 随时 | 直接说干货，不要铺垫 |
| **知乎** | 回答/文章 | SEO长尾流量，回答"如何用GitHub"类问题 | 无固定时间 | 回答高关注问题 |

#### 3.2 小红书专项策略（你的主战场）

**为什么小红书优先？**
- 你的核心受众（产品实习生、想转行的人）大量活跃在小红书
- 算法推荐制，不需要粉丝基础也能获得曝光
- "0基础""改造""GitHub"在小红书是稀缺内容

**小红书封面模板：**

```
┌──────────────────────┐
│   深蓝/深灰背景        │
│                      │
│   0代码基础           │  ← 白色大字
│   我把GitHub上的      │  ← 白色大字
│   AI项目改造成了       │  ← 白色大字
│   自动周报工具         │  ← 亮黄色大字
│                      │
│   [改造前截图] → [改造后截图]  ← 对比图
│                      │
│   改造家XX · 第23期    │  ← 小字，建立系列感
└──────────────────────┘
```

#### 3.3 冷启动顺序（按哈伦经验，展开版）

```
第1周：
  └─ 先发3篇小红书（测试哪种选题反馈好）
  └─ 同步发1篇公众号长文（作为深度内容沉淀）

第2周：
  └─ 看小红书数据：哪篇点击率最高？收藏最多？
  └─ 把反馈好的选题方向，再出2篇变体

第3周：
  └─ 公众号发第一篇付费文章（9.9元）
  └─ 标题：《我改造GitHub项目的完整SOP：从搜索到跑起来的全流程》
  └─ 目的不是赚钱，是验证"用户到底为什么买单"

第4周：
  └─ 复盘：付费文章卖了多少份？用户反馈是什么？
  └─ 根据反馈调整内容方向
  └─ 确定主战场（小红书 or 公众号 or 两者并行）
```

#### 3.4 关键数据指标（详细版）

| 指标 | 说明 | 健康值 | 低了怎么调 | 高了说明什么 |
|------|------|--------|-----------|------------|
| **点击率** | 标题和封面是否吸引人 | >5% | 改标题，加数字、加痛点、加"0基础" | 选题方向对了 |
| **完读率/观看时长** | 内容是否留住人 | >40% | 开头钩子不够狠，正文太啰嗦，删掉前3段试试 | 内容质量高 |
| **收藏率** | 内容是否有实用价值 | >3% | 增加"可带走"的干货（清单、步骤、链接） | 有长尾价值 |
| **转发率** | 是否戳中痛点 | >1% | 标题没击中焦虑点 | 选题有传播力 |
| **私信/评论** | 用户真实需求 | 有就是好事 | 没有→在结尾加提问引导 | 有变现潜力 |
| **涨粉率** | 单篇带来的粉丝 | 视基数 | 内容没有"关注的理由"→在结尾强化"关注我你能获得什么" | 人设立住了 |


### 第四阶段：产品化与变现（第11周起）

**目标：** 把流量变成收入，建立商业模式。

#### 4.1 你的变现路径（按优先级，展开版）

**第一优先：卖判断（知识付费）**

你积累的不是"技术"，是"判断力"——什么项目值得用、怎么改、踩过什么坑。这些判断力是最值钱的。

| 产品 | 内容 | 定价 | 上架平台 |
|------|------|------|---------|
| 《GitHub AI项目选型手册》 | 你测评过的50个项目，按场景分类推荐 | 29-49元 | 公众号付费文章/小报童 |
| 《0基础改造避坑指南》 | 你踩过的所有坑+解决方案 | 19-39元 | 小报童 |
| 《AI产品转型路线图》 | 从传统PM转AIPM的完整路径 | 49-99元 | 公众号付费文章 |

**第二优先：卖陪跑（社群/训练营）**

| 产品 | 内容 | 定价 | 形态 |
|------|------|------|------|
| "30天GitHub项目改造陪跑" | 每天1个任务+答疑 | 199-399元 | 微信群+直播 |
| 产品人AI工具箱社群 | 持续更新的项目推荐+改造教程 | 99-199元/年 | 知识星球 |

**第三优先：卖成果（服务/代做）**

| 产品 | 内容 | 定价 | 前提 |
|------|------|------|------|
| 帮你改造一个GitHub项目 | 1v1需求沟通+改造交付 | 按项目500-2000元 | 有3个以上成功案例 |
| AI产品原型代做 | 帮你快速做出可演示的Demo | 按项目1000-5000元 | 有案例背书 |

**不建议做的事：**
- 接广告（粉丝量没到，且低价广告伤信任）
- 卖别人的课（没有一手货源，利润低且伤信任）
- 开实体店铺（跟定位完全不符）
- 做免费社群（精力消耗大，没有筛选机制）

#### 4.2 付费产品包装（参考哈伦"命名问题"法）

**命名公式：** 你的现状（痛点）+ 时间/成本 + 具体成果 + 消除顾虑

| ❌ 错误 | ✅ 正确 |
|---------|---------|
| 《GitHub项目改造教程》 | 《0基础产品人30天GitHub项目改造实战：从"连README都看不懂"到"简历上多了一个AI项目"》 |
| 《AI产品经理课程》 | 《转AI产品的最后一公里：用GitHub项目补齐你的简历空白》 |
| 《产品人社群》 | 《GitHub改造营：每周带你改造1个AI项目，3个月攒出一份AI产品作品集》 |

#### 4.3 推广节奏（详细版）

| 频率 | 动作 | 具体怎么做 |
|------|------|-----------|
| 每天 | 日更免费内容（保持活跃） | A线为主，过程记录型最容易日更 |
| 每2-3天 | 发一篇"推广型"内容 | 免费文章结尾埋钩子："完整的改造SOP我整理在了付费小册里" |
| 每周 | 做一次"成果展示" | 学员反馈截图/自己新改造的项目效果 |
| 每月 | 做一次"付费直发" | 当天只发一篇付费文章，配朋友圈预热 |
| 每季度 | 做一次"大促" | 训练营/社群开放招募，限时优惠 |

#### 4.4 变现的心理准备

| 你会遇到的情况 | 怎么应对 |
|---------------|---------|
| 付费文章只卖了3份 | 3份就是3个真实用户，问他们为什么买、想要什么 |
| 有人说"这内容免费也有" | 不用回复，你的价值不是信息本身，是"筛选+整理+实操验证" |

# 第二篇：深化版定位与内容矩阵方案

## 一、先想清楚一个根本问题：你到底在卖什么？

很多人做内容，上来就问"我发什么""我在哪发"。但在那之前，你需要想清楚一个更本质的问题——**你的内容到底在解决什么人的什么问题？你的不可替代性在哪里？**

### 你不是在教技术，你是在卖"翻译"

技术博主教的是：这个代码怎么写。
你教的是：**这段代码意味着什么、能做什么、我怎么让它为我所用。**

这就是"翻译"——把技术语言翻译成产品语言。

| 他们在卖的 | 你在卖的 |
|-----------|---------|
| 怎么写Python | 怎么判断一个Python项目值不值得用 |
| 怎么部署模型 | 怎么判断你的需求需要什么样的模型 |
| 怎么改代码 | 怎么让AI帮你改代码，你只需要知道改哪里 |
| 技术能力 | **判断力** |

**判断力比技术能力值钱。** 因为技术能力可以被AI替代，但"知道该让AI做什么"的判断力，目前还替代不了。

### 你的三层价值模型

```
表层价值：我能帮你跑起来一个GitHub项目
    ↓
中层价值：我能教你怎么选、怎么改、怎么判断
    ↓
深层价值：我能帮你建立"把技术变成产品"的思维方式
```

表层吸引流量，中层建立信任，深层支撑高价产品。三层缺一不可。


## 二、"三位一体"内容矩阵——三条线并行，互相放大

| 线 | 定位 | 解决什么问题 | 内容形式 | 频率 |
|---|---|---|---|---|
| **A线：GitHub改造实战** | 0代码产品人改造AI项目 | "我想做但做不到" | 教程/过程记录 | 每周2-3篇 |
| **B线：产品拆解** | 提升产品内力 | "我看不懂产品逻辑" | 深度分析 | 每周1篇 |
| **C线：AI产品转行** | 面向转AI产品/实习生 | "我想入行但不知道怎么准备" | 经验/路径 | 每周1篇 |

**三条线的关系（为什么必须三条并行）：**

```
A线是"钩子"（最吸睛，差异化最强）
    → 用户因为A线关注你

B线是"内功"（建立专业信任）
    → 用户因为B线信任你

C线是"变现"（课程/咨询/社群）
    → 用户因为C线付费给你

缺A线：没人关注你（内容太学术）
缺B线：没人信任你（只有技巧没有深度）
缺C线：关注了也不付费（没有变现路径）
```

**三条线如何互相引流：**
- A线文章结尾："想知道我为什么能这么改？因为我拆过100个产品（→关注看B线）"
- B线文章结尾："看完拆解想自己动手？下周我带你改一个（→关注看A线）"
- C线文章结尾："想系统学？我整理了一份实战手册（→付费产品）"


## 三、A线深度展开：GitHub改造实战

### 3.1 内容公式

### 3.2 你反复要教的"核心方法论"（做成固定框架，每篇都引用）

**GitHub项目判断三步法：**

```
第一步：看README前3段 → 能不能用一句话说清它是做什么的？
  → 不能 → 跳过
  → 能 → 进入第二步

第二步：看右边栏的"Last commit" → 最近6个月内有更新吗？
  → 没有 → 谨慎，可能依赖过时
  → 有 → 进入第三步

第三步：用AI辅助 → 把README丢给AI，问"如果我想用它做XX，可行吗？需要改什么？"
  → AI说"不可行" → 跳过
  → AI说"可行，需要改XX" → 你有了改造路线图
```

**这个方法论要在每篇A线内容里反复出现。** 一开始读者觉得"又来了"，看3次以后就记住了——这就是你的品牌认知。

## 四、B线深度展开：产品拆解

### 4.1 拆什么？——你的拆解对象选择策略

**不要拆大众都在拆的（微信、抖音、淘宝）。** 为什么？因为你的受众不关心——他们关心的是"我怎么转AI产品"，不是"微信为什么成功"。

**你应该拆的：**

| 对象 | 拆什么角度 | 为什么你的受众关心 |
|------|-----------|-----------------|
| GitHub Star数高的AI项目 | 产品逻辑、用户路径、商业化可能 | "我想改造的项目到底好不好？" |
| AI原生应用（ChatGPT/Claude的插件生态等） | 交互设计、Prompt工程、付费转化 | "AI产品的交互应该怎么做？" |
| 开源项目的"产品化缺陷" | 从代码到产品，缺了什么？ | "我改造项目时应该注意什么？" |
| 新发布的AI产品 | 用了什么开源技术？产品化做得怎样？ | "这个新产品我能复刻吗？" |

### 4.2 拆解框架（固定结构，降低生产门槛，也让读者形成阅读习惯）

```
1. 一句话定义：这个产品/项目是什么？
2. 目标用户画像：谁会用这个？（具体到职业、场景、痛点）
3. 核心场景：用户在什么情况下打开它？
4. 用户路径拆解：从进入到完成任务，每一步截图+标注
5. 产品亮点（值得学的3个点）
6. 产品缺陷（能改的3个点）
7. 如果是我，我会怎么改？（→链接到A线：我真的去改了）
```

**第7点是关键——它是B线和A线的桥梁。** 每次拆解完都说"如果是我我会怎么改"，然后在A线里真的去改。这样两条线就形成了闭环。

### 4.3 B线的独特价值：建立"产品判断力"标签

A线教的是"怎么做"，B线教的是"怎么想"。做久了，你的标签就从"那个教改GitHub项目的人"升级成"那个对AI产品有判断力的人"。

**这就是从"教程博主"升级到"产品分析博主"的路径。** 教程有天花板（方法会过时），但判断力不会过时。

### 4.4 B线的内容升级路径

| 阶段 | 拆解深度 | 内容形式 |
|------|---------|---------|
| 初期（第1-2月） | 单个产品拆解 | 图文/短文 |
| 中期（第3-4月） | 横向对比（同类产品3-5个） | 长文/系列 |
| 后期（第5月+） | 趋势分析（"AI产品正在往XX方向走"） | 深度文章/行业洞察 |

## 七、关键执行原则（深度版）

### 原则1："过程即内容"

你不需要"准备内容"。你日常改项目的过程，拍下来、截下来、写下来，就是最好的内容。

**为什么？** 因为"准备"出来的内容会有"表演感"，而记录出来的内容有"真实感"。你的受众是小白，他们最需要的就是"看到一个和我一样的人，是怎么一步步做到的"。

### 原则2："越笨越值钱"

不要装大神。展示你的笨拙：
- "我看不懂这个README，问了AI 5遍才懂"
- "我报了8个错，最后一个花了2小时"
- "这个项目的作者回复了我，但英文我也没看懂，又去翻译了一遍"

**为什么笨拙是资产？** 因为大神教人没有代入感——他们忘了"不会"是什么感觉。你正好记得。你的笨拙就是你的受众的现状，他们看到你会想"他和我一样，他能做到我也能"。

### 原则3："A线带B线，B线托C线"

三条线不是独立的，是互相引流的漏斗：
```
A线（广撒网）→ 吸引"想动手"的人
  ↓ 部分人想深入
B线（建信任）→ 吸引"想理解"的人
  ↓ 部分人想改变
C线（变现）→ 吸引"想转行"的人 → 付费
```

### 原则4："先小红书，后公众号"

- **小红书是流量入口**（算法推荐制，冷启动快，0粉也能爆）
- **公众号是信任沉淀**（深度内容，适合付费转化，内容有长尾价值）
- **即刻是圈子建设**（产品经理聚集地，适合建人脉和口碑）
- **视频号是附带的**（把小红书内容同步搬，不用额外投入）

### 原则5："听数据的话，但不被数据绑架"

- 点击率高但收藏低 → 标题党了，内容没干货
- 收藏高但涨粉低 → 内容好但人设不清晰，别人不知道你是谁
- 涨粉高但私信少 → 人设立住了，但没有触发行动的内容
- 私信多但付费少 → 需求是真的，但你的付费产品没有精准匹配

### 原则6："免费给框架，付费给细节"

免费内容告诉用户"应该怎么做"（框架、方向、思路），付费内容告诉用户"具体怎么做"（步骤、模板、工具、答疑）。

**这不是藏私，这是筛选。** 愿意为细节付费的人，才是真正会去执行的人。免费领走的人90%不会做，你把细节给了他们也是浪费。

### 原则7："有商业想法，半小时内必须行动"

这是哈伦的核心原则。你今天看完这两篇文章，如果3小时内没有发出第一条内容，那你大概率也不会发了。

**不是"准备好了再开始"，是"开始了才会准备好"。**
