> [偷星九月333](https://space.bilibili.com/349950942) | [BV1idcbzhEX6](https://www.bilibili.com/video/BV1idcbzhEX6) | 时长 12:40

## Agent架构演进概览
- **传统单Agent范式**：Agent兴起初期的最简单模式，核心为“大模型+工具调用”。
- **双Agent协同模式**：包含一个**Planner（规划者）**和一个**Executor（执行者）**。Planner负责制定计划，Executor负责调用工具执行计划。
- **多Agent架构**：主要有两种模式：
  1. **Agent as Tools（智能体即工具）**：将一个Agent视为一个工具，不同子Agent之间**无交互**。一个主Agent根据任务选择调用不同的子Agent（如数据分析Agent、网络搜索Agent），每个子Agent配有专用工具。
  2. **有向图模式**：不同Agent之间**相互协同**。常使用LangGraph框架搭建，例如流程为：制定计划 → 执行 → 核查结果 → 更新计划 → 再执行，循环直至完成任务，最后生成报告。

## KIMI K2.5的Agent Swarm架构
### 核心目的
- 解决**串行执行效率低**的问题。通过并行执行显著降低任务完成时间。
- **示例**：搜索三个领域的论文。串行需调用3次Web Search工具，而Agent Swarm只需创建一个Web Search Agent，分发三个并行任务。

### 架构与组成
- **协调者（Coordinator）**：
  - 负责**创建子Agent**及**向子Agent分发任务**。
  - **特有工具**：创建子Agent工具、任务分发工具，以及搜索、浏览器等基础工具。
- **子Agent（Sub-agent）**：
  - **角色（系统提示词）由协调者动态指定**。
  - 仅配备搜索、浏览器等**基础工具**，没有创建和分发工具。

### 工作流程与智能决策
1. **任务路由**：
   - 对于**复杂任务**：协调者会**创建子Agent并分发任务**进行并行执行。
   - 对于**简单任务**：协调者会**自行调用基础工具**（如搜索）直接解决。
2. **工具参数**：
   - **创建子Agent工具**：参数为`系统提示词`，用于定义子Agent的角色和能力。
   - **任务分发工具**：参数为`agent`（指定目标子Agent）和`提示词`（具体任务描述）。

### 与传统多Agent的区别
- **子Agent生成方式**：传统多Agent架构中，子Agent通常由人类通过编写不同的系统提示词**预先静态创建**。
- **Agent Swarm中**：子Agent由协调者模型根据具体任务需求**动态自主创建**，原生具备任务拆解和调度能力，效果优于单纯使用提示词约束。

## 训练Agent Swarm的方法
### 核心方法：Parallel Agent强化学习
- **目标**：优化协调者的**动态调度能力**。
- **奖励函数（由三部分组成）**：
  1. **并发调度奖励**：鼓励模型创建并调度多个子Agent（并行执行）。
  2. **反奖励黑客惩罚**：防止为获取更多奖励而创建无用子Agent。奖励基于**所创建子任务是否成功完成**。
  3. **任务完成奖励**：优化的**主要目标**，即成功完成用户给定的具体任务。
- **训练策略**：
  - **奖励权重动态调整**：训练后期，动态增加任务完成奖励的权重。
  - **子Agent参数冻结**：只优化协调者。这解决了强化学习中的**信用分配难题**和**训练不稳定问题**。
  - **间接优化原理**：优化协调者的任务分解和描述能力，间接提升其创建的子Agent的执行成功率。

## 总结
Agent Swarm是一种面向复杂任务的高效多智能体架构，其核心是通过一个拥有动态创建与任务分发能力的协调者，来实现子任务的并行执行，从而大幅提高执行效率。它与传统多Agent架构的关键区别在于子Agent的生成是模型动态完成的。通过专门的Parallel Agent强化学习方法（采用复合奖励函数并冻结子Agent参数）来训练协调者，最终使其能够智能地根据任务复杂度自主决策是否分发任务以及如何最优地创建和调度子Agent。
