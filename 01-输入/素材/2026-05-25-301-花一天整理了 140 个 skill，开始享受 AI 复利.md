# 花一天整理了 140 个 skill，开始享受 AI 复利

> [空格的键盘](https://space.bilibili.com/14762998) | [BV1SfXxBpE7t](https://www.bilibili.com/video/BV1SfXxBpE7t) | 时长 7:21

## AI Skill 整理与管理概述
- **背景**：UP主整理了自己过去几个月使用的 AI skill，共计145个，包括自己创作和收藏的。
- **管理方法**：参考刘比歇夫工作法，记录每天时间花销以优化工作方式。Skill 作为 AI 时间日志，反映使用动机、频率和效果，类似于管理公司员工（如100个员工需定期对齐工作）。
- **愿景**：当拥有大量 AI agent（如1000、1万甚至1亿个）时，可以突破个人能力和时间限制，享受 AI 复利效应。

## Skill 分类与示例
UP主将145个 skill 分为7类，具体内容如下：
- **写作类**（数量最多，适合内容创作者）：
  - 例子：在极客X上发布动态，输入文章后生成模仿个人风格的动态，再编辑发布；播客处理 skill，输入几万字访谈内容，自动总结核心观点。
- **信息类**：
  - 例子：对接 OpenCloud 上的飞书，接收 RSS 和播客总结，每6小时和24小时自动汇报，如同皇帝批奏折；自动推送 RSS 内容到网站，提取播客字幕转写成文章，传到飞书转成卡片并发布。
- **工具类**：
  - 例子：飞书文档推送 skill，将本地文件推送到飞书协同；记录 Agent 记忆、网络浏览等工具。
- **产品设计类**：
  - 例子：用 AI 画原型图，以前需2天，现在20分钟完成，修改也用 AI，输出格式符合企业要求；自动化工具如飞书文档推送。
- **计划类**：
  - 例子：发送“开始我的一天”后，根据上下文发送今日待办事项。
- **数据类**：
  - 例子：缺失部分，UP主指出自己缺少数据复盘 skill。
- **其他类**：
  - 包括记忆系统、文件调用、模画工具、创作管理等。

## Skill 使用场景与自动化
- **写作任务管理**：创建前端界面管理30多个写作类 skill，打开页面即可选择适用 skill。
- **信息处理自动化**：
  - 播客处理工作流：自动拉更新、选播客、提取字幕、总结存飞书，每个步骤独立 skill，合并后实现全自动。
  - RSS 推送：每6小时和24小时自动汇报，内容推送到网站。
- **产品设计效率**：用 AI 生成原型图和文档，格式标准化，节省时间。
- **设备分工**：
  - 轻量化任务（如 RSS 更新）放手机处理，推送至飞书 IM。
  - 复杂任务（如代码修改、原型制作）放电脑处理，需浏览器查看效果。

## 创建 Skill 的四种方法
UP主总结了从工作中提炼 skill 的方法，具体步骤如下：
1. **反对法（从交付物入手）**：
   - 步骤：找一份最满意的交付物（如产品设计文档PRD、周报、分析报告），发给 AI 讨论，制作 skill 以产出类似内容，确保一致性。
2. **流程法（从流程入手）**：
   - 步骤：以播客处理为例，先分多个 skill 实现子流程（拉更新、选播客、提取字幕、总结存飞书），每个跑顺后合并为一个 skill。
3. **归纳法（从聊天记录入手）**：
   - 步骤：查看与 AI 的聊天记录，发现重复任务（如每次取标题），制作 skill（如标题制作 skill），替代手动提示。
4. **经验提炼法（从隐性知识入手）**：
   - 步骤：将置可议会的知识（如写作风格、审核标准、代码规范）建立 skill，逐步优化完善。

## 提升 Skill 使用效率的方法
- **记录常用 skill**：在 The Cloud 的 MD 文档中记录常用 skill（如写作风格、审核标准），AI 执行时严格遵守。
- **创建管理界面**：针对 skill 多的情况（如30多个写作 skill），制作前端界面，便于快速选择。
- **自动化处理**：将 skill 自动化，如播客处理无需人工参与，完全自动运行。
- **分设备管理**：轻量化任务（如 RSS 更新）放手机处理，复杂任务（如代码修改）放电脑处理。
- **统一管理**：用 GitHub 管理 skill 文件夹，确保跨电脑使用；用软链接同步 skill（类似 Windows 快捷方式），让多个应用程序（如 Codex、Cloud Code、OpenCloud）访问同一 skill。
- **环境变量管理 API-K**：将 API-K 保存在环境变量中，避免泄露，通过 Cloud Code 设置。
- **迭代 skill**：
  - 记录迭代：使用 Skill Logger 和 Skill 分析 skill，记录使用日志和分析频率，优化 skill。
  - 自迭代：在 skill 中加入自迭代能力，每次对交付物不满意时，修复并更新 skill 内容，实现持续优化。

## 总结
UP主通过整理145个 AI skill，参考刘比歇夫工作法进行管理，将 skill 分类为7类并应用于写作、信息处理、产品设计等场景，提升工作效率。他分享了创建 skill 的四种方法（反对法、流程法、归纳法、经验提炼法）和提升使用效率的策略（如记录、界面管理、自动化、设备分工、GitHub管理、软链接同步、环境变量保护、迭代优化），强调通过系统化管理实现 AI 复利效应，突破个人能力限制。核心要点是：AI skill 作为个人生产力工具，需持续创建、分类、自动化和迭代，以最大化效率和创新。
