# Hermes Agent从入门到精通（五）-实战篇，从零构建多代理AI团队

> [极客魔导师](https://space.bilibili.com/55035111) | [BV1StowBcEMN](https://www.bilibili.com/video/BV1StowBcEMN) | 时长 9:36

## 引言：问题与解决方案
### 单一AI助手的局限性
- 使用一个AI助手处理所有任务时，短期可行，但长期会导致：
  - AI人格混杂、上下文混乱、记忆嘈杂、工作流无条理。
  - 这不是用户的问题，而是方法的问题：单一通才助手无法应对复杂任务。

### Hermes Profiles的作用
- Hermes Profiles 是一个关键功能，能将AI从单一助手升级为多代理团队协作。
- 它不是简单给AI换名字或头像，而是创建隔离的代理环境，每个Profile独立拥有自己的记忆、绘画、技能、人格、配置、cron状态和网关状态。
- 核心观点：构建一个团队，而不是让一个通才超载。

## Hermes Profiles详解
### 隔离的七个维度
- 每个Profile可以隔离：配置、绘画、记忆、技能、人格、cron状态、网关状态（共七个维度）。
- 例如：研究专家代理不会继承工程师代理的技术黑化；作家代理不会被调试器的日志信息污染。
- 每个角色保持专业性和一致性，就像真实办公室中财务和营销不会共用同一个邮箱。

### 隔离的重要性
- 当所有东西共享同一份记忆和语气时，多代理系统会失败。
- 隔离确保每个角色有清晰界限，才能持久专业化。
- 隔离使代理能随时间变得更加一致和稳定，避免混乱混合体。

## 四角色模型
- 推荐新手使用四角色模型，映射真实工作流程，易于上手和实用。

### 协调者
- 角色：团队的门面和项目经理。
- 核心职责：接收用户需求、拆解任务、分配工作、汇总成果。
- 核心能力：任务拆解和判断力（例如，将“做一个APP”拆解为市场调研、技术选型、UI设计、编码、测试、上线等环节并分配给对应代理）。
- 特点：不需要深入执行，但必须清楚全局。

### 研究专家
- 角色：团队的信息守门人。
- 核心职责：搜索和验证信息（例如，查找技术文档、验证方案可行性、收集竞争对手信息）。
- 核心能力：好奇、严谨、深入，追根究底找到最可靠答案。
- 作用：降低系统中的“幻觉自信”（AI对错误信息的过度自信）。

### 作家
- 角色：内容的雕琢者。
- 核心职责：将验证过的信息转化成用户能理解、愿意看、有价值的内容。
- 核心能力：叙事和表达（口语化表达、逻辑组织、情绪把控）。
- 特点：不用兼任研究员或调试器时，可全身心投入写作，提升输出质量。

### 工程师
- 角色：团队的技术执行者。
- 核心职责：将方案变成真实可运行的代码，处理技术实现细节（代码编写、bug修复、性能优化、技术债务清理）。
- 核心能力：实现和调试，务实、精确、严谨。
- 特点：允许保持深度技术性，不用操心表达，提升实现质量。

- 四角色协作流程：协调者接受任务→拆解分配→研究专家收集信息→作家组织内容→工程师实现方案→协调者汇总交付。

## 配置文件：Soul.md与agents.md
### Soul.md的定义
- 位于Profile目录下的markdown文件，定义代理的核心特质。
- 内容包括：价值观、沟通风格、行为禁忌、专长领域等。
- 作用：赋予代理个性，使研究专家不会用工程师口吻说话，作家不会像工程师思考，确保真正多代理团队。

### agents.md的作用
- 配套文件，存放所有Profile共享的任务上下文。
- 包含团队级别的规则和共享信息，用于项目级上下文。
- 分工：Soul.md创造不同代理，agents.md提供共享上下文。

## 实战步骤
- 前提：Hermes已正确安装并可正常运行。

### 创建新Profile
- 使用 `clone` 参数基于现有配置创建新Profile。
- 使用 `Hermes Profile List` 确认所有Profile已正确创建。
- 建议：编辑Profile配置前，先切换到该Profile，使用命令 `Hermes Profile Use` + 名称（例如 `Hermes Profile Use Allen`）。
- 使用 `Hermes Profile` 确认是否进入。
- 使用 `Hermes Config Path` 查看Hermes配置目录位置。

### 编辑Soul.md和agents.md
- 为每个角色编写完整的Soul.md配置（以研究者为例，其他Profile步骤相同）。
- 编辑Soul.md：将对应内容复制进去。
- 编辑agents.md：存放团队级别规则和共享信息。
- 编辑 `.hermes` 下的 `teamagents.md`：团队架构说明文件，记录团队整体架构、角色职责、编辑和协作流程。

### 测试任务
- 切换到协调者Profile：输入命令 `Hermes Profile Use Default`。
- 启动Agent：输入 `Hermes`。
- 输入任务示例：我需要一篇关于2026年人工智能发展趋势的深度分析文章，目标读者是技术决策者，要求内容准确有数据支撑、结构清晰。
- 观察Hermes输出：协调者会拆解任务并分配给其他角色（研究专家、作家、工程师）协作完成。

## 总结
- 核心要点：通过Hermes Profiles功能，可以创建隔离的多代理团队（如协调者、研究专家、作家、工程师），每个角色拥有独立的记忆、人格等七维度隔离，确保专业性和一致性。使用Soul.md定义角色灵魂、agents.md提供共享上下文，结合实战步骤搭建团队，能有效解决单一AI助手导致的混乱问题，提升任务处理效率和输出质量。
