# 放弃写文档吧，这个Skill让原型自带说明书

> [AI产品经理养成记](https://space.bilibili.com/47333744) | [BV1JioTBwEyS](https://www.bilibili.com/video/BV1JioTBwEyS) | 时长 2:40

## 问题背景
### 原型交付的常见痛点
- 产品经理或原型交付时，画完原型后还需编写一份需求说明文档（如PRD），两者必须保持一致。
- 修改一处内容需同步更新原型和文档，过程繁琐且易出错。
- 最终常导致原型与文档版本不一致，影响团队协作和研发效率。

## 解决方案：Prototype Spec Editor Skill
### Skill概述
- 一个基于AI的技能（Skill），名为“Prototype Spec Editor”，用于将普通原型页面升级为一体化原型。
- 一体化原型自带结构化需求说明，使原型成为可阅读、可维护的需求文档，替代传统的原型加PRD文档组合。

## 实操步骤
### 如何使用Skill
1. **执行Skill**：运行Prototype Spec Editor Skill。
2. **指定任务**：告诉它“给全部页面添加页面需求描述”。
3. **生成结果**：自动生成一体化原型页面，其中包含结构化需求说明。
4. **编辑与维护**：
   - 点击“编辑需求说明”按钮，可对已有需求描述进行编辑或删除。
   - 通过“增加章节”功能新增规则或字段：输入字段名称、选择组件类型（如选填），保存即生效。
5. **版本管理**：Skill自动记录版本；点击“恢复生成版本”可回退到初始版本。

## 功能详解
### 一体化原型的特点
- **结构化信息展示**：需求描述按页面模块、字段结构化生成，而非简单文字段落。
- **实时可编辑性**：支持在线编辑、删除和新增章节，保存后立即生效。
- **内置版本控制**：自动跟踪修改历史，允许恢复至生成时的初始版本，确保可追溯性。

## Skill的作用与优势
- **合二为一的交付**：将原型和说明文档融合，现在只需交付一个一体化原型即可覆盖所有需求，减少文档维护成本。
- **提升效率与准确性**：避免双文档同步更新的麻烦，降低不一致风险，加快产品交付流程。
- **当前状态与优化**：Skill仍在打磨中，UP主邀请用户反馈优化建议，以共同完善工具。

## 总结
Prototype Spec Editor Skill通过AI技术将原型与结构化需求说明结合，解决了传统原型交付中文档维护繁琐、易出错的问题，使原型本身成为可维护的需求文档，显著提升产品经理的工作效率。UP主正在持续优化该技能，并欢迎社区讨论以改进功能。
