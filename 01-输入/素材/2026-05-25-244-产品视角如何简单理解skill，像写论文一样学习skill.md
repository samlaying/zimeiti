# 产品视角如何简单理解skill，像写论文一样学习skill

> [可爱爱吃大可爱多](https://space.bilibili.com/3546772550978314) | [BV1ziddBhEND](https://www.bilibili.com/video/BV1ziddBhEND) | 时长 10:09

## 第一部分：Skill的基础认知

### Skill的定义和作用
- Skill是AI的技能，类似于一个外挂，将常用信息组合成标准文件（如提示词）。
- 作用：作为AI的任务说明书，帮助AI执行特定任务，例如智能点奶茶功能。
- 通俗理解：Skill就像我们通过学习或考证书获得的技能（如做饭、开车），AI通过Skill学习如何完成任务。

### Skill的目录结构
- Skill文件（必须存在）：核心任务说明书，定义AI的基本功能、输入输出信息、执行逻辑等。
- Reference（资料手册）：存放参考资料，如规则、流程、数据库等，用于查询操作流程和确认信息。
- Scripts（工具脚本）：存放可执行的脚本或工具，用于调用外部接口（如外卖平台API）。
- Assets（参考文件）：存放非必要资源，如图片、示例文件，用于提供参考（如奶茶图片）。

### 以智能点奶茶为例的Skill拆解
- **Skill文件**：明确目标任务（如收集下单信息：口味、糖量、地址等）。
- **Reference**：基于最新信息查询店铺和规格，筛选符合条件的店铺。
- **Scripts**：执行下单操作，调用外卖平台API生成订单。
- **Assets**：提供参考奶茶图片（可选，非必要文件）。

### Skill的执行步骤（8个步骤）
1. **用户输入**：用户通过平台输入信息。
2. **AI识别**：AI解析用户输入。
3. **查看Skill**：AI查看Skill文件明确任务。
4. **核对信息**：核对基础信息（如订单细节）。
5. **查新参考信息**：从Reference查询操作流程和库存信息。
6. **执行脚本**：调用平台接口执行下单。
7. **组织语言回复**：基于结果组织回复语言。
8. **用户查看结果**：用户接收最终结果（如订单确认和图片）。

### Skill的优势
- 配置、逻辑、数据、资源分离，使结构清晰。
- 提高查询速度，降低Token使用成本。
- 类似写论文：有框架、分支任务、书写规范和来源参考，便于理解和扩展。

## 第二部分：常用Skill的推荐与安装

### 常用Skill推荐（产品经理视角）
产品经理高频使用的Skill包括：
1. **需求完善**：优化和细化产品需求。
2. **下拉生成**：生成下拉菜单或类似元素（可能指UI组件）。
3. **流程图**：自动生成业务流程图。
4. **用爱江湖类的Skill**：可能指用户故事或用例生成。
5. **数据埋点**：帮助设计和规划数据埋点。
6. **查找SQL**：辅助查询数据库相关技能。
- 注：最后两个用得较少，前四个常从微信文档等平台使用。

### Skill的查找与下载
- **方法一：通过AI平台查找**
  - 给AI（如多包）发送需求说明，例如：“我需要一个UI的Skill，请帮我从网上找，我需要安装在什么平台，请给我安装指令。”
  - AI通常能提供准确答复和安装指令，准确率约99%。
- **方法二：从Skill网站查找**
  - 使用关键词检索相关Skill网站。
  - 找到Skill后，查看详情页，获取DipTub地址（可能指GitHub或其他代码托管平台）。
  - 点击代码部分，下载安装包。

### Skill的安装实操步骤
1. **使用命令行安装**：
   - 通常使用MPX（可能指npm或类似包管理器）执行安装命令，例如：`npm install skill-name`（具体命令根据Skill而定）。
   - 安装过程中会显示提示，等待安装成功。
2. **验证安装**：
   - 在项目文件中检查是否生成对应文件（如Skill、Reference等文件夹）。
3. **设置全局应用**：
   - 如需所有项目应用，可在设置中将Skill配置为全局。
4. **替代安装方式**：
   - 从网站下载安装包后，手动放入项目文件夹，然后使用命令行安装（如解压后执行安装命令）。

### 其他平台参考
- 国内平台如Code Party、Code等，使用方式类似，可尝试探索。

## 核心要点总结
理解Skill就像写论文一样：它将AI的任务拆解为结构化的模块（如任务说明书、参考资料、脚本、资源），通过分离配置、逻辑和数据，提高效率并降低资源消耗。产品经理可以从AI平台或Skill网站查找并安装常用Skill（如需求完善、流程图），使用命令行工具快速部署，从而在需求文档和开发对接中提升工作效率。
