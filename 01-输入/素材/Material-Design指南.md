# 📘 UI设计笔记：关于谷歌 Material Design 你需要知道的一切

## 一、什么是 Material Design？

### 1.1 基本定义

- Material Design 是谷歌的**开源设计系统**
- 被认为是最**经典、权威**的设计规范之一
- 任何 UX/UI 设计师和前端开发人员都应该熟悉并遵循

### 1.2 起源与发展

```
时间线：
├── 谷歌收购安卓 → 开始在安卓/Web 环境下建立设计规范
│   （按钮、导航栏、Snackbar 交互方式、动效、文案等）
│
├── 2014年 → 谷歌开发者大会首次推出 Material Design（M1）
│   ├── 打破当时扁平化设计趋势
│   ├── 设计灵感源自真实世界的材料
│   ├── 提倡鲜活色彩、有层次的界面、遵循物理规律的动效
│   └── 引入 Z 轴高度/深度概念，利用阴影表现元素状态
│
├── 2021年 → 推出 Material Design 3（M3 / Material You）
│   └── 在 M2 基础上提出新概念和功能
│
└── 目标 → 在安卓乃至整个交互领域，建立一套
           通用的跨平台设计语言
```

### 1.3 核心目的

| 目的 | 说明 |
|------|------|
| **规范标准化** | 帮助设计师和程序员统一视觉和交互元素 |
| **一致性** | 提高用户体验的一致性和质量 |
| **跨平台** | 建立通用的跨平台设计语言 |
| **开源共享** | 为设计师提供设计准则素材，为程序员提供组件库 |

## 二、Material Design 对设计师的价值

### 2.1 90% 的 UI 组件是通用的

> 作为设计师，我们所使用的组件 **90% 都是通用的**。

常见通用组件：
- 卡片（Card）
- 下拉菜单（Dropdown）
- 导航栏（Navigation Bar）
- 底部弹出面板（Bottom Sheet）
- 等等

**关键信息：**
- 开发人员可以直接从**苹果、安卓或 Web 的各种 UI 组件库**中调用
- 然后根据需要调整颜色、字体
- **完全不需要重新设计这些组件的交互方式和功能**——因为它们已经足够完善，被用户所接受

### 2.2 Material Design 作为"教科书"

设计师需要对通用组件非常熟悉，而 Material Design 可以作为你的教科书：

| 学习内容 | 说明 |
|---------|------|
| **组件功能** | 某个组件是做什么用的 |
| **交互方式** | 用户如何与它互动 |
| **使用原则** | 什么场景下使用、什么场景下不使用 |
| **响应式设计** | 在不同设备上如何适配 |
| **无障碍要求** | 无障碍设计的具体规范 |

> 这些规范都是基于大量实际应用和用户反馈的总结，即 **Best Practice（最佳实践）**。

### 2.3 作为设计素材来源

**Figma 中的使用：**
- Figma 已将 Material Design 素材库添加进来
- 可以直接找到需要的组件
- 以此为基础进一步调整

**程序员的使用：**
- Material Design 本身提供组件代码
- 程序员可以直接拿来使用
- 需要根据项目类型决定：是直接用 Material Design，还是用其他更基础的组件库

### 2.4 Material Design 包含的内容

```
Material Design 内容体系
├── 组件规范（Components）
├── 无障碍设计规范（Accessibility）
├── Design Token
├── 布局系统（Layout）
├── 颜色系统（Color System）
├── 文字系统（Typography）
├── 图标（Icons）
└── 动效（Motion）
```

> 这些都是**非常实用、权威**的 UX/UI 学习资料。

### 2.5 使用建议

> ⚠️ Material Design 体量过于庞大，**并不一定适合你的项目**。

- 对于**颜色、文字系统**等基础部分 → 建议**作为参考**，而不是直接照搬
- 需要根据项目类型灵活调整
- **Material Design 网站本身没有中文版**，但网上可以找到很多中文翻译版本

## 三、Material Design 3（M3）核心概念

### 3.1 颜色系统：动态颜色方案

#### 传统颜色系统（对比）

```
传统方式：
├── 定义主颜色（Primary）
├── 定义辅助色（Secondary）
├── 定义中性色（Neutral）
└── 给不同元素指定颜色（如：按钮用主颜色，边框用浅灰色）
```

#### M3 的动态颜色方案

```
M3 动态颜色流程：
├── 输入源：用户的手机壁纸 / App 中某张图片
├── 算法提取 → 生成一个「原色」
├── 从原色 → 生成 5 个关键色
├── 从关键色 → 生成一系列色板
└── 把颜色赋予到不同功能的 UI 元素上
    → 实现用户界面的动态颜色方案
    → 提供更具个性化的视觉体验
```

#### 关于动态颜色的评价

| 优点 | 缺点 |
|------|------|
| 个性化视觉体验 | **极大弱化品牌形象** |
| 技术上很超前 | 使产品逐渐成为安卓的一部分 |
| 用户参与感强 | 并不适合目前大多数产品 |

> **总结：** 这个功能很超前，但很可能并不适合目前大多数产品。

---

### 3.2 文字系统（Typography）

#### M3 字体分类

| 功能分类 | 说明 |
|---------|------|
| **Display（显示）** | 最大号，用于视觉焦点 |
| **Headline（大标题）** | 页面主要标题 |
| **Title（标题）** | 区域或段落标题 |
| **Body（正文）** | 正文内容 |
| **Label（标签）** | 按钮文字、标签等小型文字 |

#### 每个分类下的尺寸分级

每个类别下又分为 **S（Small）/ M（Medium）/ L（Large）**

→ 让设计师可以根据**屏幕尺寸**进一步选择合适的字号

#### 相比 M2 的改进

- **命名方式更加逻辑清晰**
- 使用了 **Design Token**，可以灵活地修改各种参数

---

### 3.3 动效（Motion）

#### M3 强调的两个核心要素

##### ① 缓动（Easing）

> 模拟**真实世界的运动效果**，做出逐渐的加速和减速，**而不是匀速运动**。

| 运动类型 | 描述 |
|---------|------|
| ✅ 缓动效果 | 逐渐加速 → 逐渐减速（自然感） |
| ❌ 匀速运动 | 恒定速度（机械感、不自然） |

**Token 化：**
- 动效已 Token 化
- 可以根据使用情景选择合适的缓动效果

##### ② 持续时间（Duration）

> 动画的持续时间**不宜过快或过慢**，否则会造成视觉上的不和谐。

**具体规范示例：**

| 场景 | 规则 |
|------|------|
| **关闭/退出** 转场 | 动效时间**小于**进入新页面的动效时间 |
| **进入新页面** 转场 | 动效时间**大于**退出的动效时间 |

**原因：** 用户需要更多的注意力放在**新的任务**上。

---

### 3.4 图标（Icons）

#### M3 图标的四个可调节属性

| 属性 | 说明 |
|------|------|
| **粗细（Weight）** | 线条的粗细程度 |
| **填充（Fill）** | 是否填充内部 |
| **等级（Grade）** | 视觉权重 |
| **视觉尺寸（Optical Size）** | 图标的显示大小 |

#### 在 Figma 中使用

- 打开 **Material Symbols** 界面
- 找到合适的图标
- 调整四个属性参数
- 直接拿来使用

---

### 3.5 多层级（Elevation）

#### 概念

> Material Design 提倡给每个 UI 元素都赋予一个 **Z 轴上的坐标**，即每个元素都有自己的高度。

**共 5 个层级。**

#### M2 vs M3 的区别

| 版本 | 层级表达方式 |
|------|------------|
| **M2** | 高度用**阴影**来表达 |
| **M3** | **首先使用不同颜色**来表达，阴影在必要时用作**区分或强调** |

> M3 的改变：颜色优先，阴影辅助。

## 五、学习建议

- [ ] 将 Material Design 作为 UX/UI 教材，**经常查阅**
- [ ] 帮助自己理解 UI 元素的**底层逻辑**
- [ ] 设计出更符合**用户习惯**的交互产品
- [ ] 重点学习：组件规范、颜色系统、文字系统、动效规范
- [ ] 理解 Design Token 的概念和应用
- [ ] 在 Figma 中实际操作 Material Design 组件库
- [ ] 根据自己项目特点灵活运用，而非照搬照抄
