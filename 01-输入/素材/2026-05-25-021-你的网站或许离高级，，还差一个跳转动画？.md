## 转场动画教程：提升网站交互体验

### 一、核心原理与目标
*   **目标**：在页面跳转过程中加入流畅的转场动画，使网站交互更丰富、高级。
*   **基本原理**：打断原生的“点击 -> 地址栏变化 -> 内容刷新”流程。具体步骤为：
    1.  点击跳转链接。
    2.  播放转场动画入场，覆盖整个页面。
    3.  在动画覆盖下，后台加载新页面内容（或进行路由切换）。
    4.  新内容就绪后，播放转场动画退出，露出新页面。
*   **动画结构**：通常分为三部分：
    *   **入场动画**：页面即将跳转前，动画元素进入并包裹页面。
    *   **填充/持续状态**：动画内容贯穿于页面跳转的整个过程。
    *   **退场动画**：新页面加载完成后，动画元素退出，露出新页面内容。

### 二、传统实现方案（基于页面刷新）

#### 1. 基本实现步骤
*   **准备页面**：创建两个内容相似但有区别的HTML页面（如`page1.html`和`page2.html`）。
*   **添加跳转逻辑**：在页面中为可点击元素（如`<p>`标签）绑定点击事件。在JavaScript中编写跳转函数，调用`window.location.href`进行跳转。
    ```javascript
    function jumpToPage(url) {
        window.location.href = url;
    }
    // 调用示例：jumpToPage(‘./page2.html’);
    ```
*   **创建转场动画**：使用HTML/CSS编写一个全屏的转场动画元素。关键CSS样式：
    ```css
    .loading {
        position: fixed; /* 固定定位 */
        width: 100vw;
        height: 100vh;
        z-index: 9999; /* 确保在最顶层 */
        /* 其他动画样式 */
    }
    ```
*   **控制动画生命周期**：
    *   **动画退出**：监听`window`的`load`事件，页面加载完成后为动画元素添加退出类名（如`loading-out`）。
    *   **动画进入与跳转**：将跳转函数改为先触发动画进入（如添加`loading-in`类名），然后设置一个定时器（如1秒），延迟执行`window.location.href`跳转，以确保动画入场完成。

#### 2. 此方案的严重不足
*   **问题一（动画流畅性）**：页面跳转本身是一次**刷新**。在网络加载慢（如模拟慢速3G）时，动画会在页面刷新时**被打断并重新播放**，导致衔接不自然。
*   **问题二（交互完整性）**：通过浏览器的**前进/后退按钮**进行页面跳转时，**不会触发**我们编写的JavaScript跳转函数，因此**转场动画完全失效**。

### 三、进阶方案：Vue无刷新跳转实现

为了克服传统刷新跳转的缺陷，需要采用**无刷新跳转**（SPA单页应用模式）。以下使用Vue 2框架进行演示。

#### 1. 方案优势
*   **完美解决动画中断**：页面内容切换通过路由完成，**不触发浏览器刷新**，因此入场动画不会被中断。
*   **完美覆盖浏览器导航**：通过监听路由变化，无论是通过链接点击还是浏览器前进/后退按钮，都能触发转场动画，保证交互一致性。

#### 2. 实现步骤（Vue 2 示例）
*   **组件抽离**：将转场动画（如`<Loading>`组件）独立出来，并在根组件（`App.vue`）中引入显示。
*   **动画退出逻辑**：
    *   由于是单页应用，无法使用`window.load`事件。需要在组件的`mounted`生命周期钩子中**主动检查**页面加载状态。
    *   使用定时器轮询`document.readyState`，当状态为`“complete”`时，清除定时器并触发`Loading`组件的退出方法。
    ```javascript
    // 在App.vue的mounted中
    checkLoading() {
        const timer = setInterval(() => {
            if (document.readyState === “complete”) {
                clearInterval(timer);
                this.$refs.loading.loadingOut(); // 调用子组件的退出方法
            }
        }, 30);
    }
    ```
*   **动画进入与路由跳转**：
    *   在`router`实例上配置全局前置守卫`beforeEach`。
    *   在守卫中调用`Loading`组件的进入方法，并将路由跳转的`next`函数作为参数传入。
    *   在`Loading`组件的进入方法中，播放入场动画，并设置一个延迟（如1秒）后执行`next()`，以允许动画播放完毕。
    ```javascript
    // 在App.vue的mounted中配置路由守卫
    this.$router.beforeEach((to, from, next) => {
        this.$refs.loading.loadingIn(next); // 传入next函数
    });

    // 在Loading组件中
    loadingIn(next) {
        // 播放入场动画...
        setTimeout(() => {
            next(); // 1秒后执行路由跳转
            // 同时可能需要通知App重新检查加载状态
        }, 1000);
    }
    ```

### 总结
核心要点在于：为了实现流畅、完整的页面跳转转场动画，必须采用**无刷新跳转**技术。传统基于`window.location`刷新的方案存在动画被中断和浏览器导航按钮失效两大硬伤。使用现代前端框架（如Vue、React）的路由功能，可以轻松控制页面内容的异步切换，从而在不刷新页面的前提下，完美地将转场动画融入整个用户交互流程中，显著提升网站的高级感和用户体验。
