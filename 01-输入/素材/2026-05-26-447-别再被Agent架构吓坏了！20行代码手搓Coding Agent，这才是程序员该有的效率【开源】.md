# 别再被Agent架构吓坏了！20行代码手搓Coding Agent，这才是程序员该有的效率【开源】

> [杰哥_ai_team](https://space.bilibili.com/270076899) | [BV1VJZMBRESf](https://www.bilibili.com/video/BV1VJZMBRESf) | 时长 4:06

## 一、引言：Agent架构的常见问题
- 市面上很多Agent教程将本来简单的东西复杂化，堆砌了各种概念如chain、graph，导致初学者一看就晕、一用就不会。
- 对于大多数业务场景、个人使用或学习来说，根本不需要那么重的架构。
- 虽然像Cloud Code、OpenCloud等产品功能强大，但剥离应用层后，底层的Agent逻辑其实非常纯粹和简单。

## 二、Agent的本质
- Agent的本质是：**模型思考 + 外部工具调用 + 多次循环迭代反思**。
- 这是一个简单的循环过程，核心在于工具使用和迭代优化。

## 三、实现方法：20行代码手搓Coding Agent
- UP主演示了用**20行代码**创建一个Coding Agent。
- 框架开源，可在GitHub搜索UP主提供的资源进行学习和使用。
- 强调了简洁高效的实现，避免过度工程化。

## 四、实操演示：创建React Landing Page项目
- **具体例子**：使用Agent在本地创建一个React技术栈的landing page项目，并部署到**3010端口**进行预览。
- **实操步骤**（基于字幕描述）：
  1. 启动Agent，命令它执行创建React landing page的任务。
  2. Agent自动创建项目目录。
  3. 安装NPM包（由于Agent会进行多轮自我迭代和工具调用，需要一些时间）。
  4. 项目完成后，部署到3010端口启动服务。
  5. 测试功能，如页面跳转（示例中测试了跳转到GitHub，虽因网络问题稍作调整，但最终成功）。
- **结果**：项目虽看起来简陋，但功能较完善，展示了Agent的自主开发能力。

## 五、内置工具和自定义支持
- 框架内部内置了**11个核心工具**，包括：文件操作、命令行、联网搜索、subagent等。
- 支持用户**自定义工具**，增强了灵活性和适用性。
- 工具调用使Agent能执行多种任务，如创建文件、运行命令、搜索信息等。

## 六、总结与建议
- Agent架构并不复杂，其核心是模型与工具的结合以及迭代优化。
- 不要被网上复杂的教程或框架限制想象力，应根据实际需求选择简单高效的方法。
- 鼓励程序员探索和使用开源工具，提升开发效率。

**核心要点总结**：这个视频通过20行代码手搓Coding Agent的例子，揭示了Agent的本质就是模型思考与外部工具调用的循环迭代过程，强调了在大多数场景下不需要复杂架构，提倡使用简单、开源的方法来提高编程效率，从而避免被过度工程化的概念所困扰。
