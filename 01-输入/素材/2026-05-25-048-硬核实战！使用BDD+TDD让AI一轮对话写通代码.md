# 硬核实战！使用BDD+TDD让AI一轮对话写通代码

> [硅基包工头](https://space.bilibili.com/3422876) | [BV1NpRmBLEbp](https://www.bilibili.com/video/BV1NpRmBLEbp) | 时长 4:50

## 一、背景与核心思想
- **BDD (行为驱动开发) 与 TDD (测试驱动开发)** 在AI编程时代变得尤为重要。
- **核心流程**：先定义产品行为（BDD用例），再编写测试代码（TDD），最后实现功能代码。
- **AI时代的变化**：传统BDD/TDD流程（如多人协作、使用特定框架）可能过于繁琐，应根据“生产力模型已变”的实际情况灵活运用，知其所以然。
- **本视频目的**：演示如何在Cloud Code中通过BDD+TDD，让AI一次性准确理解需求并完成编码。

## 二、准备工作与配置
### 2.1 安装与准备TDD Skill
1.  **安装Super Powers插件**：
    - 在Cloud Code中输入 `/plugin` 命令回车，搜索“Super Powers”并安装。
    - 该插件已内置TDD的Skill。
2.  **手动配置TDD Skill**：
    - 作者未完整安装Super Powers，而是将其TDD Skill的代码从GitHub仓库中拷贝出来。
    - 将代码粘贴到项目的 `.cloud/skills` 目录下。

### 2.2 配置BDD规则（无专用Skill）
- **反直觉做法**：作者没有使用任何BDD Skill，而是在项目的 `CLAUDE.MD` 文件中添加了两行规则。
- **关键决策**：让AI模型将BDD的用例描述作为注释直接写在对应的代码文件中。
- **理由**：
    - 传统BDD流程（如使用Gherkin语言写feature文件）适用于团队沟通，但在单人开发中作用有限。
    - **核心价值**：BDD描述用例的方式是标准的，大模型经训练后看到“BDD”一词就知道如何组织需求描述。
    - **流程简化**：跳过了编写独立feature文件等步骤，专注于让AI理解标准化的用例描述。

## 三、实战演示：为战棋游戏增加“毒蛇”随从
### 3.1 明确需求
- **项目**：一个使用高豆特（Godot）制作的美少女战棋游戏。
- **基础规则**：双方使用卡牌消耗魔力召唤随从，消灭对方主将获胜。
- **新增需求**：
    1.  新增一个“毒蛇”随从单位。
    2.  新增“中毒”机制：毒蛇可使用技能给敌方施加中毒状态。
    3.  中毒单位每回合结束时损失1点生命。
    4.  **关键细节**：中毒状态**不可叠加**。

### 3.2 BDD用例定义
- 将需求发送给AI（演示中使用minimax M2.7模型），让其编写BDD用例描述。
- AI生成了用例，但最初遗漏了“中毒不可叠加”的细节，经一轮对话修正后得到准确用例。

### 3.3 TDD测试与实现
1.  **引用TDD Skill**：使用 `/` 命令选择TDD技能，使过程更稳定。
2.  **AI执行TDD流程**：
    - **第一步**：AI先编写无法通过的测试代码。
    - **第二步**：AI开始编写实现代码以使测试通过。
3.  **容错性演示**：
    - 使用的低成本国产模型minimax M2.7错误率较高。
    - AI自行经历了多轮“修改-测试失败”的循环。
    - **核心价值**：AI在明确的测试目标下，能够自动化地迭代、修正，最终在约10分钟后让所有测试通过。
4.  **验证与重构**：
    - 实际运行游戏，功能完美实现。
    - 即使测试通过，如果实现方式不合理，也可以借助已有的测试套件，放心地让AI进行多轮代码重构。

## 四、方法论的适用场景与限制
### 4.1 优势
1.  **促进良好代码结构**：能够轻松编写测试代码，前提是代码结构可测试（如本例中将游戏逻辑剥离为独立的状态层）。TDD天生导向好结构。
2.  **适合明确接口的业务逻辑**：非常适合开发可明确定义输入输出的业务逻辑。
3.  **提升AI编码可靠性**：给AI提供了明确的“对错”验证标准（测试），使其能自动、准确地完成工作，减少人工干预。

### 4.2 限制与不适用场景
1.  **代码结构要求**：要求被测代码结构良好、可测试。
2.  **不适合GUI测试**：不太适用于图形界面的测试，应用在此类场景可能会很痛苦。

## 五、总结
本视频的核心在于展示一种面向AI时代的高效率开发范式：将BDD的标准化需求描述与TDD的验证驱动相结合。通过预先在`CLAUDE.MD`中定义BDD规则、配置TDD Skill，为AI编程搭建了清晰的“脚手架”。实战中，即使使用能力一般的模型，通过“AI先写测试再写实现，并自行迭代至测试通过”的流程，也能在需求明确后实现高度自动化的、可靠的功能开发。这种方法的本质是**利用标准化的需求描述和自动化的测试验证，来精准引导和约束AI的编码过程**，从而在保证代码质量的同时，极大提升开发效率，是单人开发者利用AI编程的利器。但其成功应用依赖于可测试的代码架构，并更适用于逻辑清晰的后端业务开发。
