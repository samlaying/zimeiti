# Claude Code Agent Teams上手指南+项目实测

> [程序员阿江-Relakkes](https://space.bilibili.com/434377496) | [BV1gwcAzkEhw](https://www.bilibili.com/video/BV1gwcAzkEhw) | 时长 10:06

## 视频笔记：Claude Code Agent Teams上手指南与项目实测

### 一、 单Agent的瓶颈与Agent Teams的提出
使用Claude Code处理复杂任务时，常遇到三个结构性瓶颈：
1.  **上下文遗忘**：对话越长，越容易忘记前面修改的内容。
2.  **任务串行**：本可并行处理的任务，只能逐个串行完成。
3.  **单一视角**：一个Agent只能从一个视角（如安全或性能）审查代码。
这些并非提示词问题，而是单Agent模式的固有限制。为此，Claude官方推出了 **Agent Teams**。

### 二、 什么是Agent Teams？
Agent Teams 允许多个Claude Code实例（Agent）协同工作。
- **结构**：包含一个团队管理者（Team Leader）和多个团队成员（Members）。
- **通信**：成员之间、以及管理者与成员之间均可实时通信。
- **工作流**：用户发出任务 → Team Leader拆解任务 → 分派给成员执行 → 成员从共享任务列表领取任务、更新状态、回传结果。

### 三、 Agent Teams 最佳使用场景
1.  **研究与审查**：对同一问题进行多角度分析。
2.  **新模块开发**：可并行开发不同组件。
3.  **竞争性假说**：针对一个Bug，让多个Agent提出并辩论不同解决方案，最终筛选出最佳答案。
4.  **跨角色并行开发**：遵循标准软件开发流程，设计完成后，前端、后端可并行开发；开发完成后，测试可立即跟进。

### 四、 Agent Teams 与 Subagent 的核心区别与选择
**核心区别**：
- **通信方式**：Subagent只能向主Agent报告结果；Agent Teams的成员之间可以直接通信。
- **自主性**：Agent Teams的成员更自主，能共享发现并协调。

**选择指南**：
- 选择 **Subagent**：当需要快速、专注的工作者完成独立任务并报告结果时。（例如：在多个目录中搜索特定模式的文件）
- 选择 **Agent Teams**：当需要成员间共享发现、互相质疑并自主协调时。（例如：进行PR代码审查，需要多个审查者互相评审对方的发现）

### 五、 配置与启用Agent Teams
1.  **开启实验功能**：
    - **方法一（持久化）**：编辑 `~/.claude/settings.json` 文件，添加配置 `"experimental": { "agentTeams": true }`。
    - **方法二（临时）**：在Claude Code终端中输入 `/config experimental.agentTeams true`。
2.  **重启终端**：配置后需重启Claude Code终端生效。
3.  **确保版本**：确保Claude Code是最新版本。

### 六、 实操演示1：并行代码审查
- **触发**：在提示中明确要求创建团队并指定角色。
- **示例提示**：“请创建一个团队，审核仓库中某个PR的代码提交。要求有三个角色：一个专注安全，一个检查性能，一个做测试覆盖率。”
- **执行**：Claude会自动创建包含三个Agent的审查团队，并行工作。
- **结果**：每个Agent独立审查PR，团队成员及管理者间可相互通信，最终整合报告。

### 七、 实操演示2：真实项目功能开发（UI重构）
1.  **提示**：要求创建一个5人团队，角色包括架构设计、后端、前端、QA、代码审查，并明确各自任务和执行顺序（架构先行 → 后端/前端并行 → QA测试 → 代码审查）。
2.  **自动创建**：Claude会自动创建团队、生成任务列表、设置任务间的依赖关系（如后端和前端任务依赖架构设计任务）。
3.  **并行执行**：
    - **架构Agent**：首先执行，完成系统设计。
    - **后端与前端Agent**：设计完成后，并行领取各自任务进行开发。每个Agent都是一个独立的Claude Code实例，拥有自己的提示词和上下文。
    - **QA Agent**：前后端开发完成后，被激活执行集成测试。
    - **Review Agent**：QA通过后，最终进行代码审查。
4.  **管理与协调**：整个过程由主Agent（Team Leader）协调，团队成员通过共享的任务列表（任务状态：待处理、进行中、已完成）和消息系统协作，任务分配使用文件锁防止冲突。

### 八、 Agent Teams 架构、通信与存储
- **架构**：
  - **领导（Leader）**：创建主会话，派生成员，管理全局任务和依赖。
  - **成员（Member）**：独立的Claude Code实例，接收提示词，领取并执行任务，更新状态。
- **通信**：
  - 每个成员有独立上下文。
  - 通过**消息系统**实现成员间、成员与领导间直接通信。
- **任务管理**：
  - 任务有依赖关系（如任务B依赖任务A完成后才能开始）。
  - 任务有状态（待处理、进行中、已完成）。
  - 通过**文件锁机制**防止任务被多个成员重复领取。
- **存储**：团队配置、任务信息等均以文件形式存储在本地 `~/.cloud` 目录下。

### 九、 测试数据与总结
- **测试项目**：一个完整的UI功能开发与重构流程。
- **完成度**：视频作者评价整体完成度约80分，过程中修复了一些前后端Bug。
- **Token消耗**：该测试任务总计消耗了约**3000万Token**。

## 核心要点总结
Claude Code Agent Teams通过引入“团队管理者-成员”的多Agent协作模式，有效解决了单Agent在处理复杂、可并行任务时面临的上下文丢失、串行瓶颈和单一视角问题。其核心在于允许成员间自主通信、通过共享任务列表协调工作、并能设置任务依赖以实现真正的流水线并行（如前后端并行开发）。尽管当前仍为实验性功能且消耗Token较大，但它为自动化处理大型软件工程任务提供了一种强大且接近人类团队工作流的范式。
