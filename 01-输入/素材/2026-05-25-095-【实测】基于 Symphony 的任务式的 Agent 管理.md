# 【实测】基于 Symphony 的任务式的 Agent 管理

> [李谱在探索](https://space.bilibili.com/394112399) | [BV1tC9YB6E7x](https://www.bilibili.com/video/BV1tC9YB6E7x) | 时长 1:50

## 基于 Symphony 的任务式 Agent 管理

### 管理方式介绍
- **任务式协作**：通过任务板管理 Agent，取代直接对话方式，用户只需发布和验收任务，无需实时监控 Agent 工作。
- **工具基础**：基于 Symphony（一个任务管理框架或工具），实现任务式 Agent 协作。
- **工作流程**：发布任务 → Agent 自动获取并执行 → 任务状态更新为 human review（需用户介入验收）→ 用户验收通过后标记为 done（完成）。

### 好处与优势
1. **异步协作**：用户与 Agent 的协作变为一步式，无需盯着 Agent 工作，只需不断发布、验收任务，提高效率。
2. **并行任务管理**：通过看板清晰查看每个任务的状态（如执行中、需验收、已完成），便于直观跟进和管理多个并行任务。
3. **注意力集中**：用户无需介入 Agent 具体执行细节，注意力可专注于更有价值的任务定义和发布，提升工作质量。

### 具体实操示例
- **任务1：研究分析报告**
  - 用户创建任务：“帮我出一份 DeepSeek V4 的分析报告”。
  - Agent 自动获取任务并执行，完成后状态变为 human review。
  - 用户验收报告：查看内容（关于 V4 的报告），确认合格后进入下一步。
- **任务2：报告转换为汇报文稿**
  - 用户基于任务1结果，发布新任务：“把刚刚的报告变成一个汇报的 HTML”。
  - Agent 生成 HTML 汇报文稿，从多个维度阐述 DeepSeek V4 特点。
  - 用户验收文稿，可在基础上修改，然后标记任务为 done。
- **关键点**：任务状态管理（如 human review）确保用户介入验收，避免自动化错误；任务链式执行（从研究报告到 HTML 转换）展示灵活协作。

### 总结核心要点
这段视频演示了基于 Symphony 的任务式 Agent 管理，通过任务板实现异步、并行的协作流程：用户发布任务，Agent 自动执行并更新状态，用户在关键节点验收。这种方式让用户从监控 Agent 工作中解放出来，专注于任务定义和发布，从而提高工作效率和注意力价值。
