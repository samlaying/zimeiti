# AI个人系统（002/X）建立你的第一个skill

> [小王小王XWXW](https://space.bilibili.com/12384464) | [BV1Q6F6z7Ex1](https://www.bilibili.com/video/BV1Q6F6z7Ex1) | 时长 5:29

## 建立你的第一个 AI Skill（能力扩展）

### 1. 为什么要自己动手创建 Skill？
- **核心观点**：虽然可以使用他人做好的 Skill，但更推荐**自己创建**。
- **原因**：你的工作方式、生活习惯、工作背景**只有你自己最清楚**。一个完全为你定制的 Skill 效用最高。
- **起点**：从你日常生活中**最常做的那件事**，或者工作中**反复困扰你的一个问题**开始。
- **例子**：UP主以“管理提醒事项（添加、修改、完成）”为例，希望通过对话而非打开APP来完成操作。

### 2. 动手前必须掌握的两个方法论
在创建 Skill 前，运用以下两个原则能决定其最终是否“好用”。

#### 2.1 Unex 原则：小而美，只做一件事
- **核心**：**一个 Skill 只做好一件事情**。
- **误区**：希望一个“任务管理 Skill”同时包含提醒、规划、拆解、写日记等功能。
- **问题**：功能堆砌会导致工具又大又复杂，用户不知何时用哪部分，最终结果是**能做但做不好**。
- **正确做法**：
    - **拆开**：将“管理提醒事项”和“写日记”设为两个独立的 Skill。
    - **组合**：多个小的、各自独立的 Skill 可以**互相调用**。例如，“写日记” Skill 可以调用“提醒事项” Skill 来检查今日完成任务。
    - **精髓**：像乐高积木，小而独立，自由组合，完成复杂任务。

#### 2.2 XY 问题：找到你真正的需求
- **核心**：区分你的**表面需求（X）** 和 **底层真正需求（Y）**。
- **例子剖析**：
    - **X 需求（表面）**：想让 AI 读取文件夹中的文件变动（哪些新建、修改）。
    - **追问“为什么”**：为什么需要知道文件变动？
    - **Y 需求（底层）**：真正需求是想让 AI **感知上下文**，了解“我今天做了什么、昨天做了什么、接下来打算做什么”，即理解我的工作状态。
    - **结论**：只做文件变动列出工具（满足X）作用有限；而**上下文感知系统**（满足Y）才是一个真正有用力的 Skill。
- **行动建议**：在动手前，多问一句“为什么需要这个？”，发现更本质的需求。

### 3. 实操工具：Skill Creator
- **它是什么**：一个帮你创建 Skill 的 **元 Skill**。
- **优势**：无需编写复杂配置文件或研究技术文档。
- **使用方式**：通过自然语言对话，告诉它你想做什么、你的场景是什么、希望它如何工作。
- **工作原理**：Skill Creator 会帮你把需求拆解清楚，并生成对应的 Skill 结构。
- **与 PromptOS 的整合**：上述 Unex 和 XY 原则已内置在 PromptOS 系统中。在与 AI 讨论、创建 Skill 时，它会自动应用这些原则，进行追问以确保聚焦本质。

### 4. 创建你的第一个 Skill 的实操步骤
1.  **选择一个切入点**：从一个**最常重复、最具体、最小**的日常任务开始。例如：管理待办事项、整理笔记、写周报。
2.  **用两大原则检验**：将你的初步想法，用 Unex 原则（是否只做一件事？）和 XY 问题（表面需求下的真正需求是什么？）告诉 AI，让它帮你审查和聚焦。
3.  **利用 Skill Creator 创建**：清晰地说出经过检验后的需求，借助 Skill Creator 工具，通过对话将其创建出来。

## 核心要点总结
视频的核心是引导你从一个**具体、微小且真实的个人问题或重复任务**出发，运用 **Unex 原则（保持单一职责、小而美）** 和 **XY 问题（深挖表面需求下的本质需求）** 两大方法论来检验和聚焦你的想法，最终借助 **Skill Creator** 这个对话式工具，亲手为你自己构建第一个 AI Skill。通过这个过程，你将开始逐步积累，最终构建出一整套完全适配你个人工作流的、可自由组合的 AI 能力系统。
