# Claude Code 更新太快跟不上？我做了个自动追踪系统帮你解决

> [阿墨讲AI编程](https://space.bilibili.com/143684723) | [BV1DuzDBdEqv](https://www.bilibili.com/video/BV1DuzDBdEqv) | 时长 17:26

## 视频笔记：Claude Code 自动更新追踪系统构建指南

### 一、背景与痛点
*   **问题**: Claude Code 更新频率极快（可能日更），导致用户（包括AI领域专家如Andrej Karpathi）难以跟上所有新功能和修复。
*   **目标**: 构建一个自动化系统，用于跟踪Claude Code的更新，解析其重要性，并通过个性化方式（如语音）通知用户，从而提升使用效率。

### 二、系统核心功能
1.  **自动抓取更新**:
    *   监控 Anthropic 官方在 GitHub 上维护的 Claude Code `changelog`（更新日志）。
    *   任何新的版本发布或更改都会被系统自动捕获。

2.  **AI 智能解析**:
    *   使用 **Gemini API**（特别是 `Gemini 3 Flash` 模型）对更新日志进行分析。
    *   功能包括：
        *   **分类标记**: 将更新条目标记为“新功能”、“修复”等类别，便于快速浏览。
        *   **生成摘要 (TLDR)**: 为每次更新生成简明扼要的总结。
        *   **聊天问答**: 用户可针对单个或多个版本的更新向AI提问，以更简单、个性化的方式理解变化。例如，可以问“LSP是什么？请像解释给八年级学生一样解释它”。

3.  **语音播报**:
    *   集成 **Gemini 的文本转语音 (TTS) API**。
    *   用户可以点击播放按钮，收听由自定义AI语音（如示例中的“Sharon”）朗读的更新摘要。
    *   支持倍速播放。

4.  **自动邮件通知**:
    *   使用 **Resend** 服务（提供慷慨的免费额度）发送电子邮件。
    *   可配置 **Cron Job**（定时任务），实现每日或每周自动发送邮件，包含最新的更新摘要。
    *   系统检测到新更新时即会发送通知。

5.  **数据缓存与持久化**:
    *   使用 **SQLite** 数据库在本地存储已分析的更新内容和生成的音频。
    *   避免重复计算和API调用，提升性能并降低成本。

### 三、技术架构与服务
系统基于以下几个核心服务和组件构建：
*   **数据源**: Anthropic GitHub 仓库的 `changelog`。
*   **AI 推理**: **Gemini API** (用于聊天分析和文本转语音)。
*   **邮件服务**: **Resend** (用于发送通知邮件)。
*   **前端**: **React** 框架构建用户界面，包含变更日志视图、关键要点视图和聊天界面。
*   **后端/数据库**: **SQLite** 本地数据库用于数据缓存。
*   **核心逻辑**: **Claude Code** 本身被用于根据提示词编写和迭代此应用程序。

### 四、完整构建步骤
1.  **前期准备与规划**:
    *   使用 **Perplexity AI** 的 Labs 功能进行快速研究，生成应用需求的 Markdown 格式规格说明文件（`changelog app.md`）。
    *   获取并准备好 **Gemini API** 的官方文档（以Markdown格式），特别关注如何使用 `Gemini 3 Flash` 模型、JSON模式以及文本转语音功能。
    *   确保文档中指定使用最新的 `Gemini 3 Flash`，并提醒模型其知识截止日期，防止它回退到旧版模型。

2.  **环境与API设置**:
    *   **Gemini API**: 在 Google AI Studio 获取 API Key。
    *   **Resend 服务**:
        *   访问 Resend 官网并登录。
        *   在账户设置中获取 API Key。
        *   配置发件邮箱和收件邮箱地址。
    *   设置本地开发环境，安装必要的依赖（如Node.js， SQLite等）。

3.  **核心开发与提示工程**:
    *   将准备好的 `changelog app.md` 规格说明和 `Gemini` 相关文档提供给 **Claude Code**。
    *   通过迭代提示，指导 Claude Code 构建应用。关键提示技巧：
        *   明确指定使用 `Gemini 3 Flash` 模型。
        *   要求分析结果缓存到本地 SQLite 数据库（“Cache AI analysis in my SQLite database by version hash, check cache before each API call.”）。
        *   对于UI，可提示其模仿 Anthropic 的设计风格，使其更美观。
        *   遇到具体问题时（如需要保存历史TLDR），可以开启新对话，让Claude帮助生成更精确的提示词。

4.  **功能集成与调试**:
    *   **抓取更新**: 编写代码定期轮询或监控 GitHub changelog 的链接。
    *   **AI解析**: 集成 Gemini API，对抓取的文本进行分析和摘要。
    *   **语音功能**: 调用 Gemini TTS API，将生成的文本转为语音，返回的 Base64 音频数据在前端转换为 WAV 文件播放，并在浏览器中缓存。
    *   **邮件通知**: 集成 Resend API，结合 Cron Job 定时器触发邮件发送。
    *   **数据库**: 实现 SQLite 的读写逻辑，存储版本哈希、AI分析结果等。

5.  **优化与收尾**:
    *   移除不必要的功能（如更新的情绪分析，因为几乎总是正面的）。
    *   确保系统可以回溯查看历史版本的TLDR。
    *   测试所有功能，包括手动播放语音、邮件接收和聊天准确性。

### 五、总结与资源
*   **系统价值**: 该系统将信息过载转化为可操作的洞察，通过自动化、AI分析和多模态交互（文字、语音、邮件），极大地提升了跟踪和使用 Claude Code 的效率，是个人开发者提升效能的优秀范例。
*   **可用资源**:
    1.  **提示工程手册与规格文档**: 视频描述中的第二个链接，包含用于构建此应用的详细Markdown文件。
    2.  **完整代码仓库**: 视频描述中的第一个链接，提供一键克隆的完整项目代码。
*   **核心要点总结**: 面对快速迭代的AI工具（如Claude Code），手动跟踪更新低效且困难。本视频展示了一套可行的解决方案：利用Claude Code自身作为开发工具，结合Gemini的文本分析和语音能力、Resend的邮件服务以及SQLite进行本地缓存，构建一个能够自动抓取、智能解析、语音播报并定时邮件通知的全自动化追踪系统，从而让用户始终掌握最新动态，聚焦于对自己最有价值的功能。
