# WEB-REVERSE v2：逆向效率翻倍！使用说明&案例展示~

> [治廷君](https://space.bilibili.com/309182597) | [BV1Xo9eBDEK1](https://www.bilibili.com/video/BV1Xo9eBDEK1) | 时长 16:20

## 一、背景介绍
- UP主（治廷君）在忙于前端项目的优化迭代，并测试了国产大模型。最终选择GLM5.1作为主力模型，因为它在一般场景下能满足需求，且比GPT系列更稳定可控。
- WEB-REVERSE v2是前端逆向工具的新版本，旨在通过优化提升逆向效率，核心是以验收为目标，并新增了搜索功能。

## 二、WEB-REVERSE v2 优化概述
- 版本更新聚焦于以验收为唯一目标，所有规则围绕任务交付。
- 设置了门禁机制，主要解决两个问题：
  - 执行偏差：任务偏离目标。
  - 虚假中断：执行中频繁中断或上报虚假结果。
- 其他优化包括增强专题模块（VM、WebAssembly、JSYMP）和新增搜索机制。

## 三、主要改进详情
### 3.1 底层逻辑转向
- 从旧版的专题类型逻辑，转向以验收为唯一目标，确保任务交付质量。

### 3.2 门禁机制
- 通过设置检查点，监控任务执行过程，减少目标偏移和中断问题。

### 3.3 专题增强
- 对VM、WebAssembly、JSYMP等逆向常见模块进行了大幅加强，提升处理复杂场景的能力。

### 3.4 新增搜索机制
- **目的**：辅助AI主动解决问题，通过互联网搜索获取相关信息，提高效率。
- **优势**：整合搜索结果作为参考，指引逆向流程，即使参考过时算法也能节省时间；测试显示效率显著提升，尤其对于XBOS、X-S等常见逆向任务。
- **实现**：使用MCP（Model Context Protocol）工具进行搜索，搜索结果为结构化数据，比谷歌搜索更全面。

## 四、搜索机制实操步骤
### 4.1 MCP工具准备
- 工具名：SpinSearchGetHub（字幕中为sbinsearchgethub，推测为SpinSearch或类似项目）。
- 需要安装此MCP工具，配置文件将随资源一起提供。

### 4.2 配置与运行步骤
1. **切换工具**：使用CC switch切换到MCP模式。
2. **添加配置**：在MCP中添加配置内容（具体配置行已提供脚本化简化）。
3. **启动服务**：运行MCP服务，在本地开启接口。
4. **使用搜索**：在AI工具中调用WebGangSearch（搜索功能名称）进行互联网搜索。
- 注意：必须先运行MCP服务才能使用搜索功能。

## 五、案例展示
### 5.1 案例一：某视频网站解密
- **难点**：涉及JSYMP和WebAssembly加密。
- **任务**：解密请求数据和响应中的加密内容。
- **过程**：通过WEB-REVERSE v2运行实例，成功解密数据，获得M3U8链接（注意：链接需额外参数才能下载，非本任务目标）。
- **结果**：视频正常播放，避免花屏问题。

### 5.2 案例二：某音乐网站榜单获取
- **任务**：获取音乐榜单信息。
- **过程**：使用AI工具快速解决，测试显示能正常获取数据。
- **效率**：对比旧版本，速度更快。

### 5.3 案例三：前端加密网站逆向
- **背景**：以前群友提供的网站，手动逆向需几小时。
- **过程**：使用WEB-REVERSE v2和AI，不到半小时全部解决。
- **报告详细**：包括算法确认、密钥长度、IV、填充模式等，以及本地测试加密/解密示例。

### 5.4 案例四：小红书API调用测试
- **任务**：X-X签名逆向分析。
- **过程**：使用搜索机制，触发搜索获取防御策略信息。
- **结果**：正常调用API，验证工具有效性。

## 六、效率提升总结
