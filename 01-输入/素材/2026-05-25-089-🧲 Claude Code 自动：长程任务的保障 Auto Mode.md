## Claude Code Auto Mode：保障长程任务的自动编辑模式

### 一、 模式概述与核心价值
*   **模式名称**: Auto Mode (自动模式)
*   **核心作用**: 实现真正意义上的**连续数小时不间断**的编程工作，无需频繁处理权限申请中断。
*   **解决的问题**:
    *   避免被各种权限提示打断工作流。
    *   相较于不安全的“YOLO模式”(暴力绕过权限)，提供了**内置的安全检查**，是更安全的长任务保障方案。

### 二、 使用条件与兼容性
*   **软件版本**: 需要 **Claude Code 2.1.83 或更高版本**。当前最新版本已稳定支持。
*   **套餐与连接方式**:
    *   **支持**: PRO套餐**以外**的所有套餐，以及**标准的Anthropic API连接方式**。
    *   **不支持**: PRO套餐；通过第三方非标准接口的连接。
*   **模型要求**: 需要**SOTA（当时最强）模型**，如 **Sonnet 4.6、Opus 4.6 或 4.7**。较老或较弱的模型（如Hiku等）不支持。

### 三、 如何开启与设置
#### 1. 临时切换模式（当前会话）
*   在Claude Code界面底部状态栏，通过快捷键 **`Shift + Tab`** 循环切换模式。
*   模式顺序：`Default`（默认询问）-> `Accept Edit`（接受编辑）-> `Plan`（计划模式） -> `Auto`（自动模式）。
*   切换至`Auto`后，底部会显示“Auto Mode”标识。

#### 2. 设置为默认模式（所有新会话）
*   **操作命令**: 输入 `/config` 进入配置界面。
*   **找到设置项**: `default permission mode`（默认权限模式）。
*   **选择**: 使用空格键选中 `auto`。
*   **推荐设置**: 同时确保 **`Use Auto Mode During Plan`**（在计划模式中使用自动模式）已开启，因为计划模式（Sub-Agent）本身也会执行大量操作。
*   确认后，未来所有新会话将默认以Auto Mode启动。

### 四、 实际使用与工作机制
#### 1. 工作流程
*   开启Auto Mode后，绝大多数操作**无需用户确认**，AI会自动执行。
*   **安全判断**: 操作由一个内置的 **`auto mode classifier`（自动模式分类器）** 进行安全评估。
    *   安全操作：直接执行，日志显示 `allowed by auto mode classifier`。
    *   危险操作（如删除文件）：会被拦截（`denied`），**分类器会主动中断任务**并询问用户下一步指示。
*   **被拦截后的处理**:
    *   认可拦截：无需操作。
    *   强制执行：在下一条指令中明确输入 `允许该操作` 或 `approve`，分类器会根据上下文调整判断逻辑并继续执行。

#### 2. 与其他模式对比（官方文档说明）
*   **Default**: 仅读取操作不询问，其他操作均拦截询问。
*   **Accept Edit**: 对读取、编辑、常见文件系统命令不询问，其他操作拦截询问。
*   **Plan**: 仅读取操作不询问。
*   **Auto**: **所有操作均由后台安全检查**。默认不询问，仅当操作被安全规则拒绝时才会中断用户。
*   **YOLO (`Bypass Permissions`)**: 绕过绝大多数权限（受保护路径除外）。**不安全，不推荐**。

### 五、 高级功能与自定义
#### 1. 管理与查看规则
*   **专用命令**: `claude automode` (一个子命令)。
*   **常用子命令**:
    *   `claude automode defaults`: 查看所有**默认的内置安全规则**（哪些会被允许/拒绝）。
    *   `claude automode config`: 打印当前的Auto Mode相关设置。
    *   `claude automode critique`: 对当前规则集进行评审。
#### 2. 自定义规则
*   规则可以在**本地进行配置**，配置方法详见官方文档的“Configuring Auto Mode”页面。
*   规则配置涉及目录结构、字段格式和条目写法。
*   **核心机制**: Auto Mode基于**可自定义的规则**运行。本地规则与线上的分类器互动，共同决定操作是否被允许。

### 六、 重要注意事项与成本
*   **额外成本**: Auto Mode的分类器运行在独立的服务器配置上，**会产生额外的Token消耗和费用**。这不是免费功能。
*   **Sub-Agent成本**: 当Auto Mode与计划模式（Sub-Agent）同时使用时，分类器会在Sub-Agent的**启动前、运行时、完成时**三个阶段进行安全检查，**成本会显著增加**。
*   **中断的本质**: 当Auto Mode“询问”你时，并非简单的权限请求，而是任务因安全规则被**中断或终止**，需要你作为决策者介入。

### 总结
Claude Code的Auto Mode是一个革命性的安全自动化功能，它通过内置的智能分类器在后台评估操作安全性，使开发者能够免于频繁的权限中断，从而专注于长达数小时的复杂编程任务。虽然它会产生额外的计算成本，且偶尔会因安全策略中断工作流，但相比默认的繁琐确认和高风险的YOLO模式，它在效率与安全性之间取得了最佳平衡。用户可以通过简单的快捷键或配置将其设为默认模式，并可以通过命令行查看和自定义安全规则，是提升AI辅助编程连续性和深度的强大工具。
