# PM必备神器🔥一键把网页变成可编辑设计稿

> [AI产品经理养成记](https://space.bilibili.com/47333744) | [BV1gbFezaE3g](https://www.bilibili.com/video/BV1gbFezaE3g) | 时长 7:09

## 一、工具概览
本次分享介绍了两款可将网页页面（HTML）快速转换为可编辑设计稿的浏览器插件，主要面向产品经理和设计师，旨在提升工作效率。

1.  **HTML to Design**
    *   **性质**：收费工具。
    *   **特点**：转换还原度高，组件（如Container, Header）会有逻辑合并，生成的文件在Figma中可能包含动态面板，便于整体查看但局部编辑稍显复杂。
    *   **成本**：提供免费额度（例如，每月或一定次数内免费）。

2.  **DragaFato**
    *   **性质**：免费工具。
    *   **特点**：所有组件均以平铺方式呈现，无逻辑合并。生成的设计稿在Figma中便于进行复杂编排和二次编辑。
    *   **成本**：完全免费，可持续使用。

## 二、以复杂表单页面为例的实操演示
### **步骤一：使用浏览器插件捕获并导出网页**
1.  在浏览器中打开目标网页（视频中使用了一个复杂表单页面）。
2.  点击浏览器工具栏中的插件图标（如“HTML to Design”或“DragaFato”）。
3.  选择“Capture Current Page”或类似选项，插件将自动获取页面所有信息。
4.  完成后，插件会提示保存一个文件（通常是 `.html` 或特定格式文件），下载保存到本地。

### **步骤二：在Figma中导入并编辑设计稿**
1.  打开Figma软件。
2.  在Figma中打开对应的插件（如“html to design”）。
3.  选择“File”选项，上传上一步中下载的文件。
4.  上传成功后，页面设计稿将呈现在Figma画布上，可直接进行编辑。
5.  **预览**：此时可对比原始网页与转换后的设计稿，观察还原度。

### **步骤三：在Figma中复制片段到其他设计工具（如Axure RP）**
*   **使用HTML to Design插件**：
    1.  在Figma中选中需要复制的页面区域。
    2.  在插件中点击“UX”或“Copy Section for RP”。
    3.  等待处理完成后（显示“Succeed”），在目标工具（如Axure RP）中执行粘贴。
    4.  **注意**：粘贴到Axure RP后，内容会包含在动态面板中，虽然预览效果正常，但编辑时可能因带样式的动态面板而稍显麻烦。
*   **使用DragaFato插件**：
    1.  在Figma中选中平铺的组件区域。
    2.  在插件中点击“Action”并选择“Copy Selection”。
    3.  成功后在目标工具中粘贴。
    4.  **优势**：由于组件都是平铺的，粘贴后的内容结构清晰，更便于进行复杂的二次编辑和重新编排。

## 三、两个插件的效果对比与选择建议
| 特性 | HTML to Design | DragaFato |
| :--- | :--- | :--- |
| **费用** | 收费（有免费额度） | **免费** |
| **组件结构** | **逻辑合并**（如Container, Header合并） | **平铺展示**（所有组件独立平铺） |
| **还原度** | **高** | 较高 |
| **编辑便利性** | 编辑时可能受动态面板影响，稍复杂 | **编辑非常方便**，适合复杂编排 |
| **免费额度示例** | 例如：29天内剩余9次免费使用额度 | 无限制，完全免费 |

### **选择建议**：
*   如果追求**极高的页面还原度**，主要用于查看和演示，且不介意少量编辑限制，可以选择 **HTML to Design**。
*   如果需要在转换后的设计稿上进行**复杂的二次编辑、重新排版和组件调整**，追求编辑自由度，推荐使用免费的 **DragaFato**。

## 四、总结
本次分享对比了两款将网页转为可编辑设计稿的浏览器插件：收费但还原度高的“HTML to Design”与免费且编辑灵活的“DragaFato”。通过在Figma中的实际演示和对比，核心发现是两者的关键区别在于组件结构——前者合并组件以求高保真，后者平铺组件以利编辑。产品经理和设计师应根据自身对还原度与编辑自由度的优先需求，以及预算考虑，选择合适的工具，从而高效地将线上页面转化为可用于迭代和协作的设计源文件。
