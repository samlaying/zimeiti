# AI产品经理必学agent skills实操

> [大模型产品经理教程](https://space.bilibili.com/3690972108032407) | [BV1HdF6zTEXw](https://www.bilibili.com/video/BV1HdF6zTEXw) | 时长 13:25

## 引言
Agent Skills 是由 Anthropic 公司（Cloud 大模型的母公司）发布的一种智能体开发标准，旨在以合理且低成本的方法构建智能体的技能包。它让智能体能够像人一样挑选和使用技能，并且技能包可以快速迁移到不同平台，通用性极强。在发布后，多个 AI 编程工具（如 Codex、VS Code、OpenAI CodeX 等）迅速接入支持。与之前的 MCP 协议相比，Agent Skills 引起了更强烈的反响。

## 技能包结构
### 基本定义
- 每个技能对应一个独立的文件夹，必须包含一个名为 `SKILL.md` 的 Markdown 格式文件（注意 `SKILL` 必须大写，这是约定格式）。
- `SKILL.md` 文件用于说明技能的相关信息，主要分为 **元数据** 和 **指令** 两部分。
  - **元数据**：包括技能的名称和说明，用于描述技能是什么、有什么用以及什么时候用。
  - **指令**：详细说明怎么用这个技能，包括执行步骤、注意点、可参考示例等，本质上是一个 prompt。
- 外部资源：技能执行可能依赖外部资源（如本地脚本或参考文档），这些资源可存储在三个子文件夹中，但不是必须提供（如果仅调用在线 API，则不需要这些文件夹）。

### 示例：PDF 文件处理工具
- 元数据部分：提供技能名称和简要说明。
- 指令部分：包含详细的使用步骤、注意事项和参考示例。

## Agent Skills 的优势
### 与现有技术的对比
- 传统方法（如 function calling 或 MCP）：通过工具定义将接口接入智能体系统，但需要打包所有工具的描述信息，导致大模型推理时上下文过长，增加成本和复杂度。
- Agent Skills 的优势：
  1. **渐进式披露机制**：将技能描述信息分层（元数据、指令、资源）。大模型决策时只提供元数据；决定调用技能时才传递完整指令和资源。这类似于人类浏览论文摘要再决定阅读正文，减少上下文长度，降低推理成本和复杂度。
  2. **更懂业务逻辑**：技能包可以包含详细的业务流程、标准和示例（如分析报告的结构要求、图表使用规范等），通过 prompt 注入，让智能体理解业务需求，避免通用逻辑导致的偏差。
  3. **运行更稳定**：技能提供详细标准，引导生成更可靠、一致的结果，减少反复修改 prompt 的尝试。

### 本质改变
- 从“工具”（Tor）升级为“技能”（Skill）：工具只规定功能，技能允许按业务标准组合功能调用逻辑，提高智能体的业务适应性和结果一致性。

## 场景化案例：智能合规系统
### 场景描述
- 公司发现供应商涉及欧盟 GDPR（数据保护条例）新规变动，需要立即审查合同。

### 对比三种方式
1. **原始 prompt 工程**：在系统提示词中塞入大量法律条文和公司规章，导致大模型容易迷失在海量信息中，给出模棱两可的建议。
2. **MCP 或 function calling**：大模型能通过接口查找最新法条，但只是“搬运工”，不理解法条背后的业务逻辑（如条款冲突时的优先策略），仍需人类律师介入。
3. **Agent Skills**：智能体识别关键词（如“欧盟”“供应商”“数据”）后，动态加载专门的 GDPR 合规技能包。技能包注入专家逻辑和执行标准（如 SOP 流程），让大模型像资深律师一样自动对比新旧版本差异，指出风险点并提出修改建议。

## 如何快速构建 Agent Skills
### 准备 Agent 引擎
- 推荐使用 **Claude Code**（Anthropic 公司开发，对自家协议适配度高）或 **OpenAI CodeX**（支持处于实验阶段）。
- 建议选择 Claude Code 以获得最佳兼容性。

### 实操步骤（完整流程）
1. **新建文件夹**：在电脑上创建一个项目根目录文件夹。
2. **创建 `.claude` 文件夹**：在项目根目录下新建一个名为 `.claude` 的文件夹。
3. **创建 `skills` 文件夹**：在 `.claude` 文件夹下创建一个用于存储所有技能包的 `skills` 文件夹。
4. **创建特定技能文件夹**：在 `skills` 文件夹下创建新技能文件夹，例如命名为“英文文本转大写”（文件夹名称可用中文）。
5. **创建 `SKILL.md` 文件**：在技能文件夹下新建 `SKILL.md` 文件，写入元数据和指令。
   - **元数据**：使用英文命名技能（如“English Text to Uppercase”），区别于文件夹的中文名称。
   - **指令**：详细说明技能用途、操作流程、注意事项，并给出示例。
6. **创建外部资源子目录**：如果需要，创建子目录存储脚本文件（如 Python 脚本）。例如，创建一个 Python 脚本用于将输入文本转为大写。
7. **启动智能体项目**：
   - 在命令行中定位到项目根目录。
   - 输入命令 `claude` 启动 Claude Code。
   - 输入命令 `--skills` 浏览可用技能列表（智能体通过检索 `SKILL.md` 获取技能，与文件夹命名无关）。
   - 在命令行中输入文本（如“请将文本‘hello’转大写”）引导智能体调用技能。
8. **执行验证**：大模型会加载技能包中的 Python 脚本执行操作，将结果输出到屏幕。

### 示例目录结构
```
项目根目录/
├── .claude/
│   └── skills/
│       └── 英文文本转大写/
│           ├── SKILL.md
│           └── scripts/  # 子目录存放外部资源
│               └── uppercase.py
```

## 总结
核心要点：Agent Skills 是一种智能体开发标准，通过结构化的技能包（包含元数据和指令）和渐进式披露机制，降低了大模型推理成本，增强了智能体对业务逻辑的理解，使运行更稳定。与传统工具定义相比，它更注重业务流程的注入，让智能体像专家一样执行复杂任务。实操中，使用 Claude Code 等引擎，按步骤创建技能文件夹和 `SKILL.md` 文件，即可快速构建可复用的技能包，提升 AI 系统的效能和可靠性。
