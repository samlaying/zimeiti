# OpenCode+摩尔线程国产算力本AIBOOK，开发AI APP有多丝滑？

> [技术爬爬虾](https://space.bilibili.com/316183842) | [BV195zeBEEjN](https://www.bilibili.com/video/BV195zeBEEjN) | 时长 10:22

## 一、 项目与硬件总览
- **项目名称**：开发一款名为“文体鸟”的手机APP。功能为录制鸟叫音频，通过AI分析识别鸟类品种并展示图文知识卡片。
- **核心开发工具链**：
    1.  **后端AI服务**：使用 `OpenCode` 开发Python FastAPI后端服务，调用本地离线AI模型。
    2.  **前端界面生成**：
        - 使用谷歌 `Stitch` 根据需求生成APP设计稿。
        - 将设计稿导入 `AAStudio`，使用 `Gemini 3` 模型生成前端网页代码。
    3.  **应用打包与调试**：
        - 使用 `OpenCode` 进行本地前后端联调。
        - 使用 `Capacitor` 将前端网页打包成安卓或iOS的APP。
- **核心开发硬件**：**摩尔线程 AIBOOK 算力本**。
    - **核心配置**：搭载国产自研长江SOC，提供 **50 TOPS** 的AI算力，拥有 **32GB** 统一内存。
    - **预装环境**：出厂预装完整的AI开发工具链（MOSA），包括Linux原生系统、Python、VS Code、Jupyter、PyTorch等，开箱即用，解决了CUDA版本冲突、依赖库混乱等环境配置难题。
    - **特色能力**：支持Windows与安卓虚拟机一键切换，可满足全平台开发需求。

## 二、 后端AI服务开发（使用OpenCode）
1.  **AI模型选择**：使用GitHub开源项目 `Bird Knight Analyzer`。这是一个基于500万参数卷积神经网络的鸟类声学识别模型，支持全球6000多种鸟类识别。
2.  **OpenCode安装与配置**：
    - AIBOOK已内置Node.js，直接通过终端使用 `npm install -g @anthropic-ai/opencode` 命令安装。
    - 配置AI模型：输入命令 `/connect`，选择OpenAI，使用GPT Plus登录方式完成认证。
3.  **后端开发流程**：
    - 向OpenCode提供提示词：要求使用Python FastAPI实现一个后端，接收音频上传，调用项目的CLI工具识别鸟类，返回JSON数据。
    - OpenCode生成代码后，按其建议执行命令安装依赖、启动服务。
    - 使用 `curl` 命令测试接口，发现初次返回数据不完整。通过查看生成的临时文件，发现有用数据在第二个CSV文件中。
    - 再次与AI沟通，要求其解析第二个CSV文件，成功获取正确的鸟类识别结果（含俗名、学名、置信度）。

## 三、 前端APP开发（Stitch + AAStudio + OpenCode）
1.  **设计阶段（使用谷歌Stitch）**：
    - 在Stitch中输入需求：设计“文体鸟”APP，包含录音功能和AI识别结果展示，要求美观、大气、清新。选择 `Gemini 3 Pro` 模型。
    - 生成初版设计稿后，通过多轮对话迭代，精简功能，优化设计至满意。
2.  **前端代码生成（使用AAStudio）**：
    - 将Stitch的设计稿导出到AAStudio。
    - 在AAStudio中详细描述后端接口格式、鸟类数据展示需求，要求其生成一个包含示例数据集的可运行前端网页代码。
    - 生成代码后，可切换至Mobile模式预览手机显示效果。
3.  **本地调试与优化（使用OpenCode）**：
    - 将AAStudio生成的代码下载到本地，使用VS Code打开并安装依赖（`npm install`）、启动项目（`npm run dev`）。
    - **发现并解决问题**：
        - **问题1**：录音按钮实为上传文件。使用OpenCode指令AI修改，使其能调用浏览器麦克风进行真正的录音。
        - **问题2**：跨域错误（CORS）。使用OpenCode指令AI在后端添加CORS中间件代码，解决问题。
        - **问题3**：前端显示优化（隐藏多余元素、添加鸟类描述文字）。再次使用OpenCode指令AI完成修改。
    - **联调配置**：使用 `ifconfig` 命令获取AIBOOK的局域网IP，将前端代码中的后端地址改为该IP，以便手机访问。

## 四、 应用打包与最终测试
1.  **打包环境准备**：
    - **目标**：将网页打包成安卓APK，使用 `Capacitor`。
    - **环境**：在AIBOOK上一键切换至其内置的 **Windows 11虚拟机**。
    - 在Windows虚拟机中安装 `Node.js` 和 `Android Studio`。
2.  **打包步骤**：
    - 将前端代码同步至Windows虚拟机。
    - 在项目目录中依次执行Capacitor的一系列初始化命令（`npm init`， `npx cap init`， `npm install @capacitor/core` 等，共七条）。
    - 项目会自动在Android Studio中打开，等待依赖下载完成。
    - **Android工程关键修改**：
        - 配置 `cleartext` 以允许HTTP连接。
        - 在清单文件中声明录音权限。
        - 允许应用与HTTP后台连接。
    - 在Android Studio中通过顶部菜单 `Build` -> `Generate Signed Bundle / APK` 打包APK。
3.  **最终测试**：
    - 将生成的APK文件传输到手机并安装。
    - 在手机端成功测试：录音 -> 上传至AIBOOK本地服务 -> 接收并显示鸟类识别结果。

## 总结
本次视频通过一个完整的“文体鸟”AI应用开发案例，展示了 **“摩尔线程AIBOOK算力本”** 与 **“OpenCode + Stitch + AAStudio + Capacitor”** 这套全新开发工具链的高效协作。AIBOOK凭借其强大的**国产算力**、**预装的MOSA环境**及**多系统支持**，为本地AI推理和跨平台开发提供了稳定且免配置的基础。而工具链中，OpenCode负责后端逻辑与联调，Stitch与AAStudio利用AI（Gemini 3）实现了从UI设计到前端代码的快速生成，Capacitor则简化了跨平台打包流程。整套方案**大幅降低了AI应用的开发门槛**，提升了从创意到成品落地的效率，展现了国产硬件与现代化AI开发工具结合的优越性。
