# skills多了应该怎么做？

> [Token杰-自然智群](https://space.bilibili.com/431598453) | [BV1yTQrBnEoj](https://www.bilibili.com/video/BV1yTQrBnEoj) | 时长 6:15

## 引言
- **问题背景**：Skills（技能）是解决工具提示词爆炸问题的有效工具，但Skills数量过多会导致性能下降和触发概率降低。
- **关键数据**：基于LongChain研究报告，Skills数量应最多不超过12个，推荐最佳范围为2-4个；超过限制会极大影响模型效率和加载效果。
- **核心议题**：本笔记分析如何设计一个Skills架构来管理多个脚本和样式内容，以UI UX Pro Max Skills（一个专门用于前端设计的Skills）为例。

## Skills架构设计分析
### 整体架构设计
- **设计目标**：允许在一个Skills中集成大量样式、脚本和参考内容，而不消耗过多上下文，确保AI模型高效运行。
- **分层结构**：
  - 入口文档（.md文件）：定义触发规则和核心原则。
  - 数据存储（CSV文件）：外部化存储细节数据如样式、配色。
  - 搜索脚本：使用算法动态检索和匹配信息。

### 详细组件说明
#### 入口文档（.md文件）
- **功能**：作为Skills的入口，定义关键词触发、规则、原则和工作流程。
- **内容规模**：约300-400行，包括：
  - 触发关键词列表（如“前端设计”、“UI样式”等）。
  - 设计原则和指南（如按钮、元素的设计规范）。
  - 经验教训和检查清单（如“避免做什么”、“交付前检查项”）。
- **示例**：UI UX Pro Max Skills的.md文档中详细描述了前端设计的搜索指导和原则。

#### 数据存储（CSV文件）
- **作用**：将大量细节数据（如样式、配色、框架兼容性）外部化存储，避免塞入单一上下文。
- **内容示例**：
  - Steel的CSV文件：列明样式列表、支持深色模式、框架兼容性（如React、Vue）。
  - 其他CSV文件：包含行业参考样式、决策规则等，便于搜索匹配。
- **优势**：允许扩展性强，可随时添加新数据而不影响Skills核心结构。

#### 搜索脚本
- **核心算法**：使用BM25文本匹配算法，基于关键词从CSV文件中检索高相关性信息。
- **脚本功能**：
  - 搜索生成器：参数化搜索多个关键词，返回组合文档。
  - 核心搜索引擎：打包为函数，便于调用。
  - 流程：搜索脚本执行匹配 → 生成器组合结果 → 返回完整设计文档给AI。
- **实操步骤**：
  1. 调用搜索脚本，输入需求关键词（如“按钮样式”）。
  2. 脚本从CSV文件中提取匹配行（如样式名称、颜色代码）。
  3. 生成器将搜索结果组合成结构化文档，包括参考样式和细节数据。
  4. AI基于该文档生成前端代码或设计建议。

### 工作流程
1. **分析阶段**：理解用户输入和需求，确定设计方向。
2. **搜索设计系统**：首先调用搜索脚本，从CSV文件中获取核心设计系统信息（如整体样式框架）。
3. **补充细节搜索**：进一步细化搜索，针对具体元素（如按钮、配色）获取详细数据。
4. **技术质量选行**：评估搜索结果的技术可行性和质量，确保兼容性和最佳实践。
5. **输出生成**：组合所有搜索结果，生成最终设计文档，包含架构、文件格式和必含组件（如避免缺失关键部分）。

## 核心价值与建议
- **价值**：这种架构设计解决了Skills扩展性难题，允许集成大量内容而不下降性能，特别适用于脚本多、需要持续增加的项目。
- **实用建议**：
  - 避免将所有参考信息直接写入Skills的.md文件或单个上下文，这会消耗过多资源。
  - 采用外部数据存储（如CSV文件）和动态搜索机制，减少AI上下文负担。
  - 参考UI UX Pro Max Skills的设计：将样式和脚本数据化，通过算法按需检索。
- **注意事项**：Skills数量应控制在2-4个以内；如果内容复杂，优先使用搜索驱动而非静态加载。

## 总结
- 核心要点：管理多个Skills时，关键在于采用搜索驱动的架构设计，将数据外部化并通过算法（如BM25）动态检索，以优化性能和扩展性。参考UI UX Pro Max Skills的实践，避免直接塞入大量内容，而是通过CSV文件存储细节、脚本执行搜索，确保AI模型高效处理复杂前端设计任务。
