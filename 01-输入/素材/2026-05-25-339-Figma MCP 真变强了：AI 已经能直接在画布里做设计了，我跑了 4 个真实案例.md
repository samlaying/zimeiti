# Figma MCP 真变强了：AI 已经能直接在画布里做设计了，我跑了 4 个真实案例

> [kate人不错](https://space.bilibili.com/346575605) | [BV1jBQSBwEjZ](https://www.bilibili.com/video/BV1jBQSBwEjZ) | 时长 15:39

## Figma MCP 核心升级介绍
Figma MCP 迎来重大升级，新增 **Use Figma Tool**。这使得 AI Agent 能够直接在 Figma 画布上进行设计和修改，而不仅仅是查看或导入。AI 可以基于你的设计系统，通过如 Claude、Codex 等 MCP 客户端来创建和编辑设计资源。

### 与旧工具的区别
- **旧工具（Generate Figma Design）**：主要将网页的 HTML 代码转化为可编辑的 Figma 图层。
- **新工具（Use Figma）**：允许 AI 直接在 Figma Canvas 上操作，实现从“导入”到“创造”的跨越。

### 新增能力与资源
1.  **Skills（技能）**：Figma 与合作伙伴分享了 9 个技能。当使用支持视觉的 AI Agent 时，它会通过截图和迭代的**自我修复循环**来不断优化输出，修正不匹配的地方。
2.  **开放访问**：现在可以访问 **CodeConnector**、**Figma Draw** 和 **FigJam** 的界面。
3.  **调用限制**：Figma 官方 MCP 对免费用户有限制，**每月最多提供 6 次工具调用**。若想充分利用 AI 在画布上做设计，需要订阅 **Dev** 或 **Full** 席位。

## Figma MCP 的安装与配置
### 在不同客户端中安装
1.  **在 Kursor 中安装**：
    - 进入设置中的 **Marketplace**。
    - 搜索 **Figma** 并点击安装。
    - 安装后需要验证 Figma 身份。
2.  **在 Claude Code (CC) 中安装**：
    - 通过命令行安装官方的 Figma MCP。
    - 安装后需验证身份。
3.  **在 Codex 中安装**：
    - 先添加各个 **Skills**。
    - 然后在设置中添加 **MCP Server**。
    - 找到 Figma 服务器，点击安装并认证。

## Figma MCP 工具详解
以下是 Figma MCP 提供的核心工具：
1.  **Get Design Context**：获取当前在 Figma 中选中的设计上下文，默认输出为 React 和 Tavinter 代码。
2.  **Generate Figma Design**：将捕获的网页导入或转化为 Figma 设计。
3.  **Get Variable Definitions**：返回设计中使用的设计变量（如颜色）。
4.  **Get Code Connect Map**：检索 Figma 节点 ID 与代码库中相应代码组件之间的映射关系。
5.  **Use Figma**：**核心工具**，让 AI 直接在 Figma Canvas 上进行设计操作。
6.  **Search Design Assistant**：搜索所有链接的组件库。
7.  **Create a new file**：提示 AI 在 Figma 里新建一个文件，无需预先提供链接。

### 最佳实践建议
Figma 官方推荐将 **Figma Make** 与 **Figma MCP** 结合使用，以获得更佳效果。

## 四个真实案例详解
### 案例一：为可折叠手机应用生成设计
- **目标**：基于一份设计系统文档，生成可折叠手机应用的新 Figma 设计页面。
- **模型**：Claude Opus 4.6。
- **步骤**：
    1.  使用 **Plan 模式**，AI 生成了六阶段的执行计划。
    2.  执行时，AI 先检索 Figma 认证状态。
    3.  使用 **Create a new file** 工具创建新文件。
    4.  使用 **Use Figma** 工具在画布上开始设计。
    5.  依次创建设计 Token、颜色变量集、检查字体。
    6.  创建第一个屏幕（主页），并截图验证。
    7.  继续创建第二、三、四个屏幕（如书架页、阅读页）。
- **结果**：一次性生成了四个遵循设计系统的页面，包括内屏、外屏折叠态等视图。设计淡雅，配色和排版获得好评。
- **灵感来源**：该设计参考了由 Google Stitch 生成的设计语言 Markdown。

### 案例二：用不同模型设计简单页面
- **目标**：探索使用不同模型（Cursor 的 Comparser 2）通过 Figma MCP 设计页面的效果。
- **结果**：生成了两个相对简单的页面。色彩与提供的设计语言 Markdown 文件相似，但折叠态的页面设计不够理想，图标过小，内容展示较少。**教训**：未使用计划模式并限制内容展示，可能导致输出过于简单。

### 案例三：从品牌网站到完整落地页的生成
- **目标**：基于多米诺骨牌风格网站，创建一个“小鸟嗷桐理发店”的 Landing Page，最终生成可运行的代码。
- **模型**：GPT-5.4 (Medium 推理模式)。
- **关键步骤**：
    1.  **提取设计系统**：提示 AI 基于参考网站，在 Figma 中创建完整的设计系统、主件库，并拉取真实的 logo、字体和颜色。
    2.  **生成高保真设计**：AI 生成单页 Figma 设计，并根据反馈进行优化（如手动调整不协调的橘色卡片）。
    3.  **生成可运行代码**：将设计链接提供给新的 Agent 页面，AI 调用 `get design context` 并编写 React 代码。
    4.  **优化与定稿**：针对代码中使用的临时远程图片资源，AI 调用图片生成工具（如 Google 的 Nano Blalda）生成配图，并替换到页面中，最后添加动效。
- **结果**：最终生成了一个完整、可运行、风格鲜明的理发店网页。

### 案例四：为现有项目补充组件库与设计系统
- **目标**：基于一个已有的“Voices Studio”项目代码，通过 Figma MCP 补充和创建对应的 Figma 设计组件。
- **步骤**：
    1.  让 AI 分析现有代码文件，建议先创建哪些组件。
    2.  提示 AI 生成一个“Voices Studio 组件页”到 Figma Canvas（AI 会自动创建新页面）。
    3.  AI 生成组件库页面后，截图验证并自动修正了高度问题。
    4.  进一步提示 AI 将视觉组件升级为规范的设计系统，补全完整的 Web 端界面。
- **结果**：AI 创建了包含**样式集（颜色、字体、间距）**、**变量集合**和**组件库**的完整设计系统，并生成了五个不同端（Web、桌面、手机）的详细页面设计。这极大地提升了后续开发和设计的一致性。

### 案例五：修改现有设计风格
- **目标**：将 Figma 中已有的一个设计页面，修改为“瑞士设计风格”。
- **步骤与挑战**：
    1.  获取设计 URL 并提示修改。
    2.  AI 读取页面后，触发了免费计划的调用上限。
    3.  **解决方案**：升级订阅计划。
    4.  AI 开始执行设计修改，并针对空白区域等问题进行迭代调整（可通过截图反馈）。
- **结果**：虽然手动微调可能更高效，但 AI 利用 Use Figma 工具的修改速度很快，对于不精通 Figma 的用户而言，降低了设计门槛。

## 总结
Figma MCP 的这次升级通过 **Use Figma Tool** 实现了质的飞跃，使 AI 能够像设计师一样直接在 Figma 画布上进行**创建、修改和系统化设计**。从基于文档生成移动应用 UI，到从品牌网站衍生出完整落地页并生成代码，再到为现有项目补全设计系统，这四个案例全面展示了该工具在**提效、降低设计门槛和维护设计-开发一致性**方面的强大潜力。尽管存在免费调用次数限制和可能需要人工微调的问题，但它无疑为设计师和开发者提供了一个革命性的 AI 协作工具。
