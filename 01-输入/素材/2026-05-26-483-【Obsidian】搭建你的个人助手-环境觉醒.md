## Obsidian个人助手搭建：环境觉醒教程

### 1. 任务概述与背景
- **主要任务**：搭建个人助手的环境，掌握SEOE模型，学习JSON、STEAM和数据契约，并安装数据接口。本视频重点介绍“环境觉醒”部分，即设置Obsidian基础环境并接入AI模型。
- **核心思路**：Obsidian作为本地数据库，通过AI接口（如硅基流动）将数据发送到大型语言模型（如DeepSeek）进行处理，类似Notebook LLM的思路。目标是构建个人智能体（MVA），实现从传统模式向拥有自己电子大脑的模式转换。
- **关键概念**：
  - **Vault（仓库）**：在Obsidian中用于存储笔记和数据的容器，类似于文件夹或项目空间。
  - **Text Generator插件**：一个辅助AI写作的插件，可用于管理提示词模板和生成文本内容。
  - **Local First**：强调数据本地存储，保护隐私和控制权。
  - **MVA（为自己构建智能体）**：个人化的AI助手，专注于用户自身需求。

### 2. 环境准备与Obsidian设置
#### 2.1 下载与更新Obsidian
- **步骤**：
  1. 访问Obsidian官网下载最新版本并安装。
  2. 如果已安装，检查更新：打开Obsidian，进入设置，确保使用最新版本（视频中更新到最新版本以优化体验）。
- **建议**：在开始搭建前，先更新软件以避免兼容性问题。

#### 2.2 创建新仓库（Vault）
- **步骤**：
  1. 打开Obsidian，点击“Create new vault”（创建新仓库）。
  2. 设置仓库名称为“AI Agent Space”（示例名称，可根据需要自定义）。
  3. 选择仓库位置：建议将仓库移动到磁盘空间较大的分区（如从默认C盘移动到D盘），以避免未来空间不足。
  4. 完成创建后，仓库会出现在Obsidian主界面。
- **注意**：Obsidian没有快速切换仓库的内置功能，需通过“File”菜单中的“Open another vault”来切换。建议在搭建初期就规划好仓库位置，方便后续管理。

#### 2.3 仓库管理与位置设置
- **移动仓库步骤**：
  1. 关闭Obsidian中当前打开的仓库。
  2. 手动将仓库文件夹移动到目标磁盘（如D盘）。
  3. 重新在Obsidian中打开移动后的仓库。
- **建议**：在创建仓库前就决定位置，或尽早移动，以避免数据丢失或路径错误。

#### 2.4 配置默认附件位置
- **步骤**：
  1. 进入Obsidian设置（Settings）。
  2. 选择“Files & Links”（文件与链接）。
  3. 找到“Default location for new attachments”（新附件的默认位置）。
  4. 设置为“Current folder下的子文件夹”（例如，创建“attachments”子文件夹），以保持文件组织清晰。
- **数据**：这一步确保所有插入的图片、文件等附件都存储在统一位置，便于管理和备份。

### 3. 插件安装与配置
#### 3.1 安装Text Generator插件
- **步骤**：
  1. 在Obsidian设置中，进入“Community plugins”（社区插件）。
  2. 禁用“Restricted mode”（限制模式），以允许安装第三方插件。
  3. 点击“Browse”浏览插件市场，搜索“Text Generator”。
  4. 点击“Install”安装，然后启用插件。
- **信息**：Text Generator插件是一个开源工具，用于辅助AI写作和提示词模板管理，支持生成文本内容。

#### 3.2 配置AI模型API（以硅基流动为例）
- **步骤**：
  1. 打开Text Generator插件设置：在已安装插件列表中，点击齿轮图标进入配置。
  2. 在“LM Settings”中，选择模型类型为“Custom”（自定义），因为使用硅基流动等非官方API。
  3. 获取API信息：
     - 访问硅基流动官网（或类似平台），注册账号并登录。
     - 在控制台中，搜索模型（如“Deepseek AIR1003-8B”，这是一个免费模型，达到千万参数级别）。
     - 复制API端点（API endpoint）和API密钥（API key）。
  4. 将API端点粘贴到插件设置中的“API URL”字段。
  5. 将API密钥粘贴到“API Key”字段。
- **示例**：硅基流动提供免费模型，API端点通常为类似“https://api.example.com/v1”的格式，密钥为“sk-”开头的字符串。
- **建议**：确保API密钥安全，不要泄露；如果使用其他模型（如DeepSeek），需相应调整配置。

### 4. 测试与验证
- **步骤**：
  1. 在Obsidian中创建新笔记，命名为“00 Hello World”或类似名称。
  2. 在笔记中输入测试指令，如“你好”。
  3. 按下快捷键“Ctrl + G”（或根据插件设置）运行AI生成。
  4. 等待处理，观察输出。成功时会显示生成文本（如“环境觉醒成功”）。
- **调试**：如果报错，检查模型名称是否正确（如需修改为具体模型ID），或重新验证API配置。
- **结果**：成功测试后，表示环境觉醒完成，AI助手已初步搭建。

### 5. 关键概念解析
- **SEOE模型**：视频中提到但未详细展开，可能指一种AI或数据处理模型，后续课程会学习。
- **数据契约与接口**：涉及JSON和STEAM技术，用于数据交换和AI接口集成，将在后续内容中介绍。
- **MVA（为自己构建智能体）**：强调个性化AI助手，用户从被动使用转向主动构建，拥有数据控制权和定制化能力。

### 6. 总结
本视频详细介绍了在Obsidian中搭建个人助手环境的完整流程，包括创建和管理仓库、安装Text Generator插件、配置AI模型API（以硅基流动为例），以及测试验证。核心要点是：通过本地化设置（Vault和附件管理）结合AI插件，实现一个安全、可定制的智能助手基础，为后续学习SEOE模型和数据接口打下基础。环境觉醒成功标志着从传统笔记工具向个人AI智能体的转变，鼓励用户掌握技术以构建自己的“电子大脑”。
