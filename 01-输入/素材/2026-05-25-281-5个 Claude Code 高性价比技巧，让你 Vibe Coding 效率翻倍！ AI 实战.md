> [赫赫儿_](https://space.bilibili.com/630806242) | [BV1mdXWB2Et3](https://www.bilibili.com/video/BV1mdXWB2Et3) | 时长 5:36

## 5个Claude Code核心技巧，实现高效AI协同开发

视频分享了五个基于Claude Code最佳实践的高性价比技巧，旨在将无序的“Vibe Coding”转变为可控、高效的AI协同工作流。以下是对视频核心内容的结构化笔记。

### 技巧一：让AI采访你，生成需求文档
- **问题**：直接让AI写代码，结果往往与预期不符，陷入反复修改的循环。
- **解决方案**：利用Claude Code的`ask user question`工具，通过多轮问答深度挖掘需求。
- **操作步骤**：
    1. 向Claude Code发出指令，明确要求它使用此工具进行提问。例如：“我想开发一个家庭点菜网站，请使用ask user question工具对我进行深度访谈。”
    2. 根据AI提出的问题进行选择或补充回答。AI会主动引导并补全你未考虑到的需求点。
    3. 访谈结束后，AI会自动生成一份详尽的需求文档（如 `Size.md`）。
- **进阶**：如果需要修改方案（如更换数据库），可以直接在文档中手动修改，或让AI协助修改。

### 技巧二：将探索、规划与实现分开
- **核心理念**：像装修房子一样，遵循“勘测构思 -> 出图确认 -> 严格施工 -> 验收记录”的流程，确保逻辑清晰。
- **Claude Code的四阶段工作流**：
    1.  **探索**：明确要解决的问题。
    2.  **规划**：制定详细的技术方案。
    3.  **实现**：严格按照方案编写代码。
    4.  **提交**：整理并提交成果。
- **操作步骤**：
    1. 在需求文档生成后，**切换到`Plan mode`**（通过`Shift+Tab`）。在此模式下，AI不会写代码，仅专注于根据需求生成技术方案。
    2. 与AI讨论、调整方案，直到你完全满意。
    3. 满意后，**切回`Normal mode`**，让AI严格依据已确认的方案施工（写代码）。
- **注意**：对于微小问题，可让AI直接修复以节省Token。

### 技巧三：给AI提供充足的诊断信息
- **问题**：遇到bug时，只反馈“报错了”或“功能失效”信息不足。
- **解决方案**：主动收集并提供具体的错误上下文，帮助AI精准定位问题。
- **操作步骤**（以“无法添加菜品”为例）：
    1.  打开浏览器开发者工具，检查**控制台**是否有报错。
    2.  检查**网络面板**，查看相关API接口是否请求失败及其状态。
    3.  将**接口地址、具体错误信息或日志**复制给AI。
- **效果**：提供的线索越具体，AI定位和修复问题的速度就越快。

### 技巧四：为项目创建一份持久的“说明书”（Claude.md）
- **问题**：AI在长时间对话中容易“失忆”，忘记你设定的规则（如文件命名规范、代码风格）。
- **解决方案**：使用 `/init` 命令在项目根目录生成 `Claude.md` 文件。
- **文件作用**：
    - 作为一份持久的项目上下文，在每次开启新对话时，AI会优先读取。
    - 用于记录项目的**基础设定、约定和规范**，防止后续修改破坏原有结构。
- **建议在文件中写入**：
    - 非常规的文件命名约定。
    - 测试要求、代码风格指南。
    - 明确指定某些文件夹或文件不要动。
    - 使用 `important` 标记提醒AI注意特殊约定。

### 技巧五：主动管理对话上下文
- **问题**：随着对话变长，AI的上下文窗口会被塞满，导致其反应变慢、遗忘信息。
- **解决方案**：主动执行以下命令管理上下文：
    1.  **`/compact` 压缩上下文**：在一个阶段性任务（如基础功能跑通）完成后主动执行。可以在命令后跟注释，指引AI总结时聚焦的方向。
    2.  **`/clear` 清理上下文**：在一个完整的阶段（如写好需求文档）结束后，建议开启新会话或使用此命令，避免之前的冗余对话干扰AI后续的专注度。
    3.  **`/rewind` 回滚状态**：当发现一个bug连续修复两次仍未解决时，说明当前上下文已被错误代码污染。使用此命令可回滚到更早的干净状态，再带着经验教训开始新尝试。

## 总结
核心要点在于改变与AI协同的方式：**从被动应对变为主动引导和管理**。通过**深度访谈明确需求**、**分阶段规划实现**、**提供精准的诊断信息**、**建立持久的项目规则**以及**积极维护对话上下文**这五个技巧，可以将AI从可能带来混乱的助手，转变为一个理解准确、执行高效、值得信赖的“超级合伙人”。关键在于用清晰的规则和结构化的流程，来换取AI输出的确定性和可靠性。
