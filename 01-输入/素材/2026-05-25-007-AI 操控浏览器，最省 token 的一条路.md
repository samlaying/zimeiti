# AI 操控浏览器，最省 token 的一条路

> [Simon林_](https://space.bilibili.com/506233728) | [BV1px9XB9E8L](https://www.bilibili.com/video/BV1px9XB9E8L) | 时长 3:52

## 引言
视频介绍了名为 BrowserHannis 的仓库，由 BrowserUse 团队开发，专为 AI 操控浏览器设计。该仓库在 GitHub 上线 15 天内获得了 9400 星标，代码量仅为 592 行 Python 代码（迭代后约 700-800 行），强调其 Token 消耗低的优势，是目前最节省 Token 的 AI 浏览器操控方案。

## AI 操控浏览器的路径对比
目前 AI 操控浏览器有 4 条主要路径，各有特点和局限性：

### 路径1: CloudIn Chrome
- **方式**：通过浏览器扩展读取 DOM 操作页面。
- **Token 消耗**：中等。
- **问题**：经常出现连接不稳定的情况。

### 路径2: Compute Use
- **方式**：通过截屏查看像素，模拟鼠标键盘操作。
- **Token 消耗**：非常高（“爆炸”级别），不推荐使用。

### 路径3: Celenium
- **方式**：有几万行代码，传统工具，不适合 AI 专用。

### 路径4: BrowserHannis
- **方式**：通过 WebSocket 直接控制 Chrome 内核，代码量仅 600 行（迭代后约 700-800 行）。
- **Token 消耗**：非常低，原生 CDP（Chrome DevTools Protocol）实现。
- **定位**：为 AI Agent 量身定做的工具，逻辑不同于前三个路径（前三个像给人穿的鞋，第四个是给 AI 定制的）。

## BrowserHannis 详解

### 基本信息
- **仓库名称**：BrowserHannis，由 BrowserUse 团队开发。
- **代码量**：592 行 Python 代码（迭代后约 700-800 行），由 4 个简单文件组成。
- **GitHub 数据**：上线 15 天获得 9400 星标，显示市场高度认可。

### 工作原理
- **流程**：Guardcode 通过 CLI 发送指令到 Demon，Demon 再通过 CDP 的 WebSocket 直接控制 Chrome。
- **连接优势**：能连接用户正在使用的浏览器，复用现有的 cookie 和登录状态（例如登录抖音等状态完全可用）。
- **技术背景**：实现的关键在于 LLM（大语言模型）的进步，现在 LLM 能通过截图看坐标、判断状态，并自行编写 help 函数，因此只需让 LLM 连接 Chrome 即可。

### 优势特点
1. **自学习能力（“长肌肉”）**：
   - 有一个名为 `agent-health` 的文件，初始为空，Agent 可以在运行中自己编写代码来填充，持续积累工作经验。
   - 自带 76 个网站的经验（称为“A卷”），新 Agent 启动时即可读取这些经验，相当于带技能上线。
2. **正反馈循环**：使用越多，效果越好，A卷能通过不断使用沉淀经验，便于分享。
3. **Token 效率高**：对比其他方法，Token 消耗极低。

## 实操步骤
UP 主分享了通过 CC（可能指 BrowserHannis 的控制端）控制 Chrome，使用订阅版 GMI（可能指 Gemini）进行图片生成并下载的实测过程。具体步骤如下：

### 步骤1: 安装
- 安装 Git、Chrome 和 UV Tools（使用 `UV Tools Install` 命令）。

### 步骤2: 连接 Chrome
- 需要打开 Chrome 的远程调试功能，以便 BrowserHannis 连接。

### 步骤3: GMI 生图
- 通过 CC 操控浏览器，让 GMI 生成图片（例如使用 Gemini API 生成图像）。

### 步骤4: 封装技能
- 将操作封装成技能，方便复用。UP 主提到可以在其首页查看封装好的技能。

## Token 消耗对比
- **Computer Use（传统方法）**：大约需要 1.5 万个 Token。
- **BrowserHannis**：仅需约 2000 个 Token，显著降低 API 成本。
- **应用示例**：通过这种高效方式，实现让 Gemini 进行图片生成，节省资源。

## 总结
BrowserHannis 是一个为 AI 设计的浏览器操控工具，它通过 WebSocket 直接控制 Chrome 内核，代码简洁、Token 消耗低，并具备自学习能力。与传统路径相比，它能复用浏览器状态、积累经验，并形成正反馈循环，实测中展示了高效的图片生成操作。市场通过 9400 星标快速认可了这一创新，体现了 AI 工具在效率优化上的突破。
