## 如何把 Obsidian Web Clipper 价值最大化

> 来源：Next蔡蔡（Bilibili），2026-04-22

---

## 一、基础认知

- **Obsidian Web Clipper** 是 Obsidian 官方推出的 **开源浏览器插件** ，可一键将网页内容保存到 Obsidian 知识库。
- 支持保存 **文本网页** （如博客文章）和 **YouTube 视频字幕** 。
- 所有保存内容最终以 **Markdown 格式** 存储——Markdown 在 AI 时代是"硬通货"，AI 可无损理解，且跨平台可迁移、长期可存。

---

## 二、四大技巧：挖掘原版插件的 70%~80% 价值

### 技巧一：配置解释器（Interpreter）——给 Clipper 引入 AI 能力

**什么是解释器？**

- **普通变量** （如 `title` ）：插件直接从网页代码中提取对应字段。
- **解释器变量** ：插件将网页内容发给 AI 模型，AI 理解后将计算/思考的结果填回变量位置。

**核心能力：通过自然语言与网页内容交互。**

- 例如在笔记模板中添加一个解释器变量，写入提示词： `用简体中文总结和提炼这篇文章的核心内容` ，点击"解释"后，AI 会生成摘要并填入该位置。
- 用途：在剪存文章前先看 AI 总结，决定是否值得保存。

**其他应用场景：**

| 场景 | 说明 |
| --- | --- |
| 翻译文本 | 将外文内容翻译为中文 |
| 提取特定片段 | 从长文中抽取关键信息 |
| 格式转换 | 将内容转换为指定格式 |
| 任意自然语言任务 | 只要能描述清楚，就能实现 |

**配置步骤：**

1. 1.点击插件面板右上角 **设置图标** → 选择 **「解释器」** （默认关闭，需手动开启）。
2. 2.**添加供应商** ：从预置列表中选择，或自定义添加。
3. 3.无论哪种方式，都需前往对应 AI 服务商官网 **获取 API Key** 。
4. 4.**选择模型** ：推荐使用小模型（速度快、精度够），如 **Gemini 2.0 Flash** ，而非大模型。

> **模型选择建议：** 实测对比国内外多个模型后，Gemini 2.0 Flash 在速度和准确度上表现最佳。同一篇文章的处理速度明显快于 DeepSeek 等大模型。大模型会显著拉长处理时间。

---

### 技巧二：设置模板触发条件（Template Trigger）

**解决的问题：** 多个自定义模板（如视频模板、公众号模板）之间来回切换的麻烦。

**配置方法：**

1. 1.进入 **编辑模板** 页面，顶部找到 **「模板触发器」** 。
2. 2.为每个模板配置对应的 **URL 匹配规则** 。

**示例：**

| 模板 | 触发 URL 规则 |
| --- | --- |
| 视频模板 | `youtube.com/*` 、 `bilibili.com/*` |
| 公众号模板 | `mp.weixin.qq.com/*` |

这样当你在 YouTube 或 B 站页面打开插件时，会自动匹配对应模板，无需手动切换。

---

### 技巧三：高亮文本

- 阅读博客或视频文字稿时，看到受启发的内容，可直接在网页上 **划重点** 。
- 所有高亮文本会 **自动同步** 到 Obsidian 文档中，无需二次整理。
- 操作方式：选中文字 → 使用插件的高亮功能（不需跳出当前页面）。

---

### 技巧四：沉浸式翻译 + Obsidian Web Clipper

- 如果已使用 **沉浸式翻译** 做页面翻译， **无需在 Clipper 中单独配置翻译解释器** 。
- 直接在沉浸式翻译的页面中打开 Obsidian Web Clipper，插件可以 **同时抓取原文和翻译** 。

---

## 三、中文增强版：Obsidian Clipper CN

> 原版插件对中文内容支持不足，作者（Next蔡蔡）开发了中文增强版插件。

**基于官方插件，保留全部原版能力，额外支持两个平台：**

### 1\. B 站视频支持

体验与官方插件对 YouTube 的集成基本一致：

- 提取视频简介
- 提取章节字幕
- Reader Mode 中嵌入 B 站播放器，支持按时间戳自定义跳转
- 可搭配解释器一起使用（效果更佳）

### 2\. 飞书文档支持

**原版的问题：** 通过通用 DOM 解析提取飞书内容，因飞书的动态渲染机制导致内容不完整。

**CN 版方案：** 接入 **飞书开放平台 API** ，通过结构化接口完整获取文档内容。

> 当前不足：暂无法获取文档中的图片，作者后续会尝试解决。

**配置步骤：**

1. 1.前往 **飞书开放平台** ，创建一个 **自建应用** 。
2. 2.给应用开通以下权限（含文档读取相关权限 + wiki 权限）。
3. 3.**创建版本并发布** （不发布则权限不生效）。
4. 4.在「凭证与基础信息」页面获取 **App ID** 和 **App Secret** 。
5. 5.打开 Clipper CN 插件 → 右上角设置 → 常规页面底部找到 **飞书模块** → 填入 App ID 和 App Secret。

> **隐私说明：** App ID 和 App Secret 仅保存在本地浏览器存储中，不会上传到任何服务器。

## 四、为什么是独立插件而非合并到官方？

- 作者曾向官方项目提交过 PR，但未通过。
- 官方回复：Obsidian Web Clipper 只处理其另一个工具（内容处理服务端）处理后的内容。
- 作者尝试在官方的服务端项目提 PR，但因无法在本地验证 PR 在官方真实服务器上的效果，最终放弃。
- 方案改为： **基于官方插件制作中文增强版** （Obsidian Clipper CN）。

这一个优势，每天给你省的时间就够值了。

**2\. 标签页管理，Edge 远好于 Chrome**

你研究一个 AI 项目，可能同时打开：GitHub 项目页、Issues 页、YouTube 教程、B站对比视频、Product Hunt 上的相关产品……十几个标签页起步。

## 第一步：切换到 Edge 浏览器

1. 1.**下载 Edge** ：打开 microsoft.com/edge ，下载安装包，安装。
2. 2.**导入 Chrome 数据** ：首次打开 Edge 时，它会提示你导入 Chrome 的书签、密码、历史记录。点击"导入"，选择 Chrome，一键迁移。
3. 3.**设为默认浏览器** ：
	- Windows： `设置` → `应用` → `默认应用` → 把 `.html` 、 `HTTP` 、 `HTTPS` 全部改成 Edge
		- Mac：打开 `系统设置` → `通用` → `默认网页浏览器` → 选 Edge
4. 4.**Chrome 处理** ：先别卸载，保留两周。确认 Edge 上所有网站登录、密码、扩展都正常后再卸载。

对，就是这个项目。下面我按顺序讲怎么装和怎么用。

---

## 第一步：下载扩展包

1. 1.打开项目的 **Releases 页面** ： `github.com/nextcaicai/obsidian-clipper-cn/releases`
2. 2.找到最新版本（目前是 1.4.5），下载 **`obsidian-clipper-cn-*-chrome.zip`** 这个文件（Edge 属于 Chromium 系浏览器，用这个包）
3. 3.下载后 **解压** 到一个你不会误删的文件夹，比如 `D:\工具\obsidian-clipper-cn`

---

## 第二步：在 Edge 中加载扩展

1. 1.Edge 地址栏输入 `edge://extensions` 回车
2. 2.左下角打开 **"开发人员模式"** 开关
3. 3.点击出现的 **"加载解压缩的扩展"** 按钮
4. 4.选择你刚才解压出来的 **`dist`** 文件夹
5. 5.扩展加载成功后，浏览器右上角会出现 Clipper 图标

> 装完后建议点击图钉图标把它固定到工具栏，方便随时用。

---

## 第三步：安装 Obsidian

1. 1.去 obsidian.md 下载安装 Obsidian
2. 2.打开后点击 **"Create new vault"** ，名字随便起，比如 `AI项目库`
3. 3.记住这个 Vault 的存放路径，后面 Clipper 需要指向它

---

## 第四步：让 Clipper 连接 Obsidian

1. 1.点击浏览器右上角的 Clipper 图标
2. 2.首次使用会提示你选择 Obsidian Vault，选你刚才创建的那个
3. 3.如果没自动提示，在 Clipper 设置里手动指定 Vault 路径

---

## 第五步：配置解释器（让 AI 自动帮你做摘要）

1. 1.点击 Clipper 图标 → 右上角齿轮进入 **设置**
2. 2.找到 **General** 里的 **Interpreter（解释器）** 区域
3. 3.选择模型，推荐 **Gemini 2.0 Flash** （免费、速度快）
4. 4.去 Google AI Studio 申请一个 API Key，填进去
5. 5.保存

---

## 第六步：创建 3 套模板

在 Clipper 设置里找到 **Templates（模板）** ，点"+"新建：

### 模板 1：GitHub 项目

- **名称** ：GitHub 项目收录
- **匹配规则** （URL Pattern）： `github.com/*`
- **内容** 粘贴以下：

```
yaml---
title: {{title}}
url: {{url}}
tags: [GitHub项目, AI项目, 待评估]
date: {{date}}
---

## 项目概述
{{interpreter:用简体中文总结这个AI项目是做什么的，目标用户是谁，需要什么技术基础，改造难度1-10分，改造价值1-10分}}

## 我的改造想法
<!-- 剪存后手动补充 -->

## 原文
{{content}}
```

### 模板 2：视频教程

- **名称** ：视频教程收录
- **匹配规则** ： `youtube.com/*` （加一条）和 `bilibili.com/*` （再加一条）
- **内容** ：

```
yaml---
title: {{title}}
url: {{url}}
tags: [教程, AI教程, 待整理]
date: {{date}}
---

## 视频摘要
{{interpreter:用简体中文总结这个视频的核心内容，分步骤列出}}

## 适合写什么内容
{{interpreter:基于视频内容，给"教零基础的人用GitHub+AI改造项目"这个定位生成2个选题}}

## 关键步骤
{{highlight}}

## 完整文字稿
{{content}}
```

### 模板 3：行业文章

- **名称** ：行业洞察
- **匹配规则** ： `mp.weixin.qq.com/*` 、 `sspai.com/*` 、 `producthunt.com/*` （每条单独加）
- **内容** ：

```
yaml---
title: {{title}}
url: {{url}}
tags: [行业洞察, 产品灵感]
date: {{date}}
---

## 核心观点
{{interpreter:用简体中文提炼这篇文章的核心观点，不超过5条}}

## 跟我的定位的关系
{{interpreter:分析这篇文章跟"教零基础的人用GitHub+AI改造项目"有什么关联}}

## 原文
{{content}}
```

## 第七步：跑通验证

依次测试三个场景：

| 测试 | 操作 | 预期结果 |
| --- | --- | --- |
| GitHub | 打开任意 GitHub AI 项目页 → 点 Clipper 图标 | 自动匹配 GitHub 模板，解释器生成中文摘要 |
| B站视频 | 打开任意 B 站视频 → 点 Clipper 图标 | 自动匹配视频模板，提取字幕文字稿 |
| 微信文章 | 打开一篇公众号文章 → 点 Clipper 图标 | 图片和正文完整抓取 |

每次剪存后，去 Obsidian 里检查笔记是否正确生成。

---

## （可选）飞书文档支持

如果你后续用飞书协作，可以配置飞书 API：

1. 1.去 飞书开放平台 创建一个自建应用
2. 2.给应用开通权限： `docx:document:readonly` 和 `wiki:node:read`
3. 3.拿到 App ID 和 App Secret
4. 4.在 Clipper 设置 → General → **飞书 / Lark** 区域填入

---
