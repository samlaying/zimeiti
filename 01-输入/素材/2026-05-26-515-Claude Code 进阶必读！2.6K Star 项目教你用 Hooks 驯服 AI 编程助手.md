## 核心概念：Hooks 是什么？
- **本质**：Hooks 是一套**控制逻辑**，允许在 Claude Code **执行任何操作之前和之后**插入自定义脚本，从而对其行为进行干预和约束。
- **解决的问题**：改变 Claude Code 默认的“我行我素”状态，实现对其行为的精细化控制。
  - **例子**：防止 AI 在用户要求“别删文件”时仍执行删除操作；强制 AI 遵循特定的代码规范。

## 项目三大核心能力
该项目通过 Hooks 赋予了 Claude Code 三大核心能力：

### 1. 安全防护
- **功能**：智能拦截危险命令，保护系统和项目安全。
- **例子**：自动拦截 `rm -rf` 等具有破坏性的终端命令。

### 2. 智能提示
- **功能**：在 AI 采取行动（如代码提交）前，注入必要的**上下文信息**，使 Claude 的理解和输出更符合预期。
- **例子**：在提交代码前，向 Claude 注入团队的代码风格指南、项目背景知识等，让 AI “更懂你”。

### 3. 团队协作（Builder-Validator 模式）
- **功能**：模拟软件开发中的角色分工，实现多 Agent 协同工作。
- **模式**：设置不同的 Agent 角色，如：
  - **Builder**：负责编写代码。
  - **Validator**：负责审查代码。
  - **Agent**：各司其职，互相配合，形成自动化流水线。

## 关键技术亮点
- **精巧的设计**：使用 **UV 单文件脚本**。
  - 每个 Hook 都是一个**独立的 Python 文件**。
  - 自带依赖声明，**不污染用户的主项目环境**。
  - 实现了轻量化、模块化和可移植。
- **高阶玩法 - MetaAgent**：
  - 一个能够**创建和配置其他 Agent** 的元代理。
  - 被认为是该项目的高阶玩法，实现了 AI 助手的自我扩展和定制。
