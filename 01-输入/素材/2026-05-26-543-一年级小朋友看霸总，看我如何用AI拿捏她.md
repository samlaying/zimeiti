# 一年级小朋友看霸总，看我如何用AI拿捏她

> [鲁班大叔_007](https://space.bilibili.com/190795407) | [BV1L1cFz1Ejn](https://www.bilibili.com/video/BV1L1cFz1Ejn) | 时长 5:33

## 一、项目背景与目标
### 1.1 灵感来源
- UP主鲁班大叔_007发现一年级小朋友沉迷观看霸总短剧，受到Colorbot大火启发，计划利用AI技术自动监控并干预电视内容，以保护儿童免受不良影响。
- 核心想法：当检测到霸总短剧时，自动切换电视内容，并拍摄小朋友的疑惑表情以增强视频效果；进一步将电视变为学习机，通过互动题目提升教育价值。

### 1.2 最终目标
- 开发“少儿电视防沉迷智能监控系统”，实现以下功能：
  - 自动检测霸总短剧并切换应用。
  - 通过语音播报警告信息，锁屏并显示数学题或语文题。
  - 小朋友需通过语音回答问题解锁屏幕，将电视转化为学习工具。
  - 通过手机iMessage远程控制监控逻辑，灵活调整策略。

## 二、方案设计
### 2.1 系统架构
- 采用客户端-服务端模式：
  - **服务端**：使用Python编写，负责与电视通信、内容分析和控制逻辑。
  - **客户端**：安卓APK应用，部署在电视上，作为与服务端的交互界面。
- 系统名称：少儿电视防沉迷智能监控系统（简称“就不让你看”）。

### 2.2 技术栈
- **语音对话**：字节火山引擎的语音技术。
- **图像识别与AI对话**：Deepseek实现内容分析和交互。
- **通信协议**：ADB（Android Debug Bridge）协议，用于服务端与电视之间的截屏、控制等操作。
- **开发工具**：Python用于服务端，安卓Studio用于客户端开发。

### 2.3 工作流程
1. **定时监控**：服务端通过ADB协议定时对电视机进行截屏。
2. **内容分析**：截屏图像发送给Deepseek，进行霸总短剧识别。
3. **自动干预**：如果检测到霸总短剧，立即切换电视到“就不让你看”应用。
4. **语音播报**：使用TDS生成语音，在电视上播报警告信息，并锁死屏幕。
5. **互动学习**：屏幕显示数学题或语文题，小朋友通过语音识别回答解锁。
6. **远程控制**：UP主通过手机iMessage随时调整监控逻辑。

## 三、技术实现步骤
### 3.1 环境准备
- **步骤1**：下载并安装安卓Studio，用于客户端开发。
- **步骤2**：与AI进行多轮对话（如字幕所述“9981轮的对话”），完成语音对话、播报和展示功能的开发。

### 3.2 ADB连接与调试
- **步骤1**：打开电视的ADB调试开关。以TCL电视为例：
  - 进入设置 > 关于本级信息。
  - 使用遥控器按上、下、左、右键打开隐藏菜单。
  - 在菜单中启用ADB调试。
- **步骤2**：在控制台输入ADB命令连接电视：`ADB Connect [电视机的IP地址]`。
- **ADB常用命令示例**：
  - 模拟遥控器：`Key Event 24`（音量增加）、`Key Event 25`（音量减少）。
  - 截屏：`Screen Cap`命令截屏并保存到指定位置，使用`Pore`命令下载到电脑。
  - 录屏：`Screen Record`命令录制电视画面（本视频素材即通过此命令录制）。

### 3.3 服务端与客户端开发
- **服务端逻辑**：基于Python编写，集成ADB通信和AI分析功能。
- **客户端尝试**：
  - 最初计划开发安卓APK，但因TCL电视限制，改用电视浏览器方案。
  - 使用AI编写HTML控制页面，通过ADB打开电视浏览器访问。
  - 最终选用开源电视浏览器TV Blue，实现自动切屏和文字播报功能。

## 四、遇到的挑战与解决方案
### 4.1 第三方APK安装限制
- **问题**：TCL电视不支持安装第三方APK，通过ADB调试安装或上传安装均失败。
- **尝试解决方案**：
  - 使用U盘安装，但反复修改代码需重装，效率低下。
  - 改用电视浏览器：先尝试官方TCL Bluehill浏览器（海外版，无法使用），后找到开源TV Blue浏览器，成功部署。
- **建议**：在开发前确认电视品牌的应用限制，优先考虑基于浏览器的轻量级方案。

### 4.2 麦克风权限问题
- **问题**：电视麦克风权限被官方语音助手占用，第三方应用无法获取录音权限，导致语音识别功能失效。
- **尝试解决方案**：
  - AI建议关闭原厂语音功能，但无效。
  - 探索“野路子”方法，但风险高，可能损坏电视。
- **最终调整**：降低预期，简化方案，只保留文字、图片、视频发送和关机功能。

### 4.3 最终调整方案
- **简化版功能**：开发HTML控制页面，支持发送文字、图片、视频到电视，并远程关闭电视。
- **干预逻辑**：当小朋友播放霸总短剧时，播报警告文字，播放“电视机坏了”的视频，然后自动关机。
- **节目效果**：通过简单操作“拿捏”小朋友，如引导其去念毛笔字后测试系统。

## 五、总结与展望
- **核心要点**：本项目展示了如何利用AI和编程技术（如Python、ADB、Deepseek）设计智能监控系统，以干预儿童电视观看行为。尽管面临硬件限制（如TCL电视的APK安装和麦克风权限问题），但通过灵活调整方案（如使用浏览器替代APK），仍实现了基础控制功能。未来计划解决麦克风权限问题，将电视转变为互动学习机，并通过评论区交流技术难题。
- **资料获取**：项目资料可通过私信UP主获取，后续视频将填补其他技术坑点。
