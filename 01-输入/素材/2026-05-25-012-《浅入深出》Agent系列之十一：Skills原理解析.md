# 《浅入深出》Agent系列之十一：Skills原理解析

> [奇创喵](https://space.bilibili.com/3546923843717447) | [BV16URWBuEas](https://www.bilibili.com/video/BV16URWBuEas) | 时长 15:23

## Skills原理解析

### 1. Skills的定义与翻译
- Skills在中文中常翻译为“能力”，但在大模型上下文中，没有完美翻译，因此直接使用英文Skills。
- Skills本质上是一个文本文件，通常是Markdown文件，用于指导LLM（大型语言模型）的行为。

### 2. Skills的内容结构
Skills文件包含两个主要部分：

#### 2.1 Metadata（元信息）
- 包含Name（名字）和Description（描述）。
- Name：标识Skill的名称，例如“Get Weather”。
- Description：描述Skill的用途和使用场合，例如“获取本地今天天气”。

#### 2.2 Instruction（指令）
- 结构化文本，指导LLM执行步骤和约束。
- 可以包括总结性标题、步骤、输出格式等。

### 3. 实例分析：获取天气Skills
以获取天气为例，展示Skills的实际应用。

#### 3.1 Metadata示例
- Name: Get Weather
- Description: 获取本地今天天气

#### 3.2 Instruction示例
- 步骤：
  1. 使用get location获取当前地点。
  2. 使用get date获取今天的日期。
  3. 使用get weather获取天气信息。
- 输出格式：JSON格式，例如：
  ```json
  {
    "地点": "广州",
    "日期": "2025年5月6日",
    "天气": "晴天"
  }
  ```

### 4. Skills的作用
- 固化LLM的行为，避免自由发挥导致的错误。
- 提高tool calling的准确性和效率，减少链路错误。
- 解决传统方法中tool调用链路不准确的问题，例如通过Skills明确步骤。

### 5. 加载策略
#### 5.1 按需加载原理
- 当Skills数量多时，全部注入会导致上下文爆炸。
- 解决方案：按需加载，使用Metadata作为目录表。
- 初始注入Metadata列表到LLM上下文，LLM根据问题选择相关Skill。
- 类似于Retrieval，使用函数从Skills库中检索Markdown文件。

#### 5.2 Retrieval Loop
- 循环加载：LLM可能需要多个Skills来完成任务。
- 检索一个Skill后，检查是否足够或依赖其他Skill，继续检索直到满足需求。

### 6. 卸载策略
- 卸载在对话轮次结束后自动进行。
- 方法：清空缓存（cache），将Skills内容从上下文中移除。
- 保持上下文清洁，避免累积过多信息，防止注意力下降。

### 7. Skills的注入方式
- 通常作为System Prompt插入到LLM的上下文中。
- 将Skills的Markdown内容包装后注入，使LLM获得相应能力。

### 8. Skills与AgentOS
- Skills可以类比为新时代的程序，Agent作为操作系统运行这些程序。
- 例如，获取天气Skills像一个天气助手软件，发邮件Skills像一个邮件软件。
- 这体现了AgentOS的概念，但核心是规范化LLM的思考流程，不仅仅是简单的Prompt Engineering。

### 9. 总结
- Skills是LLM行为的指导文件，通过Metadata和Instruction定义行为。
- 采用按需加载和卸载策略优化上下文使用，提升Agent的效率和准确性。
- 理解Skills有助于掌握Agent构建中的关键机制，解决实际应用中的问题。

总结核心要点：Skills作为LLM的指导性Markdown文件，通过结构化Metadata和Instruction固化行为，采用按需加载和自动卸载策略管理上下文，是提升Agent效率和准确性的关键组成部分，体现了从Prompt Engineering到更复杂系统设计的演进。
