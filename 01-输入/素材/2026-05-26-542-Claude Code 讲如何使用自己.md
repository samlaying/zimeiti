# Claude Code 讲如何使用自己

## Claude Code 高效使用指南：从心态转变到实战技巧

本笔记基于视频“Claude Code 讲如何使用自己”（UP主：王树义老师）中分享的 Claude Code 创始人 Boris Cherney 和众多开发者的实战经验，旨在帮助用户提升开发效率。以下是关键知识点、具体建议和实操步骤的整理。

### 1. 心态转变：从写代码者到产品经理
- **核心理念**：使用 Claude Code 时，你不再是直接编写代码的人，而是管理 AI 写代码的产品经理。
- **最大错误**：输入模糊任务并期望完美输出。
- **正确做法**：将 Claude 视为能力强但不了解上下文的初级开发者，提供具体需求、引用文件路径、说明期望行为。

### 2. Plan Mode：从探索到执行的有序流程
- **使用频率**：Boris Cherney 的 90% 对话都从 Plan Mode 开始。
- **操作步骤**：按两次 Shift Tab 进入 Plan 模式。
- **工作原理**：
  - Claude 先探索代码库，分析结构，生成方案。
  - 用户审核方案通过后，再执行。
  - 好处：避免方向错误浪费时间，类似让新员工先写方案确认后再动手。
- **适用场景**：复杂任务；简单 bug 修复或实验代码可直接执行。

### 3. cloud.md：项目记忆与规则维护
- **作用**：Claude Code 的项目记忆文件，在每次对话开始时自动加载，帮助 Claude 了解项目。
- **关键实践**：
  - 每次 Claude 犯错，就将规则添加到 cloud.md，防止再犯。
  - Anthropic 团队每周更新此文件多次，甚至在代码审查时同步更新。
- **内容建议**：项目说明、技术栈、版本号、构建命令、代码规范、已知的坑。
- **生成方法**：使用 `/init` 命令快速生成初始版本，然后编辑添加项目特定规则。

### 4. 并行处理：多实例与工作目录管理
- **效率提升策略**：Boris Cherney 同时运行 10-20 个 Claude 实例，实现任务并行。
- **具体方法**：
  - 使用 Git worktree 为每个任务创建独立工作目录。
  - 用 Tmux 或 Warp 管理多个终端。
  - 从执行者转变为协调者，同时推进多个任务。
- **避免冲突**：使用 Subagents，在单个对话中并行委托子任务。

### 5. 自动化与外部集成：Hooks 和 MCP
- **Hooks（钩子）**：用于必须执行的自动化任务。
  - 例如：使用 Post ToolUse 钩子在每次编辑后自动格式化代码。
  - Boris 团队用此处理代码格式化的最后 10%。
- **MCP（Model Context Protocol）**：让 Claude 连接外部服务，如 Github、Slack、数据库、PlayWrite 浏览器测试。
  - 注意：MCP 会消耗上下文空间，建议使用 `/context` 命令检查并禁用不需要的服务。
- **配置建议**：在设置清单中配置 Post ToolUse Goals 以自动格式化。

### 6. 验证与反馈循环：提升输出质量
- **验证方式的重要性**：给 Claude 验证方式，输出质量可提升 2-3 倍。
- **具体实践**：
  - 让 Claude 先写测试，再实现功能。
  - 用 PlayWrite 自动化测试外部应用。
  - 用 `/chrome` 命令集成浏览器验证 UI。
  - 形成闭环反馈：写代码 → 验证结果 → 修正问题，比反复调整提示词更有效。

### 7. 30 分钟快速设置清单
以下是完整实操步骤，用于快速建立高效的 Claude Code 工作环境：
1. **安装 Claude Code**：完成基础安装。
2. **添加 Github MCP 集成**：连接外部服务以增强功能。
3. **进入项目运行 `/init`**：生成初始 cloud.md 文件。
4. **编辑 cloud.md**：添加项目特定规则，如技术栈和代码规范。
5. **配置 Post ToolUse Goals**：设置自动格式化等自动化任务。
6. **创建常用命令**：例如 `/commit` 和 `/review` 以简化工作流。
7. **进入 Plan Mode 测试任务**：验证设置，确保流程顺畅。

### 核心要点总结
使用 Claude Code 的关键在于转变心态为产品经理，充分利用 Plan Mode 进行有序探索，维护活的 cloud.md 文件以减少错误，并行处理多个任务提升效率，配置 Hooks 和 MCP 实现自动化集成，建立验证反馈循环确保质量。这些经验来自 Claude Code 创始人和社区开发者的实战总结，通过快速设置清单，用户可以在 30 分钟内搭建高效环境，从而显著提升开发效率。
