# Claude Code + Pencil 生成网页演示笔记

## 1. 背景介绍
- **UP主**：退役程序员，分享使用Pencil和Claude Code联动生成网页的经验。
- **目标**：通过AI工具自动化前端设计和开发流程，提升效率。
- **核心概念**：结合设计工具Pencil和AI助手Claude Code，从提示词生成网页设计图并转化为实际网页代码。

## 2. 工具介绍与安装
- **Pencil**：
  - 一个设计软件，用于创建网页设计图。
  - 提供免费版本，可从官网下载安装包，或在VSCode中安装名为Pencil的扩展。
  - 安装步骤：下载安装包后，拖拽到应用程序文件夹完成安装。
- **Claude Code**：
  - AI助手，用于根据设计生成代码。
  - 通过连接本地大模型登录，例如Mistro或千问三Coder模型。

## 3. 实操步骤详解
### 3.1 安装与初始设置
1. **安装Pencil**：访问Pencil官网，下载安装包，或通过VSCode扩展市场安装。
2. **创建项目目录**：在本地计算机上创建一个新目录，用于存放项目文件。
3. **登录Claude Code**：启动Claude Code，选择连接本地大模型（如Mistro或千问三Coder）。

### 3.2 生成设计图
- **检查MCP Server**：在Claude Code中输入`-mcp`命令，确认Pencil的mcp server已自动集成，无需额外操作。
- **获取提示词**：访问Pencil官网的Prompt Gallery页面，复制样例提示词（例如第二个样例）。
- **生成设计图**：在Claude Code中粘贴提示词并回车，启动AI生成过程。示例中，该过程耗时约50分钟，在Pencil画布上绘制出网页设计图。
  - **注意**：可以先在Pencil中创建空白画布以辅助操作。

### 3.3 生成网页代码
- **选择大模型**：初始使用Mistro大模型，但遇到JSON格式错误，导致生成失败。
- **调整方案**：改用千问三Coder大模型，并附加错误代码说明，指示AI调整数据类型以匹配要求。
- **执行生成**：在Claude Code中输入命令（如“根据Pencil中选中的内容创建完全一致的网页版”），AI根据设计图生成HTML格式的网页代码。
  - **结果示例**：成功生成网页，还原度较高。

### 3.4 后续尝试
- UP主尝试创建了多个网页，整体感觉生成效果受大模型稳定性和电脑性能影响较大。

## 4. 经验总结与建议
- **大模型选择**：不同大模型表现差异大，例如Mistro可能产生格式错误，而千问三Coder更稳定，建议根据需求测试多个模型。
- **硬件要求**：电脑性能直接影响生成速度和流畅度，高性能设备能提升体验。
- **过程耗时**：设计图生成可能长达数十分钟（示例为50分钟），需耐心等待。
- **效果依赖**：最终页面质量取决于AI的“心情”和稳定性，可能存在不确定性。
- **实操提示**：利用Pencil官网的Prompt Gallery获取优化提示词，可提高生成效率。

## 5. 核心要点总结
本次演示展示了如何通过Pencil和Claude Code的联动，实现从前端设计到代码生成的自动化流程。关键点包括安装设置、提示词使用、大模型切换和错误调整，强调了AI工具在前端提效中的潜力，但需注意模型选择和硬件条件对结果的影响。实践表明，这种方法可以高效生成设计图和网页代码，但成功与否依赖于AI的可靠性和环境配置。
