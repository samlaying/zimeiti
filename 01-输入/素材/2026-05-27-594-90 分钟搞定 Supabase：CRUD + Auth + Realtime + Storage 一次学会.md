# 90 分钟搞定 Supabase：CRUD + Auth + Realtime + Storage 一次学会

> [後躺平時代](https://space.bilibili.com/3690981654268705) | [BV1gMiEB4E5e](https://www.bilibili.com/video/BV1gMiEB4E5e) | 时长 1:26:45

## Supabase 快速入门笔记

### 1. 核心概念与项目设置
- **Supabase 是什么**: 一个开源的 Firebase 替代品，被称为 **BaaS (Backend as a Service)**。它允许开发者在不自行构建复杂后端的情况下，获得完整的后端功能（数据库、认证、存储、实时功能等），并专注于前端体验。
- **核心优势**: 基于 **PostgreSQL**，开源，开发者友好，易于扩展和保障安全。
- **创建账户与项目**:
    1. 访问 `supabase.com` 创建账户（支持 GitHub SSO、邮箱等）。
    2. 创建一个 **组织 (Organization)** 以管理项目，选择“个人”类型和“免费”计划。
    3. 在组织内创建 **新项目 (New Project)**，需设置项目名称、数据库密码（建议自动生成并保存）、选择离用户最近的区域（如 `West US`）。
    4. 项目创建后，可在仪表盘看到提供的核心产品：认证、存储、Edge Functions、实时数据等。

### 2. 数据库操作
- **表编辑器 (Table Editor)**: 可视化方式管理数据表。
- **创建表**:
    - 示例：创建 `users` 表，字段包括 `id` (自动生成的UUID)、`created_at` (自动时间戳)、`email` (text, 非空, 唯一)、`age` (int8, 默认值20)。
    - **重要操作**：创建表时默认启用 **RLS (行级安全)**，必须添加策略才能进行任何操作。
- **行级安全 (RLS)**:
    - **作用**：根据策略控制用户（通过角色）可以对表执行哪些操作（SELECT, INSERT, UPDATE, DELETE）。
    - **角色 (Roles)**:
        - `anon`: 匿名用户（所有人）。
        - `authenticated`: 已认证用户。
        - `dashboard_user`: 仪表盘用户（你本人）。
    - **策略模板**：推荐使用内置模板快速创建策略，如“允许所有人读取”、“仅允许认证用户插入”等。
    - **示例**：为 `users` 表创建策略：允许所有读取、仅允许认证用户插入和更新（更新仅限自己的数据）、仅允许认证用户删除。

### 3. 连接 React 应用
- **安装客户端库**: `npm install @supabase/supabase-js`
- **创建客户端实例**:
    ```typescript
    // SupabaseClient.ts
    import { createClient } from '@supabase/supabase-js'
    const supabaseUrl = '你的项目URL' // 从项目设置-API获取
    const supabaseAnonKey = '你的anon key' // 从项目设置-API获取
    export const supabase = createClient(supabaseUrl, supabaseAnonKey)
    ```
- **安全提示**: 生产环境中，URL 和 Key 应存储在 `.env` 环境变量中。

### 4. 实现 CRUD 操作 (以 `tasks` 表为例)
- **准备**：创建 `tasks` 表，字段为 `id`, `created_at`, `title` (text), `description` (text)，**暂时禁用RLS**。
- **连接代码**：
    - **插入 (Create)**:
        ```javascript
        const { error } = await supabase.from('tasks').insert({ title, description })
        ```
    - **查询 (Read)**:
        ```javascript
        const { data: tasks, error } = await supabase.from('tasks').select('*').order('created_at', { ascending: true })
        ```
    - **更新 (Update)**:
        ```javascript
        const { error } = await supabase.from('tasks').update({ description: newDescription }).eq('id', taskId)
        ```
    - **删除 (Delete)**:
        ```javascript
        const { error } = await supabase.from('tasks').delete().eq('id', taskId)
        ```
    - **错误处理**：每个操作都应检查返回的 `error` 对象。

### 5. 集成认证 (Authentication)
- **使用 Auth UI 组件**：Supabase 提供预构建的登录/注册组件。
- **手动集成认证逻辑**:
    - **注册**：`const { data, error } = await supabase.auth.signUp({ email, password })`
    - **登录**：`const { data, error } = await supabase.auth.signInWithPassword({ email, password })`
    - **登出**：`await supabase.auth.signOut()`
    - **监听会话变化**:
        ```javascript
        useEffect(() => {
          const { data: { subscription } } = supabase.auth.onAuthStateChange((event, session) => {
            setSession(session)
          })
          return () => subscription.unsubscribe()
        }, [])
        ```
- **会话管理**：
    - `supabase.auth.getSession()` 获取当前会话。
    - 根据 `session` 是否存在来决定显示主应用（如任务管理器）还是认证组件。
- **关联数据与用户**：在插入数据时，可将 `session.user.email` 存入对应列，以建立用户与数据的关联。

### 6. 启用实时功能 (Realtime)
- **在表上启用**：在表编辑器中，编辑表设置，勾选“Enable Realtime”。
- **在前端订阅变更**:
    ```javascript
    useEffect(() => {
      const channel = supabase.channel('tasks-channel')
      channel.on(
        'postgres_changes',
        { event: 'INSERT', schema: 'public', table: 'tasks' },
        (payload) => {
          const newTask = payload.new as Task
          setTasks(prev => [...prev, newTask])
        }
      ).subscribe()
      return () => { supabase.removeChannel(channel) }
    }, [])
    ```
- **效果**：任何对该表的插入操作都会实时广播并更新所有已订阅客户端的界面。

### 7. 使用文件存储 (Storage)
- **创建存储桶 (Bucket)**：在存储页面创建，如 `tasks-images`，可设为公开或应用RLS策略。
- **上传文件流程**:
    1. 在前端获取文件对象（如通过 `<input type="file">`）。
    2. 生成唯一文件路径：`const filePath = ${file.name}-${Date.now()}`。
    3. 上传到桶：`const { error } = await supabase.storage.from('tasks-images').upload(filePath, file)`。
    4. 获取公开URL：`const { data } = supabase.storage.from('tasks-images').getPublicUrl(filePath)`。
    5. 将获取的URL存储到数据库表的相应列中（如 `image_url`）。
- **显示图片**：在前端使用 `<img src={task.image_url} />`。
- **存储策略**：需要为存储桶设置RLS策略（如允许认证用户插入），否则上传会失败。

### 总结
本课程系统介绍了 Supabase 这一强大的 BaaS 平台。核心在于：**一个后端服务，五大核心功能**。从零开始创建 Supabase 项目，通过表编辑器和 SQL 管理 PostgreSQL 数据库，并**深刻理解行级安全 (RLS)** 这一安全基石。课程演示了如何将 Supabase 无缝集成到 React 应用中，完整实现了**增删改查 (CRUD)** 操作，并进一步构建了**邮箱密码认证**系统来管理用户。最后，课程展示了如何利用**实时订阅**功能实现数据的毫秒级同步，以及如何使用**存储服务**来安全地上传和访问用户文件。掌握这些内容，你已具备使用 Supabase 快速构建功能丰富的全栈应用的基础能力。
