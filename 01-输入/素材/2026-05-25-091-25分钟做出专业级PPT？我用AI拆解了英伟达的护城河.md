# 25分钟做出专业级PPT？我用AI拆解了英伟达的护城河

> [绿皮火车Roadtrip](https://space.bilibili.com/565619114) | [BV1sY9sBLELj](https://www.bilibili.com/video/BV1sY9sBLELj) | 时长 23:59

## 核心观点：专业PPT与AI一键生成PPT的本质区别
专业PPT并非一键生成，其核心在于**扎实的研究、严谨的分析推演，以及通过文案和视觉表达影响观众**。AI一键生成工具（如Gamma, Canva）产出的PPT虽视觉专业，但内容空洞、结构套路，经不起推敲。专业方案的制作应拆解为“研究、分析、表达”三幕，并始终以人的独立思考为主轴，AI是执行者，人是导演。

## 五步制作流程：以“英伟达护城河”为例
本次制作使用了三个核心工具：
1.  **Notebook LM**：研究工具，用于消化素材、形成判断、建立反幻觉防线。
2.  **Claude Code 的三个 Agent Skills**：分析工具，用于定方向、搭框架、选择论述。
3.  **Claude Design（或自制的VisoDeck）**：视觉化工具，用于将文字方案转化为PPT。

### 第一步：资料整理与知识搭建
*   **目标**：为后续所有步骤打下坚实的知识基础。
*   **方法**：
    *   从**视频**（如YouTube上老黄的采访、Lex Friedman的访谈、中文播客）入手，锚定观点方向和最新输入。
    *   使用 **Notebook LM** 处理视频，能精准引用具体时间戳原话。
    *   通过 NoteBook LM 的 **Deep Research** 功能，自动拉取投行和专业研究机构（如Semi Analysis, Loggan Stanley, ITG）的详实研报，锚定数据和深度。
*   **关键**：视频定方向和判断，研报定数据，两者结合。

### 第二步：分类定框架（使用Claude Code Classify Skill）
*   **目标**：明确PPT的类型，避免后续结构错配。
*   **操作**：
    1.  在Claude Code中启动`PPT Classify`技能。
    2.  提供核心上下文：
        *   **题目**：英伟达的护城河还立得住吗？
        *   **相关素材**：Notebook LM链接。
        *   **目标受众**：技术爱好者和关心英伟达未来发展的研究者。
        *   **预期时长**：约20分钟讲解型。
*   **产出**：技能会判定PPT类型（本次为**研究型**），并给出一个结合题目的**起始框架（非模板）**，如包含“反共识悖论”、“被研究对象的论证”、“检验路径”等六张幻灯片。
*   **重要认知**：不同PPT类型（理论型、研究型、教育型、叙事型）有完全不同的论述结构。研究型以反共识悖论开场，结论需保留可证伪性。

### 第三步：基础论据提取
*   **目标**：从Notebook LM中，为每张幻灯片的核心任务提取具体证据。
*   **方法**：让Claude Code根据每张幻灯片的核心任务，在Notebook LM中构建问题并查找信息。
*   **产出**：一份共用的**“Evidence”论据文档**。**必须先有证据，再进入写作环节**，避免AI在无证据下编造论证。

### 第四步：内容填充与初稿生成（使用Claude Code Research Skill）
*   **目标**：根据框架和证据，生成每章内容的初稿。
*   **操作**：在Claude Code中启动`PPT Research`技能，输入框架和上一步的论据文档。
*   **产出**：针对每一章（如“反共识悖论”）给出候选措辞、被研究者的逻辑链、调查路径（如直观层、反推层、盲区层）等，形成一份研究文档初稿。
*   **价值**：它沉淀了研究型方案的成熟框架（三段式），避免用户每次重新发明结构。

### 第五步：叙事评审与人工审核（使用Claude Code Review Skill）
*   **目标**：优化叙事，核查事实。
*   **操作**：启动`PPT Narrative Review`技能，输入上一步生成的研究文档。
*   **产出**：
    1.  **叙事张力问题**：指出结构上的缺陷（如两页相似构图挨在一起）。
    2.  **事实检查清单**：列出需核查的具体点位（如11个需核查点）。
*   **关键**：**Review技能是“放大镜”，不是“法官”**。它只识别问题，判断和修正需要人来完成。
*   **人工审核流程**：
    1.  **事实核查**：让AI用Notebook LM内部搜索或WebSearch核查优先级最高的点（如P0级问题）。
    2.  **结构调整**：根据Review建议手动重构章节（如将“检查路径”改为“老黄的四层防御路径并逐个审核”）。
    3.  **迭代**：通过“审核-修改-再审核”的循环提升方案质量。初稿7分，迭代后可达9分。
*   **重要**：AI在长上下文和结构搭建上仍有缺陷，结构调整必须人工介入。

## 视觉化呈现：两条路径
在文字稿完成后，进行视觉化设计。

### Option A：使用Claude Design（内测中）
*   **流程**：
    1.  上传文字稿到Claude Design网站，选择“Fly Deck”。
    2.  AI会提出关键设计决策问题（总页数、时长、观众、审美风格等），逼迫用户做决策，而非替用户决策。
    3.  生成初稿后，可进行指令微调（如“每页加一个简短有力的Takeaway”）。
*   **优点**：对话精修，视觉一致性好，速度快。
*   **缺点**：产品在内测阶段，有流量限制，转换后可能需微调字体等细节。

### Option B：使用自制VisoDeck（Claude Code Skill）
*   **流程**：
    1.  在Claude Code中启动`VisoDeck`技能，先生成一个只有版式的**结构化HTML页面**，供检查结构和逻辑。
    2.  结构稳定后，要求AI生成一系列**图片提示词**。
    3.  将提示词交给其他图像生成模型（如Gemini, Google Labs）批量出图。
    4.  将图片目录告知VisoDeck，自动完成页面嵌入，生成最终HTML，可导出为PPTX。
*   **优点**：开源，精细控制，过程可追踪，适合深度定制和对外交付。
*   **对比**：两条路径产出的视觉质量在局部效果上均可达到很高水准，可根据对控制权和定制化的需求选择。

## 关键工具与技能来源
*   **Agent Skills**：2025年AI行业重要技术成果。本次使用的三个Skills（Classify, Research, Review）受Y Combinator现任掌门人Gary Tan开源的**G-Stack**项目启发。G-Stack旨在将AI切换到具体角色（如CEO、设计师）以实现“认知换挡”。

## AI在专业PPT制作中的角色
AI可以高效执行研究、分析、表达三幕中的**搬运和执行工作**，但**贯穿三幕的主轴——独立思考（判断素材可信度、选择合适框架、决定视觉表达方式）——无法外包**。
*   研究幕：你判断哪条证据可信。
*   分析幕：你判断哪个框架对路。
*   表达幕：你判断什么场合用全幅图或分栏。
AI接管了“搬运”，留给你的是无法替代的“判断”。这不是AI抢你的工作，而是AI迫使你专注于真正值钱的判断工作。

## 总结核心要点
制作一份真正有专业水准、能用于高层汇报的PPT，绝非依赖一键生成工具可得。其核心在于建立“**研究-分析-表达**”的完整工作流，并以**人的独立思考**贯穿始终。具体方法是：利用**Notebook LM**进行深度素材挖掘与事实锚定；借助**Claude Code的Agent Skills**（如分类、研究、评审）进行结构化分析、论证搭建与逻辑审查；最后通过**Claude Design或自定义视觉化流程**将严谨内容转化为高质量视觉呈现。整个过程中，AI是提升效率的强力执行工具，而用户作为“导演”，必须主导关键决策、进行事实核查和结构优化，最终才能产出既有洞察力又有说服力的专业方案。正如视频所言：“推理很难，但它决定了你的收入。”
