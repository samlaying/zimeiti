# CSS 内凹边框：四种切角实现方式

> [没逻辑但有态度](https://space.bilibili.com/3690985208941124) | [BV1WY9eBTETo](https://www.bilibili.com/video/BV1WY9eBTETo) | 时长 15:40

## CSS 内凹（凹角）边框实现指南

### 1. 问题背景与核心心法
- **常见场景**：社交应用、外卖结账、登机牌等UI中普遍使用内凹（向内凹陷）的圆角效果。
- **CSS原生限制**：`border-radius` 只能创建向外的凸角，没有直接的 `inset` 或 `concave` 属性。
- **核心心法**：所有实现技术都基于一个几何概念——**从盒子（矩形）中减去一个位于角落外部的圆形**（`box - circle`）。

### 2. 四种生产可用的技术

#### 2.1 技术一：`box-shadow` 技巧（适用于纯色背景）
- **原理**：使用一个伪元素作为“切割刀”，通过其巨大的 `box-shadow` 覆盖周围区域，只留下内凹曲线。
- **实现步骤**：
    1.  创建一个具有 `position: relative` 的容器（如卡片）。
    2.  在容器内添加一个伪元素（如 `::before`），将其定位到目标角落（如右上角），设置为空元素，尺寸为凹角大小（如 `84px` 方形），并添加 `border-radius` 仅圆化内侧的一个角（如 `border-bottom-left-radius`）。
    3.  为该伪元素设置一个无模糊、扩展（`spread`）值等于其边长的 `box-shadow`，颜色与容器背景色相同。
- **示例代码结构**：
    ```css
    .card {
      position: relative;
      background: solid-color;
      border-radius: ...;
    }
    .card::before {
      content: '';
      position: absolute;
      top: 0;
      right: 0;
      width: 84px;
      height: 84px;
      border-bottom-left-radius: 50%; /* 圆化指向内部的角 */
      box-shadow: -14px -14px 0 14px solid-color; /* 阴影覆盖周围 */
    }
    ```
- **优点**：兼容性极好（IE9+），代码简单。
- **缺点**：只能用于**纯色背景**，不支持渐变、图片或毛玻璃效果。

#### 2.2 技术二：径向渐变遮罩（适用于任何背景）
- **原理**：使用CSS `mask`（遮罩）属性，在元素上“雕刻”出形状。遮罩中透明区域将被隐藏，实色区域正常显示。
- **实现步骤**：
    1.  在目标元素（如底部导航栏）上应用 `mask` 属性。
    2.  使用一个 `radial-gradient` 创建一个圆形遮罩，将圆形区域设为透明，其余部分设为元素背景色（或任意颜色）。
    3.  通过 `background-position` 将圆形遮罩定位到目标角落（如 `top center`）。
- **示例代码（创建一个顶部中心凹口的导航栏）**：
    ```css
    .nav {
      background: ...; /* 可以是渐变、图片等 */
      mask: radial-gradient(circle 36px at 50% 0%, transparent 98%, #000 100%);
    }
    ```
- **关键细节**：使用 `98%` 和 `100%` 的过渡是为了防止抗锯齿产生的模糊光晕。
- **优点**：支持任何背景（渐变、图片、毛玻璃），不破坏元素结构。
- **缺点**：主要适用于圆形凹角，自定义形状能力有限。

#### 2.3 技术三：`clip-path: shape()` 函数（用于自定义复杂形状）
- **背景**：`shape()` 函数已在Chrome、Safari、Firefox中稳定支持，用易读的“英文命令”替代了晦涩的SVG路径语法。
- **原理**：使用 `clip-path: shape(...)` 直接裁剪元素的可见区域来形成凹角。
- **实现示例（创建一个右侧带内凹弧度的形状）**：
    ```css
    .element {
      clip-path: shape(
        from 28px 0,
        line to calc(100% - 28px) 0,
        arc to calc(100% - 28px) 28px of 28px,
        line to 100% 28px,
        arc to calc(100% - 28px) 28px of 28px, /* 注意：原视频中此处第二个arc命令可能为笔误，此处按常见用法修正 */
        line to 100% calc(100% - 28px),
        arc to calc(100% - 28px) 100% of 28px,
        line to 28px 100%,
        arc to 28px calc(100% - 28px) of 28px,
        close
      );
    }
    ```
    **关键参数**：`from` (起点), `line to` (直线), `arc to` (圆弧，`of` 后为半径)。
- **核心优势**：通过修改 `arc to` 的方向，可以轻松实现**外凸**或**内凹**的曲线，极具灵活性。
- **重大缺点**：`clip-path` 会裁剪掉元素的边框、阴影和轮廓线，导致这些样式在裁剪边缘消失。

#### 2.4 技术四：`corner-shape` & `border-shape` 属性（未来标准，Chrome已支持）
- **现状**：Chrome 139+ 支持 `corner-shape`，Chrome 147+ 支持 `border-shape`。Safari和Firefox计划在2026年支持。
- **原理**：这是CSS原生提供的解决方案，让浏览器以正确的方式渲染和绘制边框/阴影。
    - `corner-shape: scoop`：在现有 `border-radius` 基础上，直接将圆角变为内凹的“勺形”。
    - `border-shape`：接受与 `shape()` 相同的语法，但重塑的是**整个盒模型**，因此边框、阴影、轮廓会**跟随新形状**，不会被裁剪。
- **示例代码**：
    ```css
    .element {
      border-radius: 20px;
      corner-shape: scoop; /* 将圆角变为凹角 */
      /* 或者更复杂的自定义形状 */
      /* border-shape: shape(...); */
    }
    ```
- **最大优点**：代码极其简洁、语义化，完美保留所有盒模型装饰。
- **当前缺点**：浏览器兼容性差，目前只能在Chromium内核浏览器中使用。

### 3. 技术选择决策树
根据设计需求和背景类型选择：
1.  **纯色背景**：首选最简单的 **`box-shadow` 技巧**。
2.  **渐变、图片或毛玻璃背景**：使用 **径向渐变遮罩**。
3.  **需要自定义非圆形曲线**：使用 **`clip-path: shape()`**（需接受边框阴影被裁剪）。
4.  **面向Chromium内核应用**（如内部工具、Electron应用）：直接使用 **`corner-shape`**。
5.  **兼顾兼容性与未来**：使用 `@supports` 查询，以 `box-shadow` 作为基线，为支持 `corner-shape` 的浏览器提供原生升级。

### 4. 总结
实现CSS内凹边框的核心心法是 **“盒子减去圆形”**。目前有四种主要技术：`box-shadow` 技巧（纯色兼容）、径向渐变遮罩（任意背景）、`clip-path: shape()`（自定义形状）以及新兴的 `corner-shape`/`border-shape`（原生最佳方案）。根据背景类型、形状需求和目标浏览器环境选择合适的技术，并理解其优缺点，就能高效实现这一常见而重要的UI效果。记住这个心法，下次在任何应用中看到内凹边框，你便能立刻洞察其背后的实现逻辑。
