## Chrome DevTools 精讲笔记

### 一、 DevTools 入门与基础操作

#### 1. 打开 DevTools
*   **方式一**：点击浏览器右上角“三个点” → 更多工具 → 开发者工具。
*   **方式二（快捷键）**：直接按 `F12` 键。

#### 2. 命令菜单 (Command Menu)
*   **打开快捷键**：`Ctrl + Shift + P` (Windows) / `Cmd + Shift + P` (Mac)。
*   **常用功能**：
    *   **更改主题**：输入 `Theme`，可选择 `Dark Theme` 或 `Light Theme`。
    *   **页面截屏**：
        *   `Capture Area Screenshot`：截取选定区域。
        *   `Capture full size screenshot`：截取整个可滚动页面。
        *   `Capture node screenshot`：截取选中的 DOM 节点。
    *   **调整窗口位置**：输入 `Dock`，可选择 `Dock to bottom`/`left`/`right` 或 `Undock` (独立窗口)。

### 二、 核心面板详解

#### 1. Elements 面板 (元素面板)
**功能**：检查和修改页面 DOM 结构与 CSS 样式。

*   **检查元素**：右键页面元素 → 检查，或在面板中用 `Ctrl/Cmd + F` 通过 CSS 选择器、XPath 或文本搜索。
*   **样式调试 (Styles 面板)**：
    *   在 `element.style` 中添加/修改样式。
    *   利用自动补全提示不记得的属性名（如输入 `bold` 会提示 `font-weight`）。
    *   查看样式来源（文件及行号）、继承关系、浏览器内置样式 (`user agent stylesheet`) 和伪元素。
    *   **图形化盒模型**：直观显示 `margin`、`border`、`padding` 和元素尺寸。
*   **固定元素状态**：在 `Styles` 面板右上角点击 `:hov`，可勾选 `:hover`、`:active`、`:focus` 等状态，使其样式永久生效，方便调试。
*   **编辑类名**：选中元素后，在 `Styles` 面板顶部点击 `.cls`，可以勾选/取消特定类名，仅影响当前元素，而非全局样式。
*   **复制样式**：右键元素 → `Copy` → `Copy styles`。
*   **Computed 面板**：列出最终生效的所有计算样式。可输入属性名快速定位，或按功能分组查看。
*   **Layout 面板**：
    *   **Grid**：展示并可视化页面中所有的网格布局，可显示行号、轨道大小、区域名称等。
    *   **Flexbox**：展示并可视化页面中所有的弹性布局。
*   **DOM 断点 (DOM Breakpoints)**：右键元素 → `Break on`，可在子树修改、属性修改或节点删除时中断 JS 执行。

#### 2. Console 面板 (控制台面板)
**功能**：执行 JavaScript 命令、查看日志输出和调试信息。

*   **执行 JavaScript**：直接在控制台输入表达式或语句。
*   **常用快捷变量**：
    *   `$_`：返回上一条语句的执行结果。
    *   `$0`：返回当前在 Elements 面板中选中的 DOM 节点。`$1`、`$2` 等依此类推。
*   **日志输出**：
    *   `console.log()`：标准输出。
    *   `console.error()`：错误信息。
    *   `console.warn()`：警告信息。
    *   `console.clear()`：清空控制台。
    *   `console.table()`：以表格形式输出数据，支持排序。
    *   `console.group()` / `console.groupEnd()`：分组输出日志。
    *   `console.time()` / `console.timeEnd()`：计时，用于性能分析。
*   **Live Expressions**：点击控制台左上角“眼睛”图标，可创建实时表达式，其值会随变量变化而实时更新，非常适合跟踪变量状态。
*   **过滤日志**：使用顶部的下拉菜单过滤显示 `Errors`、`Warnings`、`Info` 等级别日志。

#### 3. Sources 面板 (源代码面板)
**功能**：调试 JavaScript 代码。

*   **触发断点**：
    1.  **在代码中写入 `debugger;` 语句**：当代码执行到该行时自动暂停。
    2.  **在源代码面板行号处点击**：手动设置断点（显示蓝色标签）。
*   **调试控制**：代码暂停后，使用右上角按钮：
    *   `Step over` (F10)：单步执行，不进入函数内部。
    *   `Step into` (F11)：单步执行，进入函数内部。
    *   `Step out` (Shift+F11)：从当前函数跳出。
    *   `Resume` (F8)：继续执行，直到下一个断点。
*   **观察窗口 (Watch)**：点击 `+` 添加需要监控的变量或表达式。
*   **调用堆栈 (Call Stack)**：查看函数调用链。右击堆栈中的文件可 `Add script to ignore list`，避免调试框架代码。
*   **代码折叠**：通过命令菜单开启 `Enable code folding`，可折叠/展开代码块。
*   **事件监听器断点 (Event Listener Breakpoints)**：在 `Sources` 面板左侧，可勾选特定事件（如鼠标点击 `click`），当事件触发时中断执行。

#### 4. Network 面板 (网络面板)
**功能**：分析网络请求，优化页面加载性能。

*   **请求列表**：记录所有网络请求，包含 `Name`（名称）、`Status`（状态码）、`Type`（资源类型）、`Initiator`（发起者）、`Size`（大小）、`Time`（时间）、`Waterfall`（瀑布流）。
*   **常用功能**：
    *   **Preserve Log**：勾选后，即使页面跳转也会保留之前的请求记录。
    *   **Disable Cache**：禁用缓存，模拟首次加载。
    *   **Throttling (节流器)**：模拟不同网络环境（如 `Fast 3G`、`Slow 3G`、`Offline`），测试页面性能。
    *   **Network Conditions**：可设置自定义的 `User Agent` 和 `Accept-Encoding`，模拟特定浏览器或设备的请求头。
*   **导出/导入 HAR 文件**：`Export HAR file` 保存网络活动日志，`Import HAR file` 可导入他人日志用于分析问题。
*   **请求详情**：点击单个请求，可查看 `Headers`（请求/响应头）、`Preview`/`Response`（内容预览）、`Timing`（耗时分析）等。
*   **底部概览**：显示总请求数、传输数据量、DOM 加载完成时间、页面完全加载时间等关键指标。

#### 5. Application 面板 (应用面板)
**功能**：管理客户端存储数据。

*   **Local Storage / Session Storage**：查看、编辑、删除键值对。`Local Storage` 数据持久化，`Session Storage` 在标签页关闭后清除。
*   **Cookies**：查看、编辑网站的所有 Cookie，包括其值、域、路径、过期时间等。
*   **在控制台操作**：
    ```javascript
    localStorage.setItem(‘key‘, ‘value‘); // 设置
    localStorage.getItem(‘key‘); // 获取
    localStorage.clear(); // 清空
    document.cookie = "name=value"; // 设置cookie
    ```

### 三、 核心要点总结

Chrome DevTools 是前端开发者不可或缺的调试与分析工具。其核心在于通过 **Elements** 面板实时检查和修改页面结构与样式；利用 **Console** 和 **Sources** 面板进行强大的 JavaScript 调试，包括日志输出、断点控制、变量监控和调用堆栈分析；借助 **Network** 面板深入分析网络请求，模拟弱网环境以优化加载性能；并通过 **Application** 面板管理 Cookie、LocalStorage 等客户端数据。熟练掌握这些面板及其快捷键（如 `F12`、`Ctrl+Shift+P`），能极大提升 Web 开发与问题排查的效率。
