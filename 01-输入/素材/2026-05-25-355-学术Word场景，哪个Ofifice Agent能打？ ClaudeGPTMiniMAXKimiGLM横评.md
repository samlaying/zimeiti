# 学术Word场景，哪个Ofifice Agent能打？ Claude/GPT/MiniMAX/Kimi/GLM横评

> [脑门锃亮的刘博](https://space.bilibili.com/170460565) | [BV1sLcUzhEXt](https://www.bilibili.com/video/BV1sLcUzhEXt) | 时长 8:22

## 视频笔记：学术Word场景AI Agent横评

### 1. 视频背景
- **时间背景**：2020年开春，AI竞争从比参数转向比实战能力。
- **核心观点**：AI Agent（全能AI助手）成为国内外大模型厂商的必争之地，Office场景是第一块实验田。
- **测试目的**：比较五个顶尖AI Agent在真实Word场景下的生产级交付能力。
- **测试Agent**：Claude Sonnet 4.6、ChatGPT、MiniMAX 2.5、Kimi (K2.5)、GLM5。
- **测试方法**：所有Agent采用相同文件和提示词完成任务。

### 2. 测试场景与结果分析
#### 2.1 场景一：基于表格内容填写模板（升级表）
- **任务描述**：根据提供的简单升级表，进行填写。
- **表现结果**：
  - **Kimi (K2.5)**、**Claude Sonnet 4.6** 和 **GLM5** 表现较好，成功填写内容，但字体与模板不一致。
  - **ChatGPT** 出现首行新增空格。
  - **MiniMAX 2.5** 直接破坏了原模板格式。
- **关键点**：多数Agent能完成内容填写，但格式一致性是挑战。

#### 2.2 场景二：毕业论文模板的长文档填写
- **任务描述**：填写复杂毕业论文模板，包括封面、目录、创新点概述、摘要等部分，基于一份已发表的手稿。
- **表现结果**：
  - **只有Claude Sonnet 4.6和Kimi (K2.5)完成了封面填写**：
    - Claude Sonnet 4.6：填写了封面、目录（有重复问题）、创新概述和摘要。
    - Kimi (K2.5)：填写了封面、第二页英文封面、目录（部分未完成）、摘要（但未正确加粗）。
  - **ChatGPT**：破坏了封面格式。
  - **GLM5和MiniMAX 2.5**：填写不完整，MiniMAX仅填写封面，后续内容缺失。
  - **材料方法部分**：Kimi (K2.5)成功填写结果部分，Claude Sonnet 4.6未完成。
- **结论**：从修改工作量看，Kimi (K2.5)比Claude Sonnet 4.6稍好；其他Agent完全未完成任务，模板依从性差。

#### 2.3 场景三：论文换头场景（杂志模板转换）
- **任务描述**：论文被拒后，需将内容填充到新杂志模板，调整格式。
- **表现结果**：
  - **Kimi (K2.5)和Claude Sonnet 4.6完成度较好**：
    - Kimi (K2.5)：满足细小上标、下标要求，且唯一完成二级标题顺序标注。
    - Claude Sonnet 4.6：完成度仅次于Kimi (K2.5)，在标题顺序标注上表现好。
  - **ChatGPT**：基本可交付，但未完成二级标题顺序标注。
  - **GLM5**：丧失了原模板格式。
  - **MiniMAX 2.5**：改变了布局，需要大幅调整。
- **关键点**：格式细节处理能力是区分Agent的关键。

#### 2.4 场景四：合同标注
- **任务描述**：标注劳动合同中的问题条款，并说明原因。
- **表现结果**：
  - **Claude Sonnet 4.6、Kimi (K2.5)、GLM5达到交付级表现**：
    - Claude Sonnet 4.6：不仅标注问题，还自动删除不合理部分，支持接受更改功能。
    - Kimi (K2.5)和GLM5：标注问题并说明原因，Kimi (K2.5)的原因更详细。
  - **ChatGPT**：直接修改了合同，可能未标注问题。
  - **MiniMAX 2.5**：丧失了原格式，未理解任务。
- **操作步骤示例**：Claude Sonnet 4.6允许通过“审阅”中的“接受”功能来接受文档更改，实现一站式修正。

### 3. 整体表现总结
- **最佳表现Agent**：Kimi (K2.5) 和 Claude Sonnet 4.6，在多数场景中表现突出，完成度高。
- **中等表现Agent**：ChatGPT和GLM5，在部分场景可交付，但存在格式或内容问题。
- **较差表现Agent**：MiniMAX 2.5，在多个场景中破坏模板或未完成任务。
- **格式一致性**：所有Agent在字体、加粗等格式细节上仍有改进空间。

### 4. 核心要点总结
视频横评了五个AI Agent在学术Word场景中的实战能力，包括表格填写、毕业论文模板、论文换头和合同标注四个测试场景。结果表明，Kimi (K2.5)和Claude Sonnet 4.6在内容填充和格式处理上表现最佳，能基本满足生产级需求，但Kimi (K2.5)在细节处理上略胜一筹；ChatGPT和GLM5可应付简单任务，但稳定性不足；MiniMAX 2.5则频繁破坏模板，不适用于复杂文档场景。总体而言，AI Agent在Office场景中已能辅助提效，但用户仍需根据任务复杂度选择合适工具并准备后续人工调整。
