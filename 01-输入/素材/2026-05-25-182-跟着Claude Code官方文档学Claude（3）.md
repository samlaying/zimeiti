## 引言
本视频是“跟着Claude Code官方文档学Claude”系列的第三期，重点介绍实战中非常有用的功能——Sub Agents（子代理）。前两期涵盖了概览和工作原理，本期将深入讲解Sub Agents的概念、用途和操作方法。

## 为什么需要Sub Agents
- **上下文窗口稀缺资源**：在使用Claude Code时，一个常见痛点是上下文窗口容易被大量数据填满，如运行测试、扫描代码库、处理日志等，导致Token耗尽，真正需求无法塞入。
- **核心解决方案**：Sub Agents通过委派（delegate）任务来解决这个问题。每个Sub Agent拥有独立的上下文窗口、系统提示词和工具权限，完成任务后只返回摘要，避免污染主窗口，保持主上下文干净。

## Sub Agents的核心概念
- **自动判断与委派**：Claude会自动判断任务类型，并委派给合适的Sub Agent，无需用户手动干预。
- **独立运作**：Sub Agent处理“脏活累活”，主窗口专注于核心任务，实现分工明确。

## 出厂自带的Sub Agents
Claude Code出厂自带三个Sub Agents：
1. **Explore**：使用Hiku模型，速度快、成本低，仅支持读取操作，用于快速探索上下文。
2. **Plan**：继承主模型（只读），在计划模式下帮助搜索上下文、分析代码结构，充当参谋角色。
3. **General Purpose**：全能型，继承主模型并拥有所有工具权限，支持处理复杂、多模态任务。

## 如何创建自定义Sub Agents
- **交互式创建**：用户可以通过交互方式创建自定义Sub Agent，使用Markdown格式配置文件，包括：
  - 名字和描述。
  - 可用工具列表。
  - 使用的模型（如Hiku、Opus等）。
- **配置方式**：可以通过命令或手写YAML/Markdown文件进行设置，便于复用和沉淀团队资产。

## Sub Agents的使用场景
Sub Agents提供四大核心价值：
1. **隔离高输出操作**：将刷屏任务（如跑测试、处理日志）委派给Sub Agent，保护主窗口。
2. **并行研究多个任务**：同时启动多个Sub Agent探索不同模块，互不干扰，提升效率（如效率翻倍）。
3. **链式处理工作流**：支持链式调用，例如先让“code reviewer”找问题，再让“optimizer”修复，形成流水线作业。
4. **条件控制与验证**：通过hooks进行细粒度验证，例如设置数据库Sub Agent仅允许SELECT操作，拦截所有写操作，增强安全性。

## 调用Sub Agents的方式
从暗示到强制，有三种调用方式：
1. **自然语言暗示**：直接告诉Claude想要什么或委派什么，它自动判断并委派给Sub Agent，最轻松的方式。
2. **用@提及强制**：通过@指定Sub Agent，整个对话模式以该Agent身份运行，使用其提示词、工具和模型，实现精准控制。
- 这些方式允许用户从暗示到强制，灵活控制委派力度。

## 与Agent Team的区别
- **Sub Agents**：单会话工作，类似于老板直接管理多个高管，老板与每个高管紧密接触并跟踪进度，高管之间独立工作，不相互交流，保护主窗口。
- **Agent Team**：层级结构，老板通过CEO与高管对接，高管之间可跨会话交流，适用于复杂项目，支持并行持续迭代。
- **使用场景**：简单任务用Sub Agents，大型项目启动Agent Team（目前处于实验阶段，需在JSON文件中设置参数）。

## 总结核心要点
Sub Agents的核心是分工委派，通过独立上下文窗口保护主资源，实现任务隔离、并行处理和成本控制。它自动判断任务类型，提供出厂和自定义配置，支持从自然语言到@提及的灵活调用，与Agent Team的区别在于单会话与多层级结构。总之，Sub Agents是提升Claude Code效率的关键工具，用分工代替堆叠，用委派代替直接处理。
