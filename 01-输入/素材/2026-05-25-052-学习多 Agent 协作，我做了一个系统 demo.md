# 学习多 Agent 协作，我做了一个系统 demo

> [Jasper小笔记](https://space.bilibili.com/34935109) | [BV11sdWBUE4Q](https://www.bilibili.com/video/BV11sdWBUE4Q) | 时长 7:33

## 多Agent协作系统Demo笔记
### 一、 系统概述与目标
- **核心目的**：制作一个系统Demo，将多Agent协作的脚本代码应用于实际系统中，便于日常使用、观察问题，并在反复优化中切身感受多Agent协作的难点。
- **使用流程**：
    1.  **创建角色Agent**：例如“设计师”、“产品经理”、“工程师”等，每个角色都有各自的职责和系统提示词。
    2.  **创建协作方案**：关联需要的Agent，并选择协作模式。
    3.  **与方案交互**：在方案中输入任务，观察多个Agent的协作流程。

### 二、 四种协作模式详解与演示
系统实现了四种协作模式，并使用相同任务（“帮我写一个简单的产品介绍页面，要有标题，内容和按钮”）进行了测试。

1.  **简单路由**
    - **机制**：将用户问题转交给**一个**Agent全权处理。
    - **演示**：任务被分配给“工程师”Agent，由其独立完成。

2.  **静态计划**
    - **机制**：将用户问题预先拆分成一个计划表，依次安排不同Agent去执行，直至表中内容全部完成。
    - **演示流程**：
        1.  系统（可能由“产品经理”Agent）制定一个计划表（如：先由产品描述内容，再由设计师设计，最后由工程师编码）。
        2.  按计划表依次调度不同Agent执行。
        3.  所有步骤完成后，任务结束。

3.  **动态监督**
    - **机制**：由一个**管理者**（监督者）Agent判断任务是否执行完毕。它一次思考下一步该做什么，安排其他Agent去行动，拿到结果后再思考下一步。是更依赖大模型的灵活结构。
    - **演示流程**：
        1.  监督者将任务先派给“产品”Agent。
        2.  监督者根据产品返回的结果，决定下一步（也可能直接派给工程师）。
        3.  如此循环，直至任务完成。

4.  **同伴交接**
    - **机制**：首先将任务下发给团队中的某个角色，然后由这些角色**相互委派**，共同完成任务。它摆脱了中心化的控制，每个Agent都可以相互委托。
    - **演示流程**：
        1.  任务首先交给“产品”Agent。
        2.  产品Agent将任务委派给“设计”Agent。
        3.  设计Agent可能再委派给“工程”Agent。
        4.  通过Agent间的接力，最终完成任务。

### 三、 多Agent协作面临的挑战与优化方案
1.  **上下文过多导致核心指令失效**
    - **问题**：对话历史或任务信息过多，可能使Agent忽略最初的核心任务指令。
    - **优化方案**：
        - **提高任务准确性与具体性**：派发任务时，要求明确任务属性，如操作对象、具体内容、是否需要产物或验收文档。
        - **优化提示词位置**：将当前最新任务放在提示词的最下方，确保指令是上下文中最“新鲜”的信息。
        - **加强提示词管理**：确保各个Agent实例的一致性，避免某个异常Agent的输出误导整个状态。

2.  **工具误匹配**
    - **问题**：模型可能选择与当前任务看似相关但业务上完全无关的工具。
    - **优化方案**：
        - **遵循最小权限原则**：每个Agent只安装其当前职责所需要的工具，避免“技能污染”。
        - **提升模型的工具选择能力**：这是一个需要长期优化的方向。

3.  **“只分析不执行”问题**
    - **问题**：过度强调输出格式（如在同伴交接模式中，Agent需要输出委派指令的格式），导致Agent注意力分散，只顾生成格式而忽略了执行任务本身。
    - **优化方案**：如果在一个任务中需要同时具备“生成指令”和“执行动作”两种注意力，应将其拆分成两次独立的交互，以保证每次执行的准确性。

4.  **任务异常终止**
    - **问题**：Agent输出“任务终止”，但实际任务并未完成；或在中止后表达了下一步计划，但未在本次执行中完成。
    - **优化方案**：在指令设计中，增加“继续”这一指令类型。这样，当Agent准备结束时，它会看到“继续”选项，从而降低误终止的概率。

### 总结
本次Demo系统通过构建和演示四种多Agent协作模式（简单路由、静态计划、动态监督、同伴交接），直观展现了不同协调机制下的执行流程。同时，视频也指出了当前多Agent协作面临的四个核心挑战：上下文干扰、工具误用、执行力分散和异常终止，并提供了具体的优化思路。总体而言，多Agent协作具备潜力，但其在实用性、可靠性和智能水平上仍有很大提升空间，有待未来大模型能力的进一步发展。
