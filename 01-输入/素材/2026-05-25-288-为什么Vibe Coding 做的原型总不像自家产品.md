# 为什么Vibe Coding 做的原型总不像自家产品

> [AI产品经理养成记](https://space.bilibili.com/47333744) | [BV19p9JB2ERx](https://www.bilibili.com/video/BV19p9JB2ERx) | 时长 4:11

## 一、 问题：AI生成的原型为何“形似神不似”？
当产品经理将功能截图和详细描述交给AI，让它通过“Vibe Coding”（直觉编码）方式生成产品原型时，常遇到以下问题：
1.  **风格不符**：产品颜色、品牌色与现有产品不一致。
2.  **布局错位**：功能模块的布局结构与现有产品的设计逻辑不同。
3.  **交互不当**：交互模式不符合产品既有的操作习惯。
4.  **模板化**：AI直接套用通用的后台管理模板，缺乏产品独特性。

## 二、 核心原因分析
AI在生成原型时，通常从通用的模板和设计模式库中进行创作。它缺乏对**你们自家产品独特设计语言和规范**的具体认知，因此无法精准复刻产品的视觉与交互风格。

## 三、 解决方案：创建一个“更懂你产品”的专属AI Skill
为了解决上述问题，UP主提出并实践了一种方法：**不让AI直接画原型，而是先训练它理解你产品的“设计DNA”**。

### 1. 专属Skill是什么？
它是一个用于**后续产品迭代做原型的基础技能**。这个Skill的核心任务不是生成原型，而是**先分析你产品的代码仓库**，从中提炼出稳定、一致的设计规范。

### 2. Skill的工作原理与产出
-   **输入**：你的产品现有代码库。
-   **分析**：AI会学习并判断产品的形态（如Web、移动端、大屏）。
-   **提炼**：从源码中提取出所有稳定的设计元素，例如：
    -   **主题色**（如必须使用 `#00313D`）
    -   **布局架构**（导航栏、侧边栏、内容区结构）
    -   **公共组件**（按钮、卡片、表单的通用样式）
    -   **交互模式**（页面跳转、表单提交、弹窗逻辑）
    -   **代码规范**
-   **输出**：一个为你产品**量身定制的原型设计Skill**。

### 3. 专属Skill的适用场景
此Skill主要适用于两种产品迭代场景：
-   **功能升级**：在现有功能基础上进行改版或增加新模块。此时，可提供**功能截图**作为输入。
-   **全新功能设计**：设计一个完全陌生的功能。此时，仅需提供**需求描述**，无需参考图。

### 4. 使用效果对比
-   **未使用**：AI基于通用模板生成，结果可能颜色、布局、交互均偏离原有风格。
-   **使用后**：AI会**优先套用它学习到的产品专属UI和交互风格**进行创作。最终生成的原型在整体布局、风格上与现有产品高度一致，实现了“像你家的产品”。

## 四、 核心建议与总结
1.  **改变流程**：在让AI直接画原型之前，先投入一个步骤——教会它你产品现有的设计是什么样子。
2.  **先学习，后创造**：建立“专属设计Skill”相当于为AI提供了一本必须遵守的《产品设计规范手册》，确保其输出符合品牌调性。
3.  **追求稳定与一致性**：此方法能让AI在后续的多次迭代中，输出结果更稳定、更可预测，大幅提升原型设计的效率和质量。

---
**核心要点总结**：本视频提出了一种提升AI生成原型质量的方法论：通过分析产品现有代码库，提炼出专属的设计规范（如颜色、布局、组件、交互），并将其封装成一个“Skill”。这样，在后续迭代中，AI就不会套用通用模板，而是会依据已学习的专属设计规范进行创作，从而确保新设计的原型在风格、布局和交互上与现有产品保持高度一致，解决了“做的原型总不像自家产品”的痛点。
