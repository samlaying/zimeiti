# 苹果人机界面指南（HIG）：你需要知道的一切

> 来源：设计师凯文 CreativeKev · 2024-10-13

---

## 一、HIG 是什么？

**Human Interface Guidelines（人机界面指南）**，简称 **HIG**，是苹果为其平台制定的**设计规范和最佳实践**。

| 项目 | 说明 |
|------|------|
| 性质 | 系统级别的交互设计指南 |
| 内容 | 组件设计、基础元素定义与使用、人机交互原则与理念 |
| 目的 | 确保应用程序遵循一致的设计规范，在苹果生态系统内提供**统一的用户体验** |
| 配套资源 | 提供相应的**原生组件库**，供开发人员直接使用 |
| 语言支持 | 官网提供**中文版本** |

---

## 二、HIG 的历史

- **1987 年**，苹果推出第一台彩色显示器 Macintosh II，最早版本的 HIG 随之问世
- 其中一些设计理念产生了深远影响，沿用至今

---

## 三、HIG 的三大基本原则（贯穿全文）

### 1. Clarity（清晰）
- 用户界面（文字、图标等）一定要**清晰易读、容易理解**
- 一切设计以**功能为导向**
- 准确地**突出重点内容**

### 2. Deference（遵从）
- UI 设计的目的是帮助用户理解和交互，**服务于内容**
- **不要过度设计**，本末倒置

### 3. Depth（深度）
- 为视觉元素建立**层级与空间感**
- 帮助用户理解不同元素的关系，使**导航更加明确**
- 交互方式上**逐步引导**，让用户知道自己从哪里来、要到哪里去

---

## 四、关键交互设计理念

### 4.1 视觉一致性
> **目的：建立一个让人信任的环境**

- 一致性帮助用户更容易地**学习和使用新的 App**

### 4.2 信息反馈
> **用户需要明确的信息反馈来掌握主动**

- 用户开始操作时 → 需要**立即的、明确的信息**告知操作正在进行或已完成
- 帮助用户完成任务、减少错误
- 包括：提示任务当前状态（成功/失败）、对可能产生负面影响的行为**提前警告**
- 相关组件：
  - **Alert（警告）**——用于提示信息或警告
  - **Action Sheet（动作菜单）**——用于提供操作选项
  - （建议到 HIG 官网查看两者使用上的区别）

## 五、无障碍设计（Accessibility）

> 苹果是无障碍设计领域的**先驱**

**核心理念：** 使用简洁一致的交互，让完成任务变得更加容易

**多感官支持：**

| 感官 | 技术手段 | 说明 |
|------|----------|------|
| 视觉 | 动态字体、对比度 | 确保布局在不同字体大小下正确显示 |
| 听觉 | VoiceOver（屏幕阅读器） | 辅助视障用户理解内容 |
| 触觉 | Haptics（触感反馈） | 通过震动等触觉反馈辅助交互 |

**具体设计要求：**

| 要求 | 标准 |
|------|------|
| 可点击区域 | 按钮和控制元素至少 **44 × 44 pt** |
| 重要按钮 | 使用**填充色**，使其更加鲜明 |
| 次要按钮 | 使用**中性色**，建立视觉层级 |
| 字体 | 使用 iOS/iPadOS 的**动态字体（Dynamic Type）** |
| 布局适配 | 确保不同字体大小下都能正确显示 |

## 六、HIG 的实际用途

### 6.1 作为工具书查阅
- 包含各种**基础组件**的使用指导和最佳实践（搜索栏、弹窗、选择器等）
- 做 iOS App 设计时，绝大部分组件基于原生组件进行**样式微调**，而非从零开始

### 6.2 提供特定情景的交互指导
- 如何设计**可视化数据**
- **页面加载**时的处理方式
- **新手引导流程**中的最佳实践
- 这些都是非常硬核的交互设计教材

### 6.3 技术层面的约束与指导
苹果通过硬件和软件共同打造生态闭环，许多技术层面的设计**直接影响甚至决定**交互方案：

| 场景 | 说明 |
|------|------|
| NFC 技术 | 用于移动支付、门禁等，设计师需提前了解交互方式、提示页面，并考虑如何整合进用户流程 |
| 导航差异 | 苹果独有的 **Modal View**——允许用户在不脱离主任务的前提下完成简短的子任务，与安卓导航有区别 |
| App Store 审核 | 苹果审核非常严格，**良好的用户体验是审核标准之一**，设计开始就应以 HIG 为参考标准 |

## 七、HIG vs. Material Design 对比

| 维度 | Material Design（Google） | HIG（Apple） |
|------|--------------------------|--------------|
| 侧重点 | 对 **UI 细节、设计系统**做出详细规范和定义 | 更偏向**宏观层面**，对用户体验进行优化 |
| 价值 | 具体的组件规范与视觉标准 | 为整个交互行业奠定基础和方向 |
| 共同点 | 都是系统级别的交互设计指南，都提供原生组件库 |

---

## 八、总结

```
HIG 核心价值
├── 三大原则：清晰 · 遵从 · 深度
├── 设计理念：一致性 · 信息反馈 · 无障碍
├── 实际用途：组件工具书 · 情景交互指导 · 技术约束参考
├── 平台要求：App Store 审核的合规基准
└── 行业意义：交互设计最经典、最权威的规范之一
```

**给设计师的建议：**
1. 想掌握交互设计基础理论 → HIG 不可错过
2. 做 iOS/iPadOS/macOS 设计 → HIG 是**必读参考**
3. 把 HIG 当做**工具书**，遇到具体问题时查阅对应章节
4. 设计初期就以 HIG 为标准，确保最终产品符合质量要求，更容易通过审核
