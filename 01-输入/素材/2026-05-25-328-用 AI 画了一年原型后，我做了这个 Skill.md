# 用 AI Skill 复刻网站界面与产品经理价值探讨笔记

## 1. AI Skill 演示与功能
### 1.1 使用工具和模型
- 使用 **Cursor** 编辑器中的 **GPT 5.3 Codex** 模型来驱动 AI Skill。
- Skill 名为“圆形绘制 Skill”，用于自动化复刻网站界面。

### 1.2 复刻网站界面的实操步骤
以下为复刻 Bilibili 网站界面的完整步骤：
1. **发送请求**：在 Cursor 中，输入 Bilibili 网站地址，并调用圆形绘制 Skill 复刻首页。
2. **获取内容**：Skill 自动获取页面内容和参考文件，搭建页面结构，耗时约 4-5 分钟。
3. **优化界面**：初始复刻后，样式与原版有差异，通过追加对话要求根据图片优化，并填充网络图片以补全缺图。
4. **输出成果**：优化完成后，Skill 生成复刻界面，并产出设计文档；设计文档可按用户自定义模板修改。

### 1.3 Skill 的能力扩展
- **添加功能**：基于设计文档框架，用户可通过自然语言或图片描述，让 Skill 新增页面功能（如点击热门跳转到新页面）。
- **自定义工程文件**：用户可基于自己负责的产品复刻工程文件，持续添加功能并输出设计文档。

## 2. 开源与自定义
### 2.1 Skill 开源信息
- 该 AI Skill 已在 **GitHub** 上开源，用户可访问 UP 主的仓库查看、修改和优化。
- 开源鼓励社区协作，提升 Skill 的适用性和功能。

### 2.2 用户自定义产品可能性
- **编辑代码权限**：如果用户有编辑代码权限，可以直接基于真实生产环境代码进行设计，无需复刻步骤，进一步提效。
- **CLI 化趋势**：SaaS 产品正走向 **CLI（命令行界面）化**，例如非书（飞书）开源了 CL 能力，允许通过命令行操作产品功能（如发送消息、创建文档）。
- **定制化界面**：用户拿到 CLI 后，可自定义常用界面，替代统一界面操作，实现个性化产品体验。

## 3. 扩展思考：SaaS 产品 CLI 化
### 3.1 非书（飞书）开源 CL 能力
- 非书开源了 CL 能力，使 AI Agent 可通过命令行调用产品功能，例如在飞书中发送消息或创建文档。
- 这增强了自动化集成能力，适合 AI Agent 开发。

### 3.2 对产品操作方式的影响
- CLI 化使产品功能更模块化，用户可基于命令组合自定义工作流。
- 未来，Agent 可能接管代码和产品功能访问，进一步模糊开发与使用边界。

## 4. 产品经理的价值与未来
### 4.1 AI 对产品经理的影响
- AI 不会取代产品经理，但会 **重排产品能力的结构**，自动化表层任务如界面绘制和需求文档生成。
- 核心转变：从“把需求画出来”转向更高阶的能力，如资源分配和结果导向。

### 4.2 核心能力与岗位要求
- **稀缺能力**：判断优先级、定义目标、拿到商业结果，而非单纯的产品功能转化。
- **关键问题**：当前的核心问题是什么、现在做是否正确、做完能否提升用户价值。
- **应要求能力**：洞察、分析、决策、协同和领导力；只要有需求存在，产品岗位就不会消失。
- **未来方向**：UP 主正开发一系列关于产品经理的 Skill，以增强这些核心能力。

## 5. 总结
视频演示了 AI Skill 在复刻网站界面、生成设计文档和扩展功能上的实用价值，并开源促进社区协作。扩展讨论了 SaaS 产品 CLI 化趋势如何赋能用户自定义，以及 AI 对产品经理岗位的影响：AI 将自动化表层任务，而产品经理的核心价值在于洞察、决策和领导力，以驱动商业结果。UP 主计划通过开发更多 Skill 来支持产品经理的能力提升。
