# OpenClaw + Ollama 本地部署！无需 API，断网可用，多模型自由切换（GPT-OSS / Qwen 3 / GLM 4.7）

> [零度解说](https://space.bilibili.com/625267185) | [BV1ykcMzSE6h](https://www.bilibili.com/video/BV1ykcMzSE6h) | 时长 12:01

## OpenClaw + Ollama 本地部署完全指南

### 一、 核心价值与优势
- **完全本地化**：无需API密钥，无需付费，保护数据隐私。
- **断网可用**：所有计算和处理都在本地完成。
- **多模型切换**：支持自由切换不同的本地大模型（如Qwen、GLM、GPT-OSS等）。
- **成本低廉**：仅需下载模型文件，无持续API费用。

### 二、 环境与软件准备

#### 1. 安装 Git
- **作用**：用于后续通过命令行下载软件。
- **步骤**：
    1. 以管理员身份打开 PowerShell。
    2. 执行安装命令（具体命令需参考视频描述区链接获取）。
    3. 安装完成后关闭窗口。

#### 2. 安装 Ollama
- **作用**：本地运行和管理大模型的核心工具。
- **步骤**：
    1. 访问 Ollama 官网（链接在视频描述区）下载最新版安装程序（约1GB）。
    2. 运行安装程序，使用默认路径即可。
    3. 安装完成后，Ollama 会自动启动一个管理面板。

### 三、 选择与下载本地模型

#### 1. 模型选择建议（OpenClaw 官方推荐）
OpenClaw 建议使用 **至少60K上下文长度** 的模型。以下模型与其兼容性良好：
1.  **Qwen Coder (30B)**：针对编码任务优化，适合代码补全、修复、纠错。
2.  **GLM 4.7**：强大的通用型模型，适合聊天、处理文档、文本任务。
3.  **GLM 4.7 Flash**：兼顾性能和速度的模型，适合追求响应速度的场景。
4.  **GPT-OSS 20B**：开源GPT模型，兼顾性能和速度，适合约20GB显存的显卡。
5.  **GPT-OSS 120B**：能力最强的开源GPT模型，但对硬件要求极高（约120GB显存）。

#### 2. 下载模型步骤（以 GPT-OSS 20B 为例）
1.  在 Ollama 官网找到模型，复制其下载命令。
2.  以管理员身份打开 CMD（命令提示符）。
3.  粘贴并执行下载命令（模型文件约13GB）。
4.  等待下载完成。

### 四、 安装与配置 OpenClaw

#### 1. 安装 OpenClaw
1.  复制针对 Windows 系统的最新版安装命令（链接在视频描述区）。
2.  在 PowerShell 中执行该命令。
3.  安装完成后，直接跳过其他设置。

#### 2. 配置 OpenClaw 以连接本地模型
1.  安装完成后，执行一条命令（如 `ollama run` 类命令）来调整 OpenClaw 使用 Ollama 本地模型。
2.  系统会列出已下载的本地模型（如 GPT-OSS 20B、Qwen Coder 30B 等），使用光标选择并回车确认。

#### 3. OpenClaw 初始化设置
- **条款协议**：选择 “Yes” 同意。
- **配置选项**：选择第一个 “单用户配置”。
- **对接第三方客户端**（可选）：
    - **Telegram**：适合海外用户，可创建 Telegram 机器人进行对话。
    - **WhatsApp / Discord** 等：其他可选平台。
    - **跳过**：可选择暂不配置。
- **管理技能**：选择 “NPM” 方式。

### 五、 使用演示与进阶操作

#### 1. 基础使用
- 配置完成后，OpenClaw 会启动一个本地网页界面（Open WebUI）。
- 在网页中可以直接与本地模型对话，例如提问、要求编写代码等。

#### 2. 对接 Telegram 机器人（示例）
1.  **创建机器人**：在 Telegram 中搜索 “BotFather”，使用 “/newbot” 命令创建机器人，获取 **Token**。
2.  **配置配对**：在 OpenClaw 设置中输入机器人 Token。
3.  **完成配对**：在 Telegram 中向新机器人发送消息，会获得一个配对码。在 PowerShell 中执行命令将配对码输入，显示 “approved” 即表示成功。
4.  **使用**：之后即可在 Telegram 中直接与本地模型对话，例如查询天气、编写代码。

#### 3. 如何切换已下载的本地模型
1.  找到 OpenClaw 的配置文件（位于 `C:\Users\用户名\.openclaw\openclaw.json`）。
2.  用编辑器打开此 JSON 文件。
3.  修改 `"model"` 字段的值为目标模型名称（如将 `gpt-oss:20b` 改为 `qwen3:30b-vl`）。
4.  保存文件。
5.  重启 OpenClaw 网关服务（在 PowerShell 中执行相关命令）。
6.  重新打开网页界面，模型即切换成功。

### 六、 核心要点总结
本视频教程详细介绍了如何利用 Ollama 和 OpenClaw 搭建一个完全本地、断网可用的AI助手系统。核心流程包括：安装 Git 和 Ollama 环境，根据自身硬件（尤其是显存）选择合适的开源模型（如 Qwen、GLM、GPT-OSS 系列）并下载，然后安装 OpenClaw 并将其配置为使用 Ollama 本地模型。用户可以通过 OpenClaw 自带的网页界面或配置第三方客户端（如 Telegram 机器人）进行交互。整个过程无需API密钥和联网，实现了数据的完全本地化处理，并能通过修改配置文件轻松切换不同的本地模型，兼顾了隐私安全、灵活性与功能性。
