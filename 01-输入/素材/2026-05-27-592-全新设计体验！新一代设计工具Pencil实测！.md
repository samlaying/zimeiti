# 全新设计体验！新一代设计工具Pencil实测！

> [噗噗制造](https://space.bilibili.com/211541110) | [BV17T6JBDEkK](https://www.bilibili.com/video/BV17T6JBDEkK) | 时长 1:45

## 笔记正文

### 1. Pencil工具概述
- **定义与特点**：Pencil 是一个完全区别于 Figma 的下一代 UI 设计工具，由提示词驱动。它通过左下角的提示词输入框输入需求，全程直播看 AI 画图，过程从上到下依次搭建设计元素，工整解压，没有多余操作。
- **核心优势**：基于 AI 模型能力，审美取决于所使用的模型（如 Cloud Code API 或 GEM），可以调用其他模型提升效果。
- **使用方式**：推荐使用 Pencil 客户端接入 Cloud Code API，体验更丝滑；也可以通过 Cursor 添加插件在 IDE 工具内调用，但客户端更顺畅。

### 2. 测试案例详解
#### 2.1 案例一：机器人产品官网设计
- **操作过程**：使用 Pencil 客户端接入 Cloud Code API，输入需求后，AI 自动从上到下绘制页面元素，过程直播，条理清晰。
- **效果评估**：页面与 Figma 相似但功能较少，产出速度快，审美基于模型能力（如使用 GEM 效果更好）。
- **示例数据**：整个页面绘制完成，元素工整，过程解压。

#### 2.2 案例二：商城应用多页面绘制
- **操作过程**：绘制多个页面，如商城应用，AI 画图中间不带停，更加丝滑。
- **效果评估**：整体页面功能复杂，但产出内容可用，强调 UI 审美依赖于模型能力，可调用其他模型优化。
- **示例数据**：多个页面绘制流畅，内容复杂但实用。

#### 2.3 案例三：在 Cursor 内使用插件绘制 To-Do list
- **操作过程**：在 Cursor 内添加 Pencil 插件，输入提示词绘制 To-Do list 首页。
- **问题发现**：绘制完成后，内容被挡在空白画板下面，布局有小问题。
- **修改步骤**：
  1. 使用 Command + C 复制错误内容。
  2. 使用 Command + V 粘贴到正确位置。
  3. 通过描述出错内容（如“指哪打哪”），AI 快速修正。
- **示例建议**：修改方便，但客户端体验更佳，避免小问题。

### 3. 使用建议与体验优化
- **推荐工具**：优先使用 Pencil 客户端接入 Cloud Code API，以获得更丝滑的体验。
- **问题处理**：遇到布局或内容遮挡时，通过复制粘贴和描述错误快速修改。
- **社区交流**：可加入粉丝群与 UP 主交流，获取更多支持和技巧。

### 4. Pencil 的意义与行业影响
- **核心意义**：Pencil 不仅是设计能力的提升，更是 AI 时代下新的开发工作流的体现，它将模糊设计、产品、开发各职能的界限，推动角色定位变化。
- **行业趋势**：AI 工具如 Pencil 将重新定义 UI 设计流程，提高效率，促进跨职能协作。

### 5. 总结核心要点
Pencil 作为提示词驱动的下一代 UI 设计工具，通过 AI 自动化绘图提升了设计效率，测试案例显示其在实际项目中具有可用性，但审美效果依赖模型能力。推荐使用客户端接入 API 以获得最佳体验，同时该工具象征着 AI 时代工作流的变革，将模糊设计与开发的界限，建议用户尝试并参与社区交流以优化使用。
