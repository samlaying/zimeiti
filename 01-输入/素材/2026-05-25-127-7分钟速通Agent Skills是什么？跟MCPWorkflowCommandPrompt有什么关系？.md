# 7分钟速通Agent Skills是什么？跟MCP|Workflow|Command|Prompt有什么关系？

> [小白debug](https://space.bilibili.com/302188068) | [BV162cPzhEGU](https://www.bilibili.com/video/BV162cPzhEGU) | 时长 7:31

## 笔记正文

### 一、 核心概念解析

#### 1. Prompt（提示词）
*   **定义**：让AI执行任务的最简单指令。
*   **问题**：简单的自然语言提示词，AI输出的结果可能不符合预期（如格式、内容）。
*   **解决方案**：使用**结构化提示词**，即详细规定步骤和规则的提示词。例如，在写简历时，要求“个人信息要居中，教育经历用表格，项目经历用STAR法则”。

#### 2. Command（命令）
*   **定义**：为了方便重复输入长提示词，将其固化，并通过类似快捷键的短命令快速调用的功能。
*   **示例**：AI客户端（如Cloud Code）支持将常用提示词保存为文件，通过短命令一键输入。

#### 3. System Prompt（系统提示词）
*   **定义**：优先级高于用户输入提示词（User Prompt）的指令，能显著提升大模型的指令遵循效果。
*   **用户接入方式**：通常，用户在输入框输入的内容都是“用户提示词”。一些AI软件（如Cursor的`CursorRule`， Cloud Code的`Cloud.md`）允许用户通过特定文件（如`CLAUDE.md`）来写入“系统提示词”。

#### 4. Metadata（元数据）
*   **问题**：将多个不同场景（写简历、周报、邮件）的系统提示词放在一个文件里不合适；全部加载又会消耗大量Token。
*   **解决方案**：将不同场景拆分为独立文件，并在每个文件开头添加一段简短的**元数据**，用于描述该文件的功能和使用条件。
*   **工作流程**：
    1.  用户输入请求（如“帮我写个简历”）。
    2.  AI客户端将所有文件的**元数据**发给模型（Token消耗少）。
    3.  模型根据元数据判断所需场景，返回指令。
    4.  AI客户端加载对应文件的**完整内容**作为系统提示词，再发送给模型生成内容。

#### 5. Reference（参考资料）与 Script（脚本）
*   **渐进式披露**：为应对需求细化，可将文件继续拆分（如“写简历.md”按岗位拆分）。AI客户端通过系统命令（如`cat`）按需读取文件，避免一次性加载所有内容，进一步节省Token。
*   **Reference（参考资料）**：存放被拆分出的详细资料文件。模型有需要时，AI客户端可读取这些文件供其参考。
*   **Script（脚本）**：存放可执行的代码脚本（如Python脚本）。通过提示词指定触发条件，AI客户端便能执行本地脚本，实现生成PDF等扩展功能，突破纯文本聊天限制。

### 二、 Skill 的本质与定义

*   **构成**：将上述所有要素打包而成的**文件夹**。
    *   主提示词文件（如`skill.md`）
    *   元数据
    *   参考资料文件夹（`references`）
    *   脚本文件夹（`script`）
*   **本质**：**一个以文件夹形式存在、可动态加载的系统提示词包**。它代表了一套完整的、针对特定任务的操作经验。

### 三、 Skill 完整工作流程（以Cloud Code为例）

1.  **安装**：将Skill文件夹放入Cloud Code的指定目录。
2.  **触发**：用户在聊天框输入指令（如“帮我写个简历PDF”）。
3.  **识别**：Cloud Code加载所有本地Skill文件夹中的**元数据**，发送给大模型。
4.  **路由**：大模型根据元数据判断应使用哪个Skill，并返回指令。
5.  **加载**：Cloud Code将目标Skill文件夹的**主提示词文件**加载为系统提示词，发送给大模型。
6.  **执行**：大模型根据提示词指示，让Cloud Code按需**读取参考文件**或**执行脚本**（如生成PDF）。
7.  **返回**：大模型基于所有信息生成最终结果，返回给用户。

### 四、 概念关系辨析

#### 1. Skill 与 MCP 的关系
*   **MCP**：是大模型与外部工具交互的协议，相当于给大模型（大脑）配备的“手”，MCP插件是手上的“工具”。
*   **Skill**：是**使用工具的经验**。它规定了在什么场景下、按什么顺序、组合使用哪些工具（工具可以是MCP插件，也可以是本地Script脚本）。
*   **关系**：**Skill编排并驱动MCP等工具**。

#### 2. Skill 与 Workflow 的关系
*   **Workflow（工作流）**：通过规则（如低代码工具N8N）预先编排和调度多个步骤的流程。**流程结构在设计阶段确定**。
*   **Skill**：本质上也进行逻辑编排，但其**执行流程由大模型在运行时驱动**，灵活性更高。
*   **关系**：可以理解为 **“大模型驱动的、更灵活的工作流”**。

### 五、 核心要点总结

视频系统性地梳理了从基础**Prompt**到高级**Skill**的演化路径。为了提升AI指令遵循效果，引入了**System Prompt**；为了解决系统提示词的管理、复用和Token消耗问题，发展出了**Command**、**Metadata**、**Reference**和**Script**等机制；最终，将这些机制整合为**Skill**——一个以文件夹为载体、包含动态系统提示词和辅助资源的智能体技能包。**Skill的本质是封装了操作经验的、由大模型驱动的智能工作流**，它与提供工具能力的**MCP**是“经验”与“工具”的关系，与预设固定流程的**Workflow**是“动态驱动”与“静态配置”的关系，共同构成了构建高效AI应用的工具链。
