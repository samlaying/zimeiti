# 我把泄露的ClaudeCode源代码做成了桌面端！开源免费！手机控制+自定义模型+Computer use + 多Session控制

> [程序员阿江-Relakkes](https://space.bilibili.com/434377496) | [BV1kqdxBEEoe](https://www.bilibili.com/video/BV1kqdxBEEoe) | 时长 5:27

## 背景与动机
- UP主（程序员阿江-Relakkes）之前将泄露的Cloud Code源代码本地化并开源到GitHub，视频播放超过7000次，仓库获得6000多个star，获得社区支持。
- 基于此开发了一个桌面端应用，目的是解决官方Cloud Code的限制，提供开源免费、更灵活的工具，无需Cloud账号，下载软件并配置模型即可使用，支持国内各种模型。

## 官方桌面端的问题
- **封号严重**：官方对网络、订阅方式监管严格，UP主已被封三个账号，用户容易踩雷。
- **模型限制**：只能使用官方自家模型，无法自定义。
- **功能受限**：许多功能需要订阅付费才能解锁。
- **手机远程控制限制**：官方手机控制加了很多限制，体验不佳。
- **KYC实名认证**：Anthropic最近要求上传身份证、护照或驾照进行实名认证，国内用户完全被拦在门外。

## 自定义桌面端的功能概述
- 开源免费，无需Cloud账号，只需配置模型（如Token）即可使用。
- **源代码规模**：桌面端代码（包含测试）约5万行，全部由Claude 4.6和GPT 5.4生成，无手写代码。
- **主要功能列表**：模型自由、手机远程控制、多Session控制、定时任务、Computer use、UI优化。

### 功能1：模型自由
- 支持自定义模型供应商，包括国内模型，甚至可部署本地模型。
- 支持OpenAI格式，提供灵活配置。
- **例子**：演示中使用官方OP 4.7模型和minimax M2.7模型，分析最近Commit功能，展示模型思考过程和回复。

### 功能2：手机远程控制
- 无需持续启动服务或新开Serving，可跨Session远程下达任务。
- **演示步骤**：
  1. 通过电脑版飞书应用演示。
  2. 使用命令行选择工作目录（如临时目录tempo）。
  3. 在手机上选择项目，下达任务（如查询模型能力和skills）。
  4. 模型在选定目录下工作，返回结果（如minimax M2.7模型输出skills列表）。

### 功能3：多Session控制
- 支持同时新开多个Session进行聊天，官方后期也支持，但此桌面端已实现。
- **使用方式**：在界面中新开多个Session，分别聊天，操作简单。

### 功能4：定时任务
- 增强功能，可新建定时任务并设置参数。
- **设置步骤**：
  1. 选择权限（如全部）。
  2. 选择模型和工作目录。
  3. 定义任务名称、描述和提示词。
  4. 设置运行频率。
- **玩法**：可触发skills监控AI新闻等自动化流程。
- **管理功能**：支持增删改查任务，查看运行日志和摘要，对应对话记录。

### 功能5：Computer use
- 原代码中无此功能，桌面端补齐并支持。
- **演示**：创建任务管理应用，选择技术栈后，Agent自动执行计划、调用Skills、写代码，最终完成任务。

### 功能6：UI优化
- 提供可视化页面，操作更方便，支持多Session开窗、任务管理等。
- **例子**：界面类似CLS但更直观，便于Agent交互和任务监控。

## 实操步骤与资源
- **仓库地址**：开源仓库提供所有代码（包括Cloud Code原代码和桌面端代码），具体地址见视频描述或评论区。
- **下载使用**：
  1. 从release页面下载各版本（支持Windows、Mac等）。
  2. 配置模型：支持自定义供应商，添加后即可使用。
- **文档和教程**：
  1. 仓库中提供文档，按步骤操作。
  2. 网络问题可在飞书下载版本（苹果和Windows电脑版本）。
  3. 飞书提供机器人配置教程，逐步指导。
- **建议**：查看评论区获取更多帮助和资源。

## 总结核心要点
- 这款基于泄露Cloud Code源代码开发的开源桌面端应用，解决了官方版本的封号、模型限制和功能受限问题，提供模型自由、手机远程控制、多Session、定时任务、Computer use等实用功能，代码由AI生成，易于下载配置，资源丰富，适合开发者和用户提升效率和自动化工作流。
