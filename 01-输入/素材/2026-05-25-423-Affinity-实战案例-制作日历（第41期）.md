# Affinity-实战案例-制作日历（第41期）

> [WaveF](https://space.bilibili.com/16347141) | [BV1jKFTzaEbM](https://www.bilibili.com/video/BV1jKFTzaEbM) | 时长 25:09

## 视频概述
本期视频由UP主WaveF讲解，通过一个实战案例演示如何使用Affinity Designer的**数据合并（高级数据工具）** 功能，将Excel中的复杂表格数据转换并设计成精美的日历。案例来源于网友需求，核心挑战在于将Excel中多维度、带换行的数据结构，转换为Affinity能识别的扁平化数据格式。

## 一、 核心挑战与解决思路
1.  **问题分析**：
    *   Excel表格数据结构复杂，包含标题、星期、日期、农历、事件（带换行）等多维度信息。
    *   Affinity Designer的数据合并工具仅支持简单的**一维数据结构**（如字段：星期、日期、农历、笔记），无法直接处理Excel的复杂嵌套格式。
2.  **解决思路**：
    *   **数据转换**：需要将Excel数据先转换为扁平化的、每个数据块（每行日历）对应一组固定字段的格式。
    *   **利用AI辅助**：借助AI工具（如文中的Gimini）编写代码，实现数据格式的自动转换。

## 二、 数据准备与转换流程
### 2.1 从Excel到通用文本格式
1.  **导出为CSV**：
    *   在Excel中，将包含日历数据的表格另存为 **CSV（逗号分隔）** 格式的文件。CSV是一种纯文本格式，便于后续处理。
    *   **注意**：导出时需注意编码问题，建议选择 **UTF-8** 编码，以避免后续导入时出现乱码。
2.  **使用AI工具转换数据结构**：
    *   将CSV文件上传至AI对话工具（如文中的Gimini）。
    *   **提供给AI的提示词（关键）**：明确告知AI期望的输出格式。视频中提供的结构化提示词示例为：
        ```
        将数据转换为以下JSON结构：
        [
          {
            "Day": "星期几",
            "Date": "公历日期数字",
            "DateCN": "农历日期",
            "Note": "事件/笔记内容，原文中的换行用\\n表示"
          },
          ...
        ]
        ```
    *   **AI的作用**：根据提示词和CSV原始数据，AI会生成一个转换脚本（HTML页面形式），运行后可将CSV数据转换为结构清晰的**JSON**文件。
3.  **生成并验证JSON数据**：
    *   将AI生成的HTML代码保存为本地文件并用浏览器打开，选择CSV文件进行转换。
    *   保存生成的JSON文件，并用文本编辑器检查数据是否准确、字段是否对应正确。
    *   **处理问题**：若出现乱码，需返回检查CSV文件的编码（确保为UTF-8）；若字段错位，需微调AI提示词或转换逻辑。

## 三、 在Affinity Designer中制作日历模板
### 3.1 创建基础设计文件
1.  新建一个Affinity Designer文件。
2.  **置入背景图**和**标题**（标题中的月份建议后期用页码变量实现动态化）。
3.  保存文件为**模板**（.aftemplate）。

### 3.2 构建数据布局
1.  **分析日历结构**：确定日历表格的行列数（案例中为7列5行，对应每周7天和每月最多5周）。
2.  **绘制数据布局框架**：
    *   使用 **“文本框”** 和 **“矩形”** 等工具，手动绘制出一个**数据单元格**的样例。该单元格包含：
        *   **日期文本框**：显示“星期”和“公历日期”。
        *   **事项文本框**：显示“农历”和“事件笔记”。
        *   **分隔线**、**背景色**等样式元素。
    *   **关键技巧**：将日期和事项文本框用另一个矩形“框”起来，便于整体移动和对齐。
3.  **应用数据布局工具**：
    *   选中构成单元格的所有图层，点击 **“高级数据工具”面板** 中的 **“创建数据布局”** 按钮。
    *   设置布局的**行数**和**列数**，并调整单元格的**宽度**和**高度**，使其与绘制好的样例尺寸匹配。
4.  **完善单元格样式**：
    *   调整文字的对齐、间距、段落行距等。
    *   为文字添加列表样式（项目符号）时，需通过调整**缩进**值来对齐符号和文本。

## 四、 数据绑定与日历生成
### 4.1 绑定字段
1.  **导入数据**：在“高级数据工具”面板中，点击 **“选择数据源”**，导入之前准备好的 **1月.json** 文件。
2.  **绑定字段**：
    *   进入 **“字段”** 选项卡，面板会列出JSON文件中的所有字段（如`Day`, `DateCN`, `Note`）。
    *   **双击**日期文本框（或在文本编辑状态下光标定位处），再双击面板中的字段名（如`Date`、`DateCN`），即可插入字段变量。
    *   在插入的字段变量之间，可以加入空格或全角空格进行分隔。
3.  **预览与检查**：
    *   使用数据合并面板底部的 **“使用记录浏览”** 工具，逐条预览数据绑定效果，检查是否有数据错位或显示不全。
    *   **注意**：若预览时数据为空，说明该数据源记录可能缺失，需检查原始数据。

### 4.2 生成日历
1.  确认绑定无误后，点击 **“生成”** 按钮。Affinity会根据数据源中的记录数，生成对应页数的日历文件。
2.  **重要提示**：生成的是一个**新的文件**。若需修改样式，**强烈建议返回修改原始模板文件，然后重新生成**，而非在生成结果上修改，以避免后续批次生成时样式丢失。

## 五、 进阶调整与脚本应用
### 5.1 去除原始数据中的序号
*   **问题**：原始Excel中的事件列表自带数字序号（如“1.”、“2.”），但Affinity的数据布局工具也会自动生成列表序号，导致重复。
*   **解决方案**：使用 **“高级数据工具”中的脚本** 功能，在数据绑定前去除原始序号。
    1.  在“高级数据工具”面板的“脚本”选项卡，新建一个脚本。
    2.  编辑脚本代码，核心是使用**正则表达式**匹配并替换掉原始数据中类似“1. ”、“2. ”这样的序号文本。
    3.  保存并应用脚本后，数据中的原始序号被清除，只保留Affinity自动生成的动态列表序号。

### 5.2 实现月份动态化
*   **方法**：
    1.  不直接在标题文本框写死“1月”，而是通过 **“文本”>“插入”>“字段”>“页码”** 来插入当前页码变量。
    2.  准备多个月份的JSON数据（可复制并修改单个月份数据以模拟测试），并确保每月的数据量足以填满布局（不足的部分用空数据行占位）。
    3.  重新生成日历，每一页的标题会自动显示对应的页码（即月份）。

## 总结核心要点
本案例的核心在于通过 **“数据格式转换”** 这一关键步骤，打通了从Excel复杂数据到Affinity Designer数据合并工具的桥梁。具体流程为：**Excel → CSV（UTF-8编码）→ 利用AI生成结构化JSON → 在Affinity中创建数据布局模板并绑定JSON字段 → 生成并调整日历**。过程中需注意编码问题、数据结构的扁平化设计，以及善用高级数据工具中的脚本功能进行数据清洗，最终实现高效、可批量更新的个性化日历设计。
