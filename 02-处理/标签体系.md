# 标签体系

## 原子笔记类型标签

| 标签 | 用途 |
|------|------|
| `#类型/方法论` | 有明确操作步骤或框架，可直接指导行动 |
| `#类型/案例` | 完整的实践故事、真实案例，有细节有结果 |
| `#类型/观点` | 有传播价值的判断、洞察，反常识或有深度 |
| `#类型/数据` | 具体的数字、比例、对比，有说服力的论据 |
| `#类型/故事` | 有情节的个人经历或见闻，有代入感 |
| `#类型/金句` | 适合直接引用的高光表达，原汁原味不改写 |

## 领域标签

| 标签 | 适用内容 |
|------|----------|
| `#领域/AI工具/Skills` | Claude Code, Codex, GPT, Gemini 等工具测评和用法 |
| `#领域/编程/技术` | Vibe Coding, 前端开发, 独立开发 |
| `#领域/产品经理` | 产品思维, PRD, 需求分析, API |
| `#领域/设计/PPT` | UI设计, CSS, HTML, Figma, PPT制作 |
| `#领域/自媒体运营` | 内容创作, 博主成长, 视频制作, 变现 |
| `#领域/知识管理` | Obsidian, 笔记方法, 信息整理 |
| `#领域/浏览器自动化` | Playwright, RPA, 网页操作 |
| `#领域/视频制作` | 视频制作, 剪辑, TTS, 动画 |
| `#领域/理财/成长` | 理财知识, 个人成长, 职业发展 |

## 通用标签

| 标签 | 用途 |
|------|------|
| `#原子笔记` | 所有原子笔记的通用标记 |

## 标签使用规则

1. 每条笔记**必须**有一个 `类型/*` 标签
2. 每条笔记**必须**有一个 `领域/*` 标签
3. 所有原子笔记都带 `#原子笔记` 标签
4. 标签写在 frontmatter 的 `tags` 字段中
5. 一条笔记只属于一个领域，但可以从多个逐字稿中提取

## 4种内容变形

| 变形类型 | 结构 | 适用平台 |
|---------|------|---------|
| 观点文 | 反常识观点 → 案例/数据佐证 → 你的立场 | 公众号/即刻/小红书 |
| 实操教程 | 遇到问题 → 解决方案（带截图） → 可抄步骤 | B站/公众号 |
| 信息差/对比测评 | N个同类对比 → 推荐 → 适用场景 | B站/公众号 |
| 复盘/思考 | 做了什么 → 结果 → 学到什么 | 公众号/即刻 |
