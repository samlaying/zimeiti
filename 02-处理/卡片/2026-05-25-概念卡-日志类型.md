# 日志类型

在自动化流程（如RPA）中，用于记录程序运行状态的`打印日志`可以被划分为四种类型，以便于管理和快速识别：

1.  **调试（Debug）**：用于开发调试阶段的详细信息。
2.  **信息（Info）**：记录流程的一般运行状态和进度。
3.  **警告（Warning）**：标识潜在的问题或非预期但不影响继续运行的情况。
4.  **错误（Error）**：记录导致流程中断或失败的严重故障。

不同的日志类型在输出时通常会带有颜色或标签，实现视觉区分，是进行[[通过日志类型快速定位日志]]的基础。
