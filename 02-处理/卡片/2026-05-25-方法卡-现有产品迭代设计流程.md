# 现有产品迭代设计流程

适用于为已有代码的现有产品创建或提取设计规范，以保证后续迭代的风格一致性。

**操作步骤：**
1.  **分析现有产品**：让[[AI工具]]分析现有项目的源码。
2.  **选择或生成规范**：AI根据分析结果，从开源[[AI设计资料库]]中匹配合适的`design.md`文件；若无匹配，则自动生成一个新的`design.md`文件。
3.  **保存与验证**：将生成的`design.md`文件保存在项目文件夹下，同时生成明亮模式（preview）和暗黑模式（previewduck）的预览文件。
4.  **指导迭代**：后续开发新功能或页面时，基于这个生成的`design.md`文件向AI提需求，以提高设计还原度。
