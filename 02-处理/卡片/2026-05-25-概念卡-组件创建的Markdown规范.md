# 组件创建的Markdown规范

为了让AI工具（如[[Claude]]）能准确执行任务，需要提前创建一份 **Markdown格式的规范文档**，作为AI的“操作手册”。该文档应详细定义组件创建的所有规则，通常包含以下模块：

1.  **页面布局规范**：定义组件页面的整体结构、标题层级和对齐规则。
2.  **组件变体规范**：明确变体状态（如默认、禁用、聚焦）的命名规则。
3.  **变量引用规范**：强制要求使用项目中的[[文本变量]]、[[颜色变量]]、[[圆角变量]]等，确保设计一致性。
4.  **尺寸与间距规范**：规定具体的数值标准。

**关键技巧**：该文档无需一次性完美，可以在与AI的迭代对话中，将发现的缺失规则反向补充到文档中，使其不断进化完善。
