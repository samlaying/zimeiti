# Skills搜索工作流

一个典型的[[搜索驱动的Skills架构]]下的工作流程如下：

1.  **分析阶段**：理解用户输入和当前需求，确定需要搜索的方向。
2.  **搜索设计系统**：根据[[Skills入口文档.md]]的指引，调用搜索脚本，首先从[[外部数据存储CSV]]中获取整体的设计系统或核心框架信息。
3.  **补充细节搜索**：针对具体元素（如某个按钮的样式、特定的配色），进行更细化的关键词搜索。
4.  **技术质量筛选**：评估搜索结果的技术可行性与质量，确保兼容性和最佳实践。
5.  **输出生成**：将搜索到的所有相关结果组合成一份结构化的设计文档，供AI生成最终输出（如前端代码）。
