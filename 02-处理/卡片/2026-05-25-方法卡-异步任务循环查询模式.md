# 异步任务循环查询模式

在`[[首段视频生成工作流]]`中，调用视频生成插件后，采用**异步任务循环查询**的模式来获取结果。

### 实现流程：
1.  **触发异步任务**：调用“BVidio”插件生成视频，插件立即返回一个`任务ID`，视频在后台开始生成。
2.  **创建无限循环**：添加`循环插件`，设置为“无限循环”。
3.  **循环体逻辑**：
    - **查询**：在循环内使用“任务查询”工具，根据`任务ID`查询结果（视频链接）。
    - **判断**：使用`选择器`节点判断查询到的`视频链接`是否非空。
    - **分支**：
        - 若**不为空**（生成成功），则用`中止循环`跳出。
        - 若**为空**（仍在生成），则继续循环。
4.  **降低查询频率**：在每次循环查询前，添加`定时器`节点（如等待60秒），避免高频查询给服务器造成压力。

### 核心思想：
> 当调用一个可能耗时较长的API时，不要阻塞等待，而是通过轮询（Polling）的方式周期性地检查任务状态，直到任务完成。
