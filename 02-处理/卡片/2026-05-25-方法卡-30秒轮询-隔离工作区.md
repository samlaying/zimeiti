# 30秒轮询+隔离工作区

Symphony 的工作流程基于以下核心机制：

1.  **轮询 Linear**：默认每 30 秒扫描一次 Linear 看板，查找状态为 `to do` 或 `in progress` 且没有未关闭 blocker 的 ticket。
2.  **派发任务**：检测到符合条件的 issue 后，Symphony 会派发一个 Agent 去处理。
3.  **隔离执行**：每个 issue 拥有自己的**专属工作区**，任务在此克隆代码并执行，避免冲突。工作完成后，工作区会保留以便复用。
4.  **任务执行与更新**：Agent（运行 Codex App Server）在工作区自主完成代码修改等任务，并通过工具**自主更新** Linear issue 的状态、添加评论和消耗统计。Symphony 本身只负责调度，不直接写代码或修改 Linear。
5.  **容错机制**：失败后采用**指数退避**策略重试，不会因单次失败而彻底停止。
