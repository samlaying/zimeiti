# SSE事件流

## SSE事件流 (Server-Sent Events)

**定义**：SSE是一种服务器向客户端单向推送事件流的机制。在[[Claude Code]]的场景中，API的返回不是一次性返回完整JSON，而是通过SSE逐步推送数据，从而实现终端“逐字打印”的效果。

**[[Claude Code]]响应的SSE事件流顺序**：
1.  `Message Start`：返回消息基本信息（模型名称、ID、Token用量）。
2.  `Content Block Start`：开启文本块。
3.  `Ping`：心跳，保持连接。
4.  `Content Block Delta`（多个）：模型实际输出的文字片段。
5.  `Content Block Stop`：文本块结束。
6.  `Content Block Delta`（带`stop_reason: end_turn`）：模型主动结束回复。
7.  `Message Stop`：整个消息流结束。
