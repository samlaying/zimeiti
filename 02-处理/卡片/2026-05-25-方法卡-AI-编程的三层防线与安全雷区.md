# AI 编程的三层防线与安全雷区

AI 是极能干但无全局观的执行者。你没写的约束，他会选择最弱配置。5600 个 Web Coding 应用审计发现 2000+ 安全漏洞，AI 生成代码漏洞率是人类的 2.74 倍。

**三层防线**：
1.  **PRD 阶段就写死约束**：用约束替代描述，告诉 AI“什么不许做”比“你想要什么”更有效。
2.  **做的时候持续检查**：必须每一小段代码生成就人工审一遍，错误成本会指数级增长。
3.  **上线前全面体检**：使用 20 多项 Checklist（如密钥检查、环境隔离、数据库备份）进行验证。

**九大安全雷区**（精选致命级）：
- **密钥硬编码**：API Key 直接写在代码里，相当于将银行卡密码贴在门上。
- **客户端认证**：权限判断写在前端，极易绕过。认证必须在服务器端执行。
- **SQL 注入**：40%-62% 的 AI 生成代码含注入漏洞，习惯用字符串拼接而非参数化查询。
- **依赖幻觉**：AI 推荐的 NPM 包可能不存在，已被攻击者注册并植入恶意代码（Slop Squatting）。
