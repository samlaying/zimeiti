# 视频平台服务拆分

构建视频平台时，通常将系统拆分为四个核心[[微服务]]模块：

1. **上传服务**：负责文件上传入口、校验、[[分块上传]]与[[断点续传]]。
2. **转码服务**：负责将上传的原视频处理为多分辨率、多格式的任务，是一个上传任务对应一个主任务，主任务下包含多个子任务（如4K子任务、1080P子任务）。
3. **视频服务**：负责管理视频元数据（标题、封面、描述）并提供视频列表查询接口。
4. **流媒体服务**：在用户播放时，返回不同分辨率的资源地址，支持播放器自动切换分辨率。`

这种拆分实现了业务逻辑的解耦，便于独立开发、部署和扩展。
