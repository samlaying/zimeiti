# 数据类项目的完整工作流程

数据类项目通常包含四个核心环节：
1.  **铲除需求**：产出数据获取/预处理规则，并定义用[[准确率]]和[[召回率]]衡量的标准。
2.  **策略评估**：根据初步规则产出数据，评估召回率和准确率，迭代规则至达标。
3.  **策略上线与效果回归**：上线数据（如标签）用于制定业务策略，并根据业务效果（如转化率、成本）回归，微调策略直至稳定。
4.  **建立监控体系**：对数据指标（如标签占比）和业务效果指标进行长期监控，确保质量稳定。
