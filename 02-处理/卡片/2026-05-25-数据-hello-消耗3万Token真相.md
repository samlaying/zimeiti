# “hello”消耗3万Token真相

> 数字会说话：一次简单交互，背后是海量上下文的“预加载”。

## 核心内容
在Claude Code中输入一句“hello”，模型实际处理的上下文总量约为31000 Token。其中6个Token是用户输入本身，14000个是本次新写入缓存的系统提示词和工具定义，16000个是从已有缓存中读取的上下文。这解释了为什么简单对话Token消耗巨大。

## 原文关键段落
> Input Token是6所以你的hello加上格式标记一共就6个Token但是Cache Creation Input Token是14000这14000是什么是这次新写入缓存的内容...Cache Read Input Token所以是16000...所以你觉得自己只发了5个字但实际上模型实际处理的上下文是6加14加16一共是31000个token。
