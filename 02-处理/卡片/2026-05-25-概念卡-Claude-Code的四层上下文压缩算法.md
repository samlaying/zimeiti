# Claude Code的四层上下文压缩算法

Claude Code 采用四层递进压缩体系管理上下文，比 OpenAI 更优，能保留早期对话细节。其核心原则是：**尽可能用廉价的规则操作，延迟昂贵的大模型调用**。

1. **第一层：微压缩**：纯规则操作，清理旧的工具输出结果，保留语义信息。
2. **第二层：自动压缩**：基于算法自动压缩上下文。
3. **第三层：传统压缩**：使用传统方法进一步压缩数据。
4. **第四层：session memory压缩**：压缩会话记忆，优化长期对话管理。

这种机制使得用户在长时间对话后感觉模型“变傻”，可能源于压缩，而非如[[Claude Code不存在模型偷换]]所述的模型偷换。压缩后的信息也会影响[[Claude Code的三层记忆架构]]的运作。
