# CLI vs MCP路线之争

社区对CLI取代MCP的思路存在路线之争（CLI vs MCP）。

- **支持观点**：CLI通用、简单，适配任何大模型。
- **反对质疑**：懒发现模式（先List再Help）可能降低工具调用的准确率；多一步查询往返是否增加延迟？作者仅展示Token节省数据，未提供准确率对比。
- **行业背景**：社区发现约100个类似项目，说明CLI取代MCP的思路并非独创，目前尚无定论。
