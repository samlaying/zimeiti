# 具体构建步骤与工具链

构建[[构建自动追踪系统的核心功能]]所述系统的具体步骤与技术栈：
1.  **前期规划**：使用 [[Perplexity AI]] 的 Labs 功能生成 Markdown 格式的应用规格文档。
2.  **环境准备**：获取 [[Gemini API]] 和 [[Resend]] 的 API Key。
3.  **核心开发**：将规格文档和 Gemini API 文档提供给 [[Claude Code]]，通过迭代提示（Prompt Engineering）驱动开发。关键提示包括：明确指定模型版本、要求本地缓存（“Cache AI analysis in my SQLite database by version hash...”）、指导UI设计风格。
4.  **技术集成**：集成抓取、AI解析、语音生成（Base64转WAV播放）、邮件发送等功能。
5.  **优化迭代**：移除无关功能（如情绪分析），确保可回溯历史TLDR。

**技术栈**：前端 [[React]]，后端/数据库 [[SQLite]]，AI推理 [[Gemini API]]，邮件服务 [[Resend]]。
