# Harness实战演示案例

**目标**：构建一个浏览器使用智能体，任务是访问Hacker News并给第一篇帖子点赞。
**关键设计**：故意使用能力较弱的模型（GPT-3.5 Turbo），通过逐步增强[[AI Agent Harness 定义|Harness]]来弥补其不足。

**Harness增强演进过程：**
1.  **阶段1（基础循环）**：无Harness，智能体“撒谎”（声称成功但实际失败）。
2.  **阶段2（添加守卫栏）**：引入最大迭代次数、上下文压缩等守卫规则，并重构代码为独立的Harness模块。
3.  **阶段3（添加验证逻辑）**：创建确定性验证函数，检查操作轨迹中的成功/失败模式，使智能体能**正确报告失败**。
4.  **阶段4（注入确定性工具）**：实现登录处理器，在遇到登录页时由Harness自动完成登录（[[确定性操作注入法]]）。

**结果**：仅通过添加和增强Harness，未修改任何提示词，智能体成功完成任务。
