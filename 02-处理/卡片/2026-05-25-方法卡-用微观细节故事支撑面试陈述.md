# 用微观细节故事支撑面试陈述

为了避免AI产品面试“一轮游”，你需要将简历和项目陈述中**宏观的、结果性的描述**，转化为**微观的、过程性的、具备决策依据和落地细节的故事**。
- **核心方法**：系统地重新梳理项目中的每一个关键节点（如技术选型、数据构建、评估设计、迭代复盘）。
- **明确每个节点的**：
  1.  **角色**：你扮演的是决策者还是执行者？
  2.  **决策依据**：为什么这么做？（业务判断）
  3.  **具体行动**：你做了什么？（操作细节）
  4.  **最终产出形式**：产出的数据、文档、流程是什么？
- **目的**：当面试官深挖技术选择背后的业务判断、数据评估体系细节时，你能提供坚实、可信的支撑。
