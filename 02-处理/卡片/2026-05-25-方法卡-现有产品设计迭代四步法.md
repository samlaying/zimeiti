# 现有产品设计迭代四步法

对于一个**已有代码的现有产品**，可以通过生成或匹配 `design.md` 来指导后续迭代，提高设计还原度。步骤如下：

1.  **分析源码**：让AI分析现有项目的源码。
2.  **匹配或生成文件**：让AI从开源项目中匹配合适的 [[AI设计规范文档]]；若无合适文件，则基于分析结果生成一个新的 `design.md` 文件。
3.  **保存与预览**：将文件保存在项目文件夹下，并生成明亮模式（preview）和DUG模式（previewduck）的预览文件。
4.  **迭代生成**：后续产品迭代时，基于这个 `design.md` 文件生成新页面，确保风格一致。

生成的文件会包含产品气质、色彩、字体、组件样式规范以及设计约束等详细信息。
