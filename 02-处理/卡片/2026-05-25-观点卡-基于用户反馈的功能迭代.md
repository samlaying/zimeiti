# 基于用户反馈的功能迭代

产品功能设计应以用户直觉和使用反馈为导向，有时需要勇于“改回去”。

**案例**：Webcoding网站中“状态原点和名字”的设计迭代。
-   **初始设计**：状态原点和任务名称的功能耦合在一起。
-   **用户反馈**：有用户留言表示，更喜欢用独立的原点来切换完成状态，而任务名称应保持中立。
-   **迭代决策**：设计者接纳了这一反馈，将功能重新分离。**小原点专门用于切换学习状态（如完成/未完成），任务名字则保持其本质含义不变**。

**启示**：优秀的设计不是一成不变的。基于真实用户的反馈进行调整，让功能更符合用户的心理模型，即使这意味着要推翻自己之前的某些设计，也是必要的进化。这种反馈驱动的迭代能显著增强产品的直观性和易用性。
