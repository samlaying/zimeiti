# 交互细节结构化描述

将页面中的交互元素（如输入框、按钮、下拉框）使用表格进行结构化描述，强制逻辑严谨，避免遗漏关键细节。

**表格应包含的核心要素（以“手机号”为例）：**
1. **字段**：手机号
2. **是否必填**：是。*（有必填就要有必填性校验，防止自己遗漏）*
3. **样式**：文本框，Placeholder为灰色提示文字“请输入手机号”。
4. **逻辑**：默认为空，逻辑简单。
5. **校验与报错**：
   - 必填校验：未填写则提示“请填写手机号”。
   - 格式校验：要求为中国大陆11位纯数字，且开头为1；不符则提示“请填写正确的手机号”。
   - 关联校验：（针对验证码）需校验与手机号的匹配关系，错误则提示“验证码错误”。
6. **备注**：可选。

这种方法能将点状的需求思考，转变为系统性的逻辑梳理。
