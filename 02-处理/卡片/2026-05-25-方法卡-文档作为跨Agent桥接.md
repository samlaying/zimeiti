# 文档作为跨Agent桥接

作为“[[Harness工程]]”的第二大原则，核心思想是**Agent之间不依赖人类作为“传声筒”**，而是通过结构化的本地文档进行高效、充分的信息传递与对齐。

**关键文档类型**：
- **记忆文档**：每个Agent维护自己的 `cloud.md` 或 `agents.md`，记录当前进展，是其“长期记忆”的索引。
- **项目契约文档**：定义项目的总目标、核心功能、数据结构等顶层设计，是全体成员的“宪法”。
- **协作交付文档**：如《开发建议书》、《审计报告》、《开放议题文档》等，用于任务规划和问题传递。
- **接口文档**：后端维护独立的Markdown接口文档，供前端快速调用，避免反复阅读源码消耗上下文。
