# Git分支工作流：大树与树枝

为了安全高效地进行开发，应遵循基于[[版本控制与Git]]的分支工作流。可将项目想象成一棵大树：

- **主干 (`main` branch)**：代表项目的稳定、可运行版本。
- **新枝 (`new branch`)**：为开发新功能或修复Bug而创建的独立分支。

**正确工作流**：
1.  **创建分支**：在开发任何新功能或修复前，从主干创建一个新分支。
2.  **开发与提交**：在新分支上进行开发，并定期**提交 (commit)** 保存进度。
3.  **推送备份**：将分支**推送 (push)** 到云端（如GitHub）进行备份。
4.  **合并与清理**：功能完成且测试通过后，将分支**合并 (merge)** 回主干，并删除已合并的临时分支。

这种流程确保了主干代码的稳定性，是“[[区分初学者与进阶者的关键意识]]”的具体体现。即使代码由AI生成，开发者也应主动指导AI执行此流程。
