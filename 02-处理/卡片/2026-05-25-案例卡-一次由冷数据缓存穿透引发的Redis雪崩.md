# 一次由冷数据缓存穿透引发的Redis雪崩

## 事件：Redis集群雪崩导致APP白屏
- **背景**：[[Redis集群]]（三主三从哨兵模式）在约3万用户并发访问时，APP出现白屏。
- **触发点**：运营活动“新年节预热”推送后，用户点击打开率是预期的3倍，大量新用户涌入。
- **核心过程**：
  1. 新用户请求大量[[冷数据]]，导致缓存未命中率从2%暴跌至35%，发生**缓存穿透**。
  2. 请求穿透至MySQL，数据库被打垮。
  3. Redis因内存爆满（OOM）**主节点自动重启**，缓存清空，引发二次雪崩，所有请求再次压垮数据库。
- **根本缺陷**：系统缺乏缓存预热机制和应对突发流量的预案。
- **解决**：产品经理浩宜提出快速修复方案，包括调整Redis淘汰策略、数据库[[令牌桶限流]]和前端指数退避重试。
