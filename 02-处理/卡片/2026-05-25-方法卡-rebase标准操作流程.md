# rebase标准操作流程

在 GitGraph 等可视化工具中，使用 rebase 跟进主分支的标准操作步骤如下：

1.  **切换分支**：确保当前位于需要变基的功能分支上。
2.  **发起变基**：右键点击目标主分支（如 `main`）的标签，选择 **`Rebase Current Branch on Branch`**。
3.  **确认并解决冲突**：点击 Yes 确认。系统会将功能分支的节点逐一复制到主分支顶端，**每个节点可能都需要单独解决冲突**。
4.  **强制推送**：由于变基后本地分支历史与远程分支历史不一致，必须使用强制推送更新远程仓库。右键功能分支，选择 `Push Branch`，勾选 **`Force With Lease`**（更安全）或 `Force`，点击 Yes。
