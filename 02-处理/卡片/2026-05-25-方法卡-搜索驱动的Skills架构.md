# 搜索驱动的Skills架构

为解决大量Skills内容塞入导致性能下降的问题，一种有效的工程化架构是**搜索驱动**。其核心是将内容外部化存储，并通过算法按需检索，而非静态加载。

该架构通常包含三个核心层：
1.  **入口层 ([[Skills入口文档.md]])**：定义触发规则与核心原则。
2.  **数据层 ([[外部数据存储CSV]])**：将样式、脚本等细节数据存储于外部文件（如CSV）中。
3.  **检索层**：使用搜索脚本（如基于BM25算法）从数据层动态匹配信息，并组合成文档返回给AI。

此架构参考了UI UX Pro Max Skills的设计，成功解决了[[Skills数量黄金区间]]下的扩展性难题。
