# MCP服务的冗余信息问题

**MCP服务**（Model Context Protocol）通常封装多个工具。当前的大模型使用模式是：当需要调用某个MCP服务时，会将其下**所有工具**的完整描述（名称、参数、格式、用途等）一次性加载到模型的上下文中。

这种做法的问题在于，对于用户下达的具体任务，通常只需要调用其中2-3个工具。加载全部工具描述（可能几十甚至上百个）会产生大量**完全用不到的冗余信息**，这是一种“信息全量预加载”策略。
