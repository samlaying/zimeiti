# 规范驱动开发（SDD）

**SDD（Specification Driven Development，规范驱动开发）** 是体现高含金量[[Vibe Coding 能力定义]]的正确工作范式，避免产出难以维护的“史山代码”。其步骤如下：

1.  **制定规范**：在编码前，产出清晰的**PRD（产品需求文档）** 与**原型**。
2.  **方案定稿**：PRD与原型必须经过确认并定稿。
3.  **正式开发**：在规范体系下按顺序开发：**先前端**（确认界面） -> **再后端**（开发逻辑）。
4.  **质量保障**：开发流程中必须包含**自动化测试**环节。

产品经理需遵循此规范来驱动和管理开发过程。
