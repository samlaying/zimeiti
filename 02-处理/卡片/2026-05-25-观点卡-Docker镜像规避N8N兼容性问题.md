# Docker镜像规避N8N兼容性问题

在部署 [[n8n自动化工作流]] 时，**推荐使用UP主提供的Docker镜像**来安装运行N8N。

这是因为N8N 2.0版本中，对于代码调用、文件保存到外部文件夹等操作做了一些限制，需要额外配置。使用提供的镜像可以避免这些兼容性问题，开箱即用。
