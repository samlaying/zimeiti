# Skills数量黄金区间

Skills（技能）的数量并非越多越好，存在一个效率最优区间。

根据LongChain等研究报告，Skills数量**最多不应超过12个**，而**最佳推荐范围是2到4个**。超过这个限制，模型的触发概率和性能效率会**急剧下降**。这是设计一个高效的[[搜索驱动的Skills架构]]的首要约束。
