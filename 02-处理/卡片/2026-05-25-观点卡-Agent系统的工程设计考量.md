# Agent系统的工程设计考量

构建一个可用的Agent系统，除了核心逻辑，**关键的工程设计细节**同样决定成败。

### 主要设计点
- **上下文管理**：对话历史是动态的，需要定期压缩（如用小模型总结）和裁剪旧内容，为模型保留关键信息。
- **Human-in-the-Loop实现**：在等待用户授权时，设计应避免系统卡死。可采用开启新对话或事件触发的方式，而非简单轮询。
- **错误处理**：建立错误兜底机制，并可通过Hooks添加附加任务（如报错时发送通知）。
- **扩展性设计**：通过`vendor`文件夹管理工具二进制文件，实现版本跟踪和自定义扩展。
