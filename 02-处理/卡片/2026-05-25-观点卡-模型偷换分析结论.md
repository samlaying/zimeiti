# 模型偷换分析结论

通过对泄露的源代码分析，**Claude Code 客户端不存在偷换模型的行为**，主对话模型完全由用户控制。

- **核心控制权**：用户手动设置模型（如通过 `--model` 参数指定 Opus 4.6）后，该设置将在整个对话中保持不变，源码中无偷换功能。
- **后台模型使用**：Claude Code 会使用弱模型（如 Hiku）执行后台辅助任务（如摘要生成），但这不会替代主对话模型。
- **“变傻”的可能原因**：用户感知的模型能力下降，可能源于 [[Claude Code 四层上下文压缩算法]] 导致的信息丢失，或“Effort降级”（降低思考级别，而非更换模型），而非客户端偷换。

该结论驳斥了社区中关于模型被偷偷替换的猜测。
