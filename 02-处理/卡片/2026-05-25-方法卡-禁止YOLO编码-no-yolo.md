# 禁止YOLO编码 (no-yolo)

`no-yolo`是[[Superpowers Skills 框架]]中关于编码规范的核心技能之一。
- **含义**：禁止“YOLO”（一次性生成全部代码）式的开发。
- **要求**：AI必须采取**小步、模块化**的方式进行开发和提交。将大任务拆解为小模块，逐步编码、逐步验证。
- **理念**：这直接体现了[[“慢就是快”]]的理念，通过控制单次变更的范围，降低出错概率，便于定位问题和进行代码审查。
