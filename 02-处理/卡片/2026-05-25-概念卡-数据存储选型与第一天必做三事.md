# 数据存储选型与第一天必做三事

根据产品类型选择数据存储：
- **MVP 快速验证**：用 Supabase（内置 Auth，行级安全 RLS）。避免自建 MySQL。
- **实时协作产品**：用 Firebase。避免自建 PG + WebSockets。
- **内容型产品**：用 PostgreSQL + 对象存储。避免将图片存入数据库。
- **高并发场景**：用 Redis 缓存 + PostgreSQL。避免全打一个数据库。

**第一天必做三件事**：
1.  **行级安全(RLS)**：确保用户 A 永远看不到用户 B 的数据。
2.  **Migration 可回滚**：数据库变更用脚本管理，每条都能撤销。
3.  **自动备份**：每天全量备份，每小时增量备份，存于异地。“等以后再说”是创业者最大的自我欺骗。
