# 从物理机到MicroVM的演进

理解AI Agent沙箱技术选型，关键在于厘清从物理机到虚拟机、容器、再到MicroVM的演进逻辑：
1.  **物理机**：硬件资源丰富，但无隔离。
2.  **虚拟机 (VM)**：通过虚拟化软件将资源切分，每台VM拥有**独立操作系统和资源**，安全但**太重、启动慢**。
3.  **容器 (Docker)**：在VM基础上进一步打包应用与依赖，**启动快**，但多个容器**共享宿主机内核**，存在安全漏洞（如内核逃逸）风险。
4.  **MicroVM**：精简版VM，保留**独立内核的安全性**，同时大幅提升启动速度，成为平衡点。

[[沙箱是隔离环境]]的选型即围绕这些特性展开。
