# 版本控制与Git

**版本控制 (Version Control)** 是一种记录文件内容变化，以便将来查阅特定版本历史的系统。在软件开发中，它是管理代码变更的核心实践。

**Git** 是目前最流行的分布式版本控制系统。它的核心操作包括：
- **提交 (commit)**：保存当前工作区的快照到本地仓库。
- **推送 (push)**：将本地仓库的提交上传到远程仓库（如GitHub），进行云端备份。
- **拉取 (pull)**：从远程仓库下载最新变更并合并到本地。
