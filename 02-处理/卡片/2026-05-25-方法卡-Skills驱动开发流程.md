# Skills驱动开发流程

集成 **Superpowers Skills** 后，一个严谨的AI辅助开发流程如下，体现了[[“慢就是快”的AI编程哲学]]：

1.  **需求规划**：根据指令，创建或更新`requirements.md`（`workflow-requirements-document`）。
2.  **理解与探索**：AI必须先深入阅读和理解现有代码库，禁止盲目修改（`explore-before-act`）。
3.  **制定计划**：明确实现步骤和需要修改的文件。
4.  **编码执行**：小步快跑，模块化开发（`no-yolo`）；遵守编码标准（`coding-best-practices`）；在Git分支上开发（`using-git-branches`）。
5.  **运行验证**：修改后自动运行测试，确保无回归错误（`running-and-submitting`）。
6.  **代码审查**：完成基本功能后，主动请求用户进行审查（`requesting-code-review`）。
7.  **最终提交与总结**：通过验证后规范提交代码，并规划下一步（`next-steps`）。

整个流程强制AI“先探索、后规划、再小步实现并持续验证”。
