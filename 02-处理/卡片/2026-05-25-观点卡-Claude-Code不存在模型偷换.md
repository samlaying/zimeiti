# Claude Code不存在模型偷换

通过对源代码的分析，得出结论：**Claude Code 不存在偷换模型的行为**。主对话模型完全由用户控制，不会被偷偷降级。其变“傻”通常与[[Claude Code的四层上下文压缩算法]]等因素有关，而非客户端偷换模型。

- **主对话模型**：用户通过 `--model` 参数手动设置，在对话中保持不变。
- **后台弱模型**：仅用于辅助任务（如摘要生成），不替代主对话模型。
- **过载场景**：触发条件极其严格，切换可忽略不计。
- **Effort降级**：不换模型，仅降低思考级别（如从 max 到 high）。
