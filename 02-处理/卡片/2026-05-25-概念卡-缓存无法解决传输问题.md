# 缓存无法解决传输问题

OpenAI API 的缓存机制可以优化**计算量**，但无法减少需要**传输的数据量**。

- **缓存作用**：通过计算历史记录的哈希值，识别重复前缀，减少模型计算量（让生成首个token更快更便宜）。
- **缓存局限**：**你依然需要每次发送完整的历史记录**给服务器，服务器端才能进行哈希匹配。它优化了计算环节，但没有优化传输环节。

因此，缓存只是“减少计算的方法”，完全没有改变必须传输的流量。这是理解 [[OpenAI API转向WebSocket的根本原因]] 的关键前提。
