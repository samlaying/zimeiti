# 一体化原型

**一体化原型** 是指通过 [[Prototype Spec Editor Skill]] 将普通原型页面升级，使其自带**结构化、可编辑**的需求说明文档的原型形态。

- **核心特点**：它不再是单纯的视觉草图，而是将原型与 PRD 文档的功能融合，使原型本身成为一份可阅读、可维护的需求规格说明书。
- **信息结构**：需求说明按页面模块、字段等结构化方式生成，而非简单的文本段落。
- **可维护性**：支持在线编辑、删除和新增章节，且内置版本控制，可追溯修改历史。

这种形态解决了传统“原型 + 文档”双轨制的协作痛点，是提升产品经理交付效率的关键创新。
