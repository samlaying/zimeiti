# 构建个人文风Skill的方法

这是一个基于[[Obsidian]]和[[Codex]]构建动态写作系统的完整流程：

1.  **语料准备与DNA分析**：向AI提供3-4篇最具个人特色的代表作，指令其总结出你的“写作DNA”，识别如“口播文风”、“实操型”等具体特点。
2.  **思路讨论与方案设计**：不要直接让AI搭建，先与其讨论目标与范围。**拒绝创建宽泛的“去除AI味”Skill**，应按场景细分（如先建“口播文案风格校准Skill”），并确定[[Skill文档结构]]。
3.  **搭建Skill文档结构**：基于讨论，搭建由 `Skill.md`（核心规则与DNA）和 `Reference` 文件夹（语料库与关键的[[对比案例库]]）组成的系统。**强调轻量启动，边用边优化**。
4.  **测试、优化与长期迭代**：使用新Skill润色AI生成的文章进行测试。建立更新机制，持续将“AI稿”与“个人优化稿”的对比案例放入[[对比案例库]]，让系统持续学习进化。
