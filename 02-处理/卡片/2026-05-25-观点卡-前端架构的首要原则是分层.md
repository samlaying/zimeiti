# 前端架构的首要原则是分层

在构建复杂的前端应用时，如果只能选择一个最重要的架构原则，那就是**分层**。将代码严格划分为 UI 层、业务实体层和数据库/网络通讯层，并强制执行从上至下的调用关系，能够**降低至少30%-50%的代码复杂度**，并使功能迁移和扩展变得极其简单。
