# Pro Max Skill 的问题案例

**案例**：一个自称“Pro Max”、GitHub 热度近 5 万的 Design Skill，在评测中表现不佳，属于不推荐使用的反面案例。

**主要问题**：
- **西文大写滥用**：在需要用户阅读的大标题上使用大写字体（大写应仅用于弱化小标签），并搭配多种颜色，导致视觉混乱。
- **细节把控失败**：整体设计缺乏协调性，多次运行结果接近“AI翻车”。

**启示**：热度高不等于质量可靠。在选择时，必须通过 [[Vibe Coding 评测方法|实测]] 来检验其**细节处理**和**风格把控**能力，避免被名称或热度误导。
