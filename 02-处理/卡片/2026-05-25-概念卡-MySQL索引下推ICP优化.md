# MySQL索引下推ICP优化

## Index Condition Pushdown (ICP) 索引下推

这是MySQL 5.6引入的一种查询优化策略，旨在**减少存储引擎访问基表的次数以及MySQL Server访问存储引擎的次数**。

### 优化原理
- **传统模式**：存储引擎根据索引找到记录后，将整行数据返回给Server层，由Server层的WHERE条件进行过滤。
- **ICP模式**：将部分WHERE条件的过滤下推到存储引擎层。存储引擎在利用索引定位记录时，可以直接在索引层面判断这些条件是否满足，只将满足条件的记录返回给Server层。

### 应用场景
- 主要应用于`range`、`ref`、`eq_ref`、`ref_or_null`等查询。
- 对于[[MySQL联合索引优化策略]]中提到的包含低区分度字段（如`Status`）的联合索引查询，ICP理论上可以减少不必要的回表操作，但其优化效果取决于索引的区分度和数据分布。
