# 插件实现GitHub热板应用

UP主使用[[飞书多Agent协作插件]]，指挥飞书群内的四个AI机器人（产品、前端、后端、测试）协作开发了一个“GitHub热板分析”网页应用。

### 完整协作流程
1.  **任务分配**：用户（老板）向“产品”机器人下达指令，产品机器人主动艾特并分配任务给前端、后端和测试机器人。
2.  **方案生成**：各机器人独立回复方案（前端方案、技术方案、测试策略）。
3.  **方案汇总与排期**：产品机器人汇总方案，生成MVP功能清单和项目排期。
4.  **开发与测试**：机器人自主协作完成开发、测试、回归验证，并生成测试报告。
5.  **监控与交付**：产品机器人可设置定时任务监控进度，最终生成使用手册，实现产品完整交付。

### 特点
- 协作过程高度自动化，机器人通过互相艾特实现任务传递。
- 用户可随时介入引导或监督（引用回复并艾特特定机器人）。
