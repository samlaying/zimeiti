# 交互式rebase修改历史

**交互式 Rebase (`rebase -i`)** 是一个强大的历史编辑工具，允许你对一系列提交进行修改，如压缩（squash）、删除（drop）、重排序等。通常需要安装 **GitLens** 插件以获得友好的UI支持。

**通用操作步骤**：
1.  **定位起点**：在 GitGraph 中右键点击你希望开始影响范围的**那个提交节点**（不一定是分支头），选择 `Rebase Current Branch on This Commit`。
2.  **启用交互模式**：勾选 **`开启交互式变基`** 选项，点击 Yes。
3.  **在UI中编辑**：在 GitLens 提供的终端UI中，通过每个提交旁的下拉菜单选择操作：
    -   `Squash (s)`: 向上压缩，将当前提交合并到上一个提交。
    -   `Drop (d)`: 丢弃该提交。
    -   `Reorder`: 拖动调整提交顺序。
4.  **完成与推送**：点击 `Start Rebase`，解决可能出现的冲突，最后通过**强制推送**更新远程。
