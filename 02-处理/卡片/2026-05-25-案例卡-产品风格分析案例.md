# 产品风格分析案例

一个利用 [[AI设计规范文档]] 进行**现有产品设计迭代**的实操案例：

1.  UP主让AI分析自己产品的源码。
2.  AI在开源项目中没有找到完全匹配的设计文件，于是为该产品**生成了一个新的中文版 `design.md` 文件**。
3.  该文件内容详实，涵盖了视觉主题、产品气质、整体色彩与语义色、字体层级、排版、样式规范（如按钮、表格、开关等），以及“什么该做，什么不该做”的设计约束。
4.  同时，项目文件夹中生成了明亮模式和DUG模式的预览页面。

**效果**：预览页面的整体风格与原有产品高度类似，后续迭代均可基于此文件进行，保证了设计一致性。
