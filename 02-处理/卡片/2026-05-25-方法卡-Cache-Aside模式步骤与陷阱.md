# Cache Aside模式步骤与陷阱

## Cache Aside 模式（旁路缓存）

这是一种常用的缓存策略，核心在于**先更新数据库，再删除缓存**。

### 执行步骤
1.  读取数据时，先检查缓存。
2.  缓存命中则返回。
3.  缓存未命中，则查询数据库，将数据写入缓存，然后返回。
4.  更新数据时，先更新数据库，然后**删除**（而非更新）对应的缓存。

### 关键问题与风险
- **缓存删除失败**：如果删除缓存操作失败，会导致缓存中是旧数据，而数据库是新数据，造成不一致。
- **应对策略**：需要引入重试机制。但需考虑：
    - 重试间隔多久？
    - 重试多次仍失败怎么办？
    - 以上问题可能导致数据库和缓存的状态长时间不一致。
