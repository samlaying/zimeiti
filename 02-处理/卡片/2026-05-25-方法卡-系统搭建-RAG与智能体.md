# 系统搭建：RAG与智能体

根据任务复杂度选择系统架构：

- **简单问答用RAG**：标准[[RAG]]架构包含四个模块：
  1.  **查询改写**：将用户口语化问题转为更匹配书面文档的表达。
  2.  **检索**：从知识库中快速找到相关材料。
  3.  **精排**：从候选材料中挑选最相关的作为参考。
  4.  **答案生成**：基于参考材料生成最终答案。

- **复杂任务用智能体**：[[智能体]]具备规划、记忆、调用工具和执行行动的能力，适用于多步骤、需要外部交互的任务。

无论选择哪种架构，[[提示词工程]]都是基本功，核心原则是具体、示例化、明确输出格式和设定角色。
