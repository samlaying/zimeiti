# 标准作业流程与颗粒度

作为“[[Harness工程]]”的第三大原则，核心思想是**将项目拆分为版本，将版本拆分为阶段**，每个最小工作单元都遵循严格的闭环流程。

**核心流程**：“计划 -> 开发 -> 验收 -> 修复 -> 再验收”
1.  **规划**：与审计员讨论，形成《开发建议书》。
2.  **开发**：开发者阅读建议书并执行开发。
3.  **验收**：审计员编写测试脚本，模拟真实场景进行端到端测试，输出《审计报告》。
4.  **修复与再验收**：开发者阅读报告并修复问题，循环直至通过。

**颗粒度控制**：根据任务复杂度细化阶段，目的是**让Agent的注意力始终高度聚焦在当前最小任务单元上**，避免因任务过大导致“[[注意力稀释]]”。
