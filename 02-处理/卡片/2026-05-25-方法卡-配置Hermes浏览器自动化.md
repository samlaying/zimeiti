# 配置Hermes浏览器自动化

让Hermes能操作浏览器，实现信息检索、抓取和内容发布，相当于“给AI加上一双手和脚”。以WSL环境为例：

### 环境准备
1.  **检查Chrome版本**：需≥144版本。
2.  **创建调试模式浏览器**：
    - 复制Chrome快捷方式，重命名（如“调试模式”）。
    - 右键快捷方式 -> 属性 -> 目标，在原有路径后追加：`--remote-debugging-port=9222 --user-data-dir=“D:\你的路径\ChromeDebug”`。
    - 启动此快捷方式，确认“Remote Debugging Server”已启动。
3.  **配置WSL网络（仅WSL用户）**：在`~/.wslconfig`文件中添加镜像网络配置，然后重启WSL。

### Hermes配置与连接
1.  在Hermes中运行命令，配置Google Chrome的MCP信息（包括端口9222）。
2.  重启Hermes使配置生效。

### 测试与应用
- **测试**：让Hermes用“调试模式”打开网站或执行搜索。
- **应用**：可指令Hermes抓取资讯并生成HTML卡片报告（如[[抓取资讯生成卡片]]案例）。
