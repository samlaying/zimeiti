# 充电时自动杀死后台应用案例

这是一个在 [[MacroDroid]] 中创建“充电时杀死后台应用”自动化流程的完整案例，旨在实现充电时设备安静运行。

- **触发器**：设备状态 → 充电状态 → 已插入充电器。
- **动作**：设备控制 → 杀死所有应用。
- **约束条件**：时间 → 日落之后（可选，需授权定位）。

**实现效果**：当在日落之后插入充电器时，系统会自动杀死所有后台应用，减少耗电和干扰，实现“安静充电”。此案例演示了从[[触发器]]到[[动作]]再到[[约束条件]]的完整流程设计思路。
