# Harness（编排框架）

实现[[Token Efficiency]]杠杆的关键不是单一模型，而是**极其复杂的Harness（编排框架）**，用于实现合理的分工、规划、流程控制，达成无人值守、全自动的交付。

UP主实践的Harness关键设计包括：
1.  **基于DAG（有向无环图）的编排**：将大任务拆解为独立的、可在单上下文窗口内完成的小任务节点。
2.  **AI可迭代的编排**：构建基于CLI的系统，使编排、Agent、Prompt可复用，已跑通的编排成为知识库供AI优化。
3.  **抽卡式任务解决**：同一任务+编排可运行多次进行采样（Cherry-pick最佳结果），数据可用于优化特定Agent的技能。
4.  **模型量化评估**：同一编排可用不同模型组合运行，量化评估工具调用成功率、Token成本等指标。
