# Claude Code 四层上下文压缩算法

Claude Code 的上下文管理采用**四层递进压缩体系**，而非单一算法，这是其在长对话中能保留早期细节的核心。

- **第一层：微压缩**：纯规则操作，不调用大模型，清理旧的工具输出结果，但保留语义信息。
- **第二层：自动压缩**：基于算法自动压缩上下文。
- **第三层：传统压缩**：使用传统方法进一步压缩数据。
- **第四层：session memory压缩**：压缩会话记忆，优化长期对话管理。

核心原则是**优先使用廉价的规则操作，延迟昂贵的大模型调用**，以提高效率。这套算法比 [[OpenAI 的上下文管理]] 要优秀得多，解决了后者在长对话中容易遗忘早期细节的问题。
