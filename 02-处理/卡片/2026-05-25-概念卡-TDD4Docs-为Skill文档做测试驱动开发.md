# TDD4Docs：为Skill文档做测试驱动开发

将代码的测试驱动开发（TDD）思想完整映射到AI Skill文档的质量保证上，形成**TDD4Docs**方法论。

核心映射关系：
- 测试用例 → **压力场景**
- 生产代码 → **Skill文档本身**
- 测试失败 → 没有Skill时，Agent违规
- 测试通过 → 有Skill时，Agent在压力下遵守规则
- 重构 → 发现新漏洞后，补充Skill反驳内容

操作分为三个阶段：
1.  **Red（基线测试）**：在无Skill情况下，运行压力场景，**逐字记录**AI的真实违规方式和借口，作为Skill的“需求文档”。
2.  **Green（编写最小Skill）**：**只**针对基线记录的真实问题写Skill，保持精准、简短。过长可能降低合规率。
3.  **Refactor（迭代优化）**：继续运行场景，关闭AI新的绕过方式（如“精神vs字面”逃避、模糊措辞解读等）。
