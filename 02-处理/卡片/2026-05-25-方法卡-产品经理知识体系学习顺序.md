# 产品经理知识体系学习顺序

构建产品经理知识体系有明确的学习路径，共四个步骤：

1.  **岗位认知**：明确产品经理需要与哪些**业务角色**（如业务方、开发、设计）沟通与协作。
2.  **工具掌握**：熟练运用提升效率的工具，特别是**AI设计工具**（如Modal）和了解低代码平台。
3.  **行业理解**：深入了解目标行业的业务逻辑、核心痛点及服务群体，如电商、教育、金融等。
4.  **系统掌握**：透彻理解入职必会的**核心业务系统**（如CRM、WMS）的目的、功能及相互关联。

此顺序从“人”到“工具”，再到“业务”和“系统”，逐步构建完整的能力地图。
