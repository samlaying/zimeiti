# MCP协议的Token浪费痛点

当前的MCP协议存在一个显著的成本问题：每轮对话中，客户端必须将所有工具的完整schema（约120个Token/工具）一次性传递给AI，导致大量Token在工具未被调用时被浪费。

**量化分析**：
- 30个工具，15轮对话：浪费约54,000个Token。
- 120个工具，25轮对话：浪费高达360,000个Token。

**核心结论**：这不是一个技术缺陷，而是一个明确的「烧钱」问题，限制了在工具数量多的场景下的应用成本。
