# Redis内存危机快速修复三步法

在Redis内存爆满（OOM）导致服务崩溃后，可执行以下三步进行快速修复：

1.  **优化Redis配置**（5分钟）
    - 将`maxmemory-policy`从默认的`noeviction`改为`allkeys-lru`（LRU算法淘汰最久未使用的Key）。
    - 开启`lazy-free`，异步释放内存，防止重启后再次OOM。

2.  **为数据库接入层设置[[令牌桶限流]]**（15分钟）
    - 根据数据库安全QPS（如4000）计算限流阈值（如打8折取整至3000）。
    - 同时对低优先级接口进行临时降级。

3.  **前端增加兜底与重试策略**（10分钟）
    - 增加Loading状态和文案提示。
    - 实现[[指数退避重试]]（如3s->6s->12s...），并加入随机抖动避免重试风暴。
