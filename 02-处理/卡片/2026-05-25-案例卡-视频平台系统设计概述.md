# 视频平台系统设计概述

视频平台（如B站、抖音）的核心是一个复杂的[[分布式系统]]，其首要目标是处理每天数百万乃至上千万的[[大文件上传]]，并保证千万级用户能流畅在线观看。

系统设计的核心围绕四大能力展开：
1. **上传**：处理几GB到几十GB的大文件，必须支持[[分块上传]]和[[断点续传]]。
2. **转码**：将原始视频转换为多种分辨率（如4K， 1080P）和协议（如HLS， DASH），以适应不同设备与网络。
3. **存储**：存储原始视频与转码产物，数据量达到PB级。
4. **播放**：实现根据用户设备和网络情况动态切换资源的[[自适应播放]]。

这些能力共同构成了一个完整的视频平台技术底座。
