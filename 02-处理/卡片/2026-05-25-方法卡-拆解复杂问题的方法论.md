# 拆解复杂问题的方法论

面对复杂问题时，先拆解再行动，而非直接上手。核心方法是**横向拆解**（拆组成部分）与**纵向拆解**（在组成部分上拆具体事项）。以下是直接可套用的拆解套路：
1.  **按时间线拆解**：例如，将任务拆为“活动前、活动中、活动后”。
2.  **按参与方拆解**：例如，拆为“用户侧、平台侧、商家侧”。
3.  **按用户链路拆解**：梳理用户使用产品的完整路径，在某一环节设计新功能或优化转化率。
4.  **拆关键指标**：理解业务核心目标（如GMV），并将其拆解为可执行的具体指标（如点击率、下单率）。
