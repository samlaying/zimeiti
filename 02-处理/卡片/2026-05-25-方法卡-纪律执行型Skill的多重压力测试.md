# 纪律执行型Skill的多重压力测试

对于**纪律执行型**的Skill，必须使用**多重压力叠加**的场景进行测试，因为单一压力（如时间紧迫）下Agent通常能遵守规则。

**测试方法**：构建组合压力场景，例如：时间紧迫 + 沉没成本 + 权威施压 + 疲劳叠加。

**通过标准**：在最大叠加压力下，Agent仍能遵守规则，不找借口绕过。

**反例**：只用“项目有截止日期”这种单一压力测试，无法发现真正漏洞。需要设计如“已投入三小时+用户说只是小bug+曾有违规记录”的复杂场景。
