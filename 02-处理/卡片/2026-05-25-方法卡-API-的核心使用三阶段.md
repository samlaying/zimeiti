# API 的核心使用三阶段

使用 [[HTML-in-Canvas API：连接 DOM 与 Canvas 的桥梁]] 分为三个关键阶段：
1.  **设置 Canvas**：在要绘制的 `<canvas>` 标签上设置 `layoutsubtree` 属性，让浏览器感知 Canvas 内的内容并将其设置为可访问。
2.  **渲染 DOM 元素**：根据渲染上下文调用不同方法：
    *   2D 上下文：`drawElementImage(domElement, x, y)`
    *   WebGL：`texElementImage2D`
    *   WebGPU：`copyElementImageToTexture`
3.  **更新元素变换（至关重要）**：将 API 调用返回的变换应用到 DOM 元素的 CSS `transform` 属性。这告知浏览器元素在 Canvas 中的实际位置，**使交互和浏览器功能自动生效**。每次重绘时都必须更新，且停止绘制时必须移除元素。
