# 从盯代码到管项目

Symphony 与 Codex CLI、Claude Code 等工具的**核心差异在于入口**。

- **传统工具**：入口是**终端**。用户需要直接与 Agent 交互，指令、审查代码，过程是“**盯着 Agent 写代码**”。
- **Symphony**：入口是 **Linear 看板**。用户只需在 Linear 上创建和管理 ticket，Agent 便自动认领并执行任务，过程转变为“**管项目，让 Agent 自己开工**”。

这一转变的意义在于：**非技术人员（如产品经理、设计师、运营）** 只要能在 Linear 上提 ticket，就能调度 Agent 工作，极大地降低了项目管理的技术门槛。
