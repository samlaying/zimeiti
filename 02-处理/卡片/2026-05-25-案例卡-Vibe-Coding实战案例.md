# Vibe Coding实战案例

作者通过“[[Vibe Coding]]”从零开发一个Agent任务平台（类似opencode），完整实践了“[[Harness工程]]”思想。其开发历程演进如下：
1.  **初始**：与AI聊天形成初步规划和契约文档。
2.  **引入审计员**：为解决测试打断开发和额度消耗问题，引入第二个Agent（Codex）作为独立的规划与审计角色，建立基于文档的协作流程。
3.  **分离前端开发**：后端MVP完成后，将前端任务独立，并配备接口文档。
4.  **引入版本管理**：项目成型后，用阶段任务划分版本（如v0.2.0）。
5.  **角色专业化**：分别引入代码管理员和研究分析员角色。

这个过程表明，“[[Harness工程]]”并非预先设计，而是在实践中“遇山开路，遇水搭桥”逐步演化而成的。
