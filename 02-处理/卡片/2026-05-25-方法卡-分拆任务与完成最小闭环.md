# 分拆任务与完成最小闭环

## 核心理念
不直接实现全部功能，而是采用“小步快跑、持续验证”的迭代开发模式。

## 关键操作
1.  **分拆任务**：基于[[搭建核心架构]]，将大任务分解为可执行的小步骤。
2.  **完成最小闭环**：先实现核心主流程的最小可运行版本（MVP）。
3.  **过程控制**：每完成一小步，都必须进行验证，确认无误后再推进下一步。
4.  **上下文管理**：当对话上下文过长时，及时使用`Clear`或`Compat`等命令清理或压缩上下文，或新开对话，以避免AI信息过载和[[目标漂移]]。

## 迭代循环
重复“完成最小必要版本 -> 验证”的循环，直到所有任务完成。
