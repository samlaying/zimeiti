# 协作需统一rebase工作流

如果团队决定使用 [[rebase保持分支图简洁|rebase]] 来保持分支历史的线性与简洁，那么**所有协作者必须统一遵守该工作流**。关键实践包括：

1.  **拉取时变基**：在多人同一分支协作时，使用 `git pull --rebase` 命令拉取最新更改，避免产生不必要的合并节点。
2.  **禁止混合使用**：一旦分支中有人使用了 `merge`，该分支的历史就不再是完美单链条，后续通常无法再安全地使用 `rebase`。
3.  **沟通与规范**：需要团队明确规范并执行，否则“你合一下我合一下”的操作会使分支图迅速变得复杂。
