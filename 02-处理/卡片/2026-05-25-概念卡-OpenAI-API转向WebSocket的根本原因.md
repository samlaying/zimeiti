# OpenAI API转向WebSocket的根本原因

OpenAI API 的底层编排层是**无状态**的。请求可能被路由到不同的API节点和GPU，服务器无法低成本地在全局范围内持久化每个会话的状态。

因此，每一次请求都必须是“自包含”的，携带所有历史信息。这正是导致 [[AI智能体的“土拨鼠之日”传输问题]] 的根源——架构决定了必须重复传输。

解决方案是需要一种能**保证**客户端在一次生成周期内始终连接到同一台服务器的机制，从而让服务器可以安全地保持会话状态。
