# 故障排查：新机器人不回复

当新建的 [[OpenClaw]] 机器人不回复时，可按以下步骤排查：
1.  **查看日志**：执行 `tail -f ~/.openclaw/logs/openclaw.log` 查看实时日志。
2.  **定位OpenID**：在日志中找到新机器人收到的 `message` 信息，其 `OU` 标识即为该机器人的OpenID。
3.  **核对配置**：打开新机器人的配置文件，检查其中写入的OpenID是否与日志中的一致。**常见问题是新旧机器人的OpenID相同**，导致消息无法路由。
4.  **手动修正**：如不一致，需手动将日志中的正确OpenID替换到配置文件中，然后**重启网关**。
