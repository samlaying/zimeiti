# API通信模式解析

> 区分拉取推送和同步异步，避免沟通误解。

## 核心内容
拉取是下游系统主动获取数据，推送是上游系统主动发送数据；同步是实时反馈结果，异步是等待后回调通知。产品经理需理解这些模式以设计合理接口。

## 原文关键段落
> 拉取：下游系统主动向上游系统获取数据。推送：上游系统将数据主动发送给下游系统。同步：调用方发起请求后，实时得到结果反馈。异步：调用方发起请求后，无法立即得到最终结果，需等待回调通知。
