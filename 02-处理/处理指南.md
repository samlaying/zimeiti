# 逐字稿原子化处理指南

## 工作流总览

```
原始逐字稿 → 分析内容 → 提取原子卡片 → 添加链接 → 挂到MOC → 图谱生长
```

## 第一步：读取逐字稿

打开一篇逐字稿，重点关注：
- 标题和标签（frontmatter）
- 章节结构（## 标题）
- 关键观点、方法、案例、金句

## 第二步：提取原子卡片

### 判断标准

| 卡片类型 | 何时提取 | 举例 |
|---------|---------|------|
| **概念卡** | 出现核心术语/工具名 | "Claude Code Auto Mode是什么" |
| **方法卡** | 有明确的操作步骤 | "如何用GPT Image2生成Figma源文件" |
| **观点卡** | 有传播价值的判断 | "产品经理不需要怼研发，而是理解研发" |
| **案例卡** | 有完整的实践故事 | "0基础做刷题小程序并上架" |
| **金句卡** | 适合直接引用的表达 | "收藏的瞬间，大脑会产生已掌握的错觉" |

### 提取原则

1. **一卡一主题**：每张卡片只讲一个点
2. **自包含**：卡片本身能独立理解
3. **可链接**：主动标注与其他卡片的关系

## 第三步：写入卡片

### 文件命名

```
{日期}-{卡片类型}-{关键词}.md
```

示例：
- `20260525-概念卡-Claude-Code-Auto-Mode.md`
- `20260525-方法卡-GPT-Image2生成Figma.md`
- `20260525-观点卡-产品经理理解研发.md`

### Frontmatter 必填字段

```yaml
---
title: "卡片标题"
card_type: 概念卡  # 概念卡/方法卡/观点卡/案例卡/金句卡
layer: 处理
source: "[[原始逐字稿文件名]]"
date: 2026-05-25
tags:
  - 卡片/概念        # 必须：卡片类型
  - 主题/AI工具      # 必须：至少一个主题
  - 素材/观点        # 可选：素材用途
related:
  - "[[相关卡片]]"   # 可选：已知的关联
---
```

### 内容结构

参考 `02-处理/模板/` 下的模板文件。

## 第四步：添加双向链接

### 链接类型

1. **来源链接**：指向原始逐字稿
   ```markdown
   > 来源：[[2026-05-25-077-Canva AI生成不可编辑]]
   ```

2. **相关链接**：指向同主题的其他卡片
   ```markdown
   - 相关：[[GPT Image2生成Figma源文件]]
   ```

3. **MOC链接**：挂到对应的内容地图
   ```markdown
   - 属于：[[AI工具与Skills]]
   ```

### 链接写法

- 在正文中用 `[[文件名]]` 创建链接
- 文件名可以省略 `.md` 后缀
- 可以用 `[[文件名|显示文本]]` 自定义显示

## 第五步：更新 MOC

每张新卡片都应挂到对应的 MOC 文件下：

1. 打开 `02-处理/MOC/` 下对应的 MOC
2. 在合适的分类下添加 `- [[新卡片名]]`
3. 如果没有合适的 MOC，考虑创建新的

## 第六步：利用图谱

定期打开 Obsidian 图谱视图（Ctrl/Cmd+G），关注：

- **高连接节点** → 值得做专题内容
- **孤立节点** → 可能是独特切入角度
- **意外连接** → 爆款选题信号

## 快捷操作

### 批量处理

对同一篇逐字稿，可以一次提取多张卡片：

```
一篇逐字稿
  ├→ 概念卡：核心工具定义
  ├→ 方法卡：操作步骤
  ├→ 观点卡：核心判断
  ├→ 案例卡：实践故事
  └→ 金句卡：高光句
```

### 交叉链接

当两张卡片有关联时，双向都加上链接：

```
卡片A: "Canva AI生成可编辑元素"
  → related: [[GPT Image2生成Figma源文件]]

卡片B: "GPT Image2生成Figma源文件"
  → related: [[Canva AI生成可编辑元素]]
```

这样在图谱中会自然形成主题簇。
