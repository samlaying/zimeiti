# 逐字稿 → 原子笔记 处理指南

## 核心思路

一份20分钟的逐字稿里，可能藏着5-8个独立的、可复用的"知识点"。

你要做的是把它们从原始语境中抽离出来，变成独立的、打了标签的、可以和任何其他素材拼接的**原子笔记**。

## 工作流总览

```
原始逐字稿 → 素材领域分析 → 提取原子笔记 → 选题胚胎 → 补证据/实测 → 发布内容
```

## 第一步：提取原子笔记

### 判断标准

| 类型 | 何时提取 | 说明 |
|------|---------|------|
| **方法论** | 有明确操作步骤或框架 | 可直接指导行动 |
| **案例** | 完整的实践故事、真实案例 | 有细节、有结果 |
| **观点** | 有传播价值的判断、洞察 | 反常识或有深度 |
| **数据** | 具体的数字、比例、对比 | 有说服力的论据 |
| **故事** | 有情节的个人经历或见闻 | 有代入感 |
| **金句** | 适合直接引用的高光表达 | 原汁原味，不改写 |

### 提取原则

1. **独立性**：每条笔记脱离原文也能看懂
2. **可复用性**：标注可以用在哪些场景
3. **质量优先**：宁可少提取也不要凑数
4. **观点沉淀**："我的想法"栏是分水岭

## 第二步：填写原子笔记模板

### 文件命名

```
{日期}-{类型}-{关键词}.md
```

示例：
- `2026-05-25-方法论-公众号变现佣金模式.md`
- `2026-05-25-案例-租房vs买房成本计算.md`
- `2026-05-25-观点-财务自尊vs财务自由.md`

### Frontmatter

```yaml
---
title: "原子笔记标题"
note_type: 方法论  # 方法论/案例/观点/数据/故事/金句
domain: 自媒体运营  # 9大领域之一
source: "[[原始逐字稿文件名]]"
date: 2026-05-25
reusable: "具体说明可以用在什么场景"
tags:
  - 原子笔记
  - 类型/方法论
  - 领域/自媒体运营
---
```

### 内容结构

```markdown
# 标题

> 一句话记忆锚点

## 核心内容
（用自己的话写2-5句话概括）

## 原文关键段落
> （直接摘录最有价值的1-2段原文）

## 我的想法/延伸
（这个点跟你的什么经历/想法有关？可以怎么加工？）

## 可复用场景
（具体说明可以用在什么内容里）

> 来源：[[原始逐字稿文件名]]
```

## 第三步：按领域归类 → 选题池

9大领域：
1. AI工具/Skills
2. 编程/技术
3. 产品经理
4. 设计/PPT
5. 自媒体运营
6. 知识管理
7. 浏览器自动化
8. 视频制作
9. 理财/成长

运行 `--update-pool` 自动生成选题池文件。

## 第四步：让素材长成选题胚胎

当 `卡片/` 还没有原子笔记时，也可以直接从 `01-输入/素材` 孵化。脚本会按标题和正文做领域归类，优先挑选痛点强、可教程化、可实测、受众清晰的素材。

```bash
# 先看整个素材库领域分布
python3 process-transcript.py --analyze-materials

# 从原始素材里孵化10条选题胚胎
python3 process-transcript.py --incubate 10 --incubate-source raw

# 孵化时生成更完整的发布草稿骨架
python3 process-transcript.py --incubate 10 --incubate-source raw --draft

# 只预览优先级，不调用模型、不写文件
python3 process-transcript.py --incubate 10 --incubate-source raw --dry-run
```

选题胚胎会输出到 `02-处理/选题胚胎/`，每条都包含：
- 目标读者与痛点
- 新鲜观点
- 核心立场
- 3-5个钩子
- 还需要补的证据/实测
- 小红书/B站/公众号版本
- 发布草稿骨架
- 爆款评分与风险

## 第五步：4种内容变形

同一个知识点，可以变成4种不同的发布内容：

### 变形1：观点文（公众号/即刻/小红书）
```
一个反常识的观点 → 用案例/数据佐证 → 你的立场
```
例：「"不写代码"不是缺点，反而是一种超能力」

### 变形2：实操教程（B站/公众号）
```
我遇到了什么问题 → 我怎么解决的（带截图） → 你可以直接抄的步骤
```
例：「0基础，我是怎么把GitHub项目改造成自己想要的AI工具的」

### 变形3：信息差/对比测评（B站/公众号）
```
N个同类东西的横向对比 → 我的推荐 → 适用场景
```
例：「6种AI做PPT的工具，我全试了，只推荐这2个」

### 变形4：复盘/思考（公众号/即刻）
```
我做了什么 → 结果是什么 → 我学到了什么
```
例：「收藏了567个视频后，我发现自己的兴趣能被归类成9件事」

## 快捷操作

### 批量处理
```bash
# 处理所有未处理的逐字稿
python3 process-transcript.py --batch

# 只处理10篇
python3 process-transcript.py --batch 10

# 从第50篇开始
python3 process-transcript.py --batch --from 50
```

### 单篇处理
```bash
# 处理指定文件
python3 process-transcript.py "01-输入/素材/xxx.md"

# 强制重新处理
python3 process-transcript.py "xxx.md" --force
```

### 更新选题池
```bash
python3 process-transcript.py --update-pool
```

### 分析和孵化
```bash
# 生成素材分析
python3 process-transcript.py --analyze-materials

# 生成选题胚胎
python3 process-transcript.py --incubate 10 --incubate-source raw
```

### 查看状态
```bash
python3 process-transcript.py --status
```
