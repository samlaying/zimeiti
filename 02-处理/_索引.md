# 02-处理 — AI 处理规则

## 定位

这个目录**不放内容**，只放 AI 处理的规则和模板。是输入层和输出层之间的自动化桥梁。

## AI 处理流程

当用户说「我新加了这一篇」时，AI 执行以下步骤：

### 第一步：读取并分析

1. 读取输入层新文档的内容
2. 判断 `input_type`（素材/认知）
3. 分析具体内容类型

### 第二步：自动补充 frontmatter

根据分析结果，自动写入：

| 字段 | 说明 | 示例值 |
|------|------|--------|
| `type` | 内容类型 | 个人故事 / 教程 / 课程笔记 / 观点 / 原文 / 用户案例 |
| `tags` | 主题标签 | [AI, 编程, 自媒体] |
| `summary` | 一句话摘要 | "介绍了 Vibe Coding 的五步入门流程" |
| `related` | 关联文档 | [[另一篇相关文档]] |
| `publishable` | 是否可直接用于发布 | true / false |
| `suggested_platform` | 建议发布平台 | 公众号 / 小红书 / twitter |

### 第三步：判断是否生成输出草稿

| 判断 | 动作 |
|------|------|
| 内容完整、有明确受众 | 生成适配草稿 → 放到 `04-输出/` 对应平台目录 |
| 内容有价值但不完整 | 保留在输入层，标注待补充项 |
| 纯原始素材 | 保留在输入层，标注待加工 |

### 第四步：建立关联

- 用 `[[]]` 链接相关文档
- 同一主题的文档互相引用

## 内容类型判断标准

| 类型 | 判断依据 |
|------|---------|
| `个人故事` | 有明确的时间、地点、人物、事件经过、个人感悟 |
| `教程` | 面向读者的操作指南，有步骤、有示例 |
| `课程笔记` | 来自课程/播客/视频的结构化笔记 |
| `读书笔记` | 来自书籍的摘录和批注 |
| `观点` | 对某事的看法、立场、分析 |
| `原文` | 外部来源的原文复制，无个人加工 |
| `用户案例` | 完整的用户转变记录（问题→方法→结果） |
| `用户反馈` | 零散的好评/反馈 |

## 复盘处理

发布后的复盘数据也由 AI 处理：

1. 读取各平台的数据（阅读量、互动量等）
2. 分析 Top 内容的共同特征
3. 生成复盘结论
4. 记录到 `04-输出/复盘/`
