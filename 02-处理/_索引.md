# 02-处理 — 原子笔记处理中心

## 定位

这个目录**不放内容**，只放处理规则和输出结果。是逐字稿和发布内容之间的加工层。

## 核心思路

一份20分钟的逐字稿里，藏着5-8个独立的、可复用的"知识点"。

把它们从原始语境中抽离出来，变成独立的、打了标签的、可以和任何其他素材拼接的**原子笔记**。

## 处理流程

```
原始逐字稿 → 素材领域分析 → 提取原子笔记 → 选题胚胎 → 补证据/实测 → 发布内容
```

### 第一步：分析素材库

先看素材主要集中在哪些领域，以及哪些最值得优先孵化：

```bash
python3 脚本/process-transcript.py --analyze-materials
```

### 第二步：提取原子笔记

运行脚本自动提取：
```bash
# 单篇处理
python3 脚本/process-transcript.py "01-输入/素材/xxx.md"

# 批量处理
python3 脚本/process-transcript.py --batch

# 查看状态
python3 脚本/process-transcript.py --status
```

### 第三步：填写"我的想法"

每条原子笔记的"我的想法/延伸"栏是关键——这是从"搬运"变成"原创"的分水岭。

### 第四步：更新选题池

```bash
python3 脚本/process-transcript.py --update-pool
```

自动生成9个领域的选题池文件。

### 第五步：孵化选题胚胎

当 `卡片/` 为空时，可以直接从原始素材孵化：

```bash
python3 脚本/process-transcript.py --incubate 10 --incubate-source raw --draft
```

输出会包含目标读者、痛点、新鲜观点、钩子、证据缺口、平台版本和草稿骨架。

### 第六步：4种内容变形

| 变形类型 | 结构 | 适用平台 |
|---------|------|---------|
| 观点文 | 反常识观点 → 案例/数据佐证 → 你的立场 | 公众号/即刻/小红书 |
| 实操教程 | 遇到问题 → 解决方案（带截图） → 可抄步骤 | B站/公众号 |
| 信息差/对比测评 | N个同类对比 → 推荐 → 适用场景 | B站/公众号 |
| 复盘/思考 | 做了什么 → 结果 → 学到什么 | 公众号/即刻 |
