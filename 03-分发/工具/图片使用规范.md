# 文章图片使用规范

## 图片出现的位置

| 位置         | 类型          | 说明                                       |
| ---------- | ----------- | ---------------------------------------- |
| **封面图**    | 单独文件        | `--cover cover.png` 或 `--cover-media-id` |
| **正文插图**   | Markdown 内嵌 | 截图、示意图、效果图、对比图                           |
| **AI 生成图** | Markdown 内嵌 | `__generate:prompt__` 语法自动生成             |

## 三种图片引用语法

### 1. 本地截图/图片

适用：你自己截的图、本地准备好的图片

```markdown
![截图说明](./images/step1.png)
```

**规则：**
- 图片放在与 `.md` 同级的 `images/` 子目录下
- 路径用 `./images/xxx.png` 相对路径
- 支持 png/jpg/jpeg/gif/webp
- convert 时会自动上传到微信素材库并替换 URL

### 2. 在线图片

适用：引用网上的公开图片

```markdown
![图片说明](https://example.com/photo.jpg)
```

**规则：**
- 必须是可公开访问的 URL
- convert 时会自动下载 → 上传微信 → 替换 URL
- 不稳定的外链可能导致失败，优先下载到本地用语法 1

### 3. AI 生成图片（ChatGPT）

适用：需要 AI 根据文章内容生成的插图

```markdown
![示意图](__generate:一张展示 Vibe Coding 工作流的流程图，从用户输入到 AI 输出代码__)
```

**规则：**
- 格式严格为 `![任意alt](__generate:你的提示词__)`
- 双下划线包裹，提示词写具体
- convert --upload 或 --draft 时会调用 AI 生成 → 上传微信 → 替换
- 当前需配合 `IMAGE_API_KEY` 使用；手动流程下先用 ChatGPT 生成后下载到本地，改用语法 1

## 目录结构规范

```
20-公众号/
├── vibe-coding-入门指南.md
└── images/                          ← 与文章同名的图片目录（可选）
    ├── hero-screenshot.png          ← 手动截图
    ├── workflow-diagram.png         ← ChatGPT 生成后下载
    └── comparison-table.png         ← 其他插图
```

## 完整写作流程

### 写作时
1. 先写 Markdown 正文，需要插图的位置用三种语法之一标记
2. 截图存到 `images/` 目录
3. AI 图片可以先写 `__generate:prompt__` 占位，后续替换

### 发布前检查
```bash
# inspect 会列出所有图片及其状态
md2wechat inspect article.md
# 检查项：
#   - 本地图片是否存在（LOCAL_IMAGE_MISSING）
#   - 在线图片是否可访问
#   - AI 图片是否配了 IMAGE_API_KEY
```

### 发布
```bash
# API 模式（一键完成）
md2wechat convert article.md --mode api --theme ocean-calm \
  --draft --cover cover.png

# AI 模式 + 手动图片流程
# Step 1: 把 AI 占位图先用 ChatGPT 生成并下载
# Step 2: 替换 __generate:xxx__ 为本地路径 ./images/xxx.png
# Step 3: convert + draft
md2wechat convert article.md --mode ai --theme ocean-calm \
  --draft --cover cover.png
```

## 禁止事项

- 不用微信不支持的格式（bmp、svg、tiff）
- 单张图片不超过 10MB
- 不用含空格或中文的文件名（用短横线连接）
- 不引用需要登录才能访问的图片 URL
